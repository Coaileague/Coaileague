# AutoForce™

## Overview
AutoForce™ (Autonomous Workforce Management Solutions) is a comprehensive platform for emergency services and other service-related industries. Its core purpose is to streamline operations, reduce administrative overhead, and establish a single source of truth for workforce management. Key capabilities include advanced time tracking, automated invoice and payroll generation, smart hiring, compliance auditing, and real-time analytics. The platform operates on an "OS" design philosophy for extensibility and aims for a hybrid subscription and usage-based AI revenue model. The business vision is to provide autonomous financial operations while ensuring compliance and maintaining audit trails.

## System Architecture
AutoForce™ is built on a modular "OS" design philosophy, incorporating 6 major autonomous systems to promote clean code, automation, and extensibility. It features comprehensive Role-Based Access Control (RBAC) and Tier Gating across Free, Starter, Professional, and Enterprise tiers, with a two-tier role hierarchy.

### UI/UX Decisions
The platform features a professional aesthetic using Deep Charcoal, Platinum neutrals, and accent colors (Evergreen/Steel Blue/Professional Teal). It prioritizes a mobile-first approach with responsive layouts, accessible touch targets, and PWA capabilities. Branding includes an "AF" lightning bolt logo. Navigation features a floating header, visible labels, and contextual breadcrumbs. Responsive typography and a responsive table framework are key UI features. The navigation system uses a Gmail-style peek rail with three layout modes and a 6-family organization (CommOS™, OperationsOS™, BillOS™, IntelligenceOS™, AuditOS™, MarketingOS™) for structured menu navigation.

### Technical Implementations
-   **Frontend**: React, Vite, TypeScript, Wouter, TanStack Query, shadcn/ui, `react-hook-form`, `zod`.
-   **Backend**: Express.js, TypeScript, Zod for validation.
-   **Database**: PostgreSQL with Drizzle ORM.
-   **Authentication**: Custom session-based authentication supporting Replit Auth (OIDC) and Custom Auth, with account locking and password reset.
-   **Multi-Tenancy**: Data isolation managed on a workspace basis.
-   **External Identifier System**: Human-readable IDs (e.g., ORG-XXXX) for various entities, auto-generated with collision-resistant logic.
-   **RBAC Employee Tracking**: Unified employee data access via `useEmployee` hook across mobile and desktop for consistent RBAC enforcement.
-   **Autonomous Automation System**: Anchor-based biweekly scheduling for BillOS™ and OperationsOS™ ensuring consistent cadence for invoicing and payroll, with FLSA-compliant calculations.
-   **Security**: Stripe webhook validation, payroll data protection, strict Zod validation, workspace scoping, audit trails, comprehensive XSS protection via DOMPurify, IPv6-compliant rate limiting, and conversation-level access control for file uploads. DB transaction safety for all critical workflows.
-   **CommOS™ Workroom System**: Shift-linked room creation, multi-file upload system, automated room lifecycle management, participant management, comprehensive audit trail, and workspace-scoped access control.
-   **Premium Chat Features**: Real-time WebSocket chat with typing indicators, read receipts, participant tracking, and quick-insert macros. Automated WebSocket connection cleanup service. Secure typing indicator API. Enhanced WebSocket hook with Map-based state management.
-   **Mobile Experience**: Fully responsive mobile interface with auto-detection and dedicated mobile pages. Static HTML loading screen. Mobile chat and dashboard fully integrated with RBAC tracking.
-   **Responsive Loading System**: Viewport-aware loading screens via `ResponsiveLoading` wrapper component. Desktop users (≥768px) see `UniversalLoading` with AutoForce™ logo and blue/cyan wave animation. Mobile users (<768px) see `MobileLoading` with AF Core Scan radial progress, percentage display, and personalized messages. Deployed across 13 core pages (dashboard, clients, employees, settings, private-messages, query-os, budget-os, disputes, reports, engagement-dashboard, integration-os, time-tracking, training-os) for consistent branded loading UX.
-   **Navigation Protection System**: Reusable `useNavigationProtection` hook provides three-layer protection against accidental navigation from active chat sessions (beforeunload warnings, popstate handling, in-app route interception).
-   **Partner API Usage Tracking**: Middleware-based tracking system with caller-supplied deterministic IDs for idempotency.
-   **Cost Aggregation & Billing**: Automated cost calculation and Stripe invoice generation, aggregating AI usage and partner API costs per workspace with tier-based markup rates.
-   **Partner OAuth Integration**: Secure OAuth 2.0 implementation for QuickBooks Online and Gusto, featuring AES-256-GCM encryption, PKCE, CSRF protection, auto-refresh logic, and multi-tenant isolation.
-   **Unified Gemini AI Brain**: AutoForce™ uses Google Gemini 2.0 Flash Exp as the single AI provider for all autonomous features. **HelpOS™** (customer support AI) and **ScheduleOS™ Smart AI** (auto-scheduling engine) both leverage Gemini for cost-effective, intelligent automation. Automatic usage billing via `usageMeteringService` tracks tokens per workspace with feature keys `helpos_gemini_support` and `scheduleos_smart_ai`. **Critical Gemini Fix (Nov 2025)**: Fixed conversation history bug where Gemini API would reject history not starting with 'user' role. Added validation in `GeminiProvider.chat()` to ensure history always starts with 'user' or is empty, preventing "First content should be with role 'user', got model" errors.
-   **HelpOS™ 3-Tier Chat System**: Complete multi-level support system featuring: (1) FloatingSupportChat (MiniHelp) - anonymous guest chat with AI-powered escalation detection, (2) Guest Escalation Flow - captures name/email/issue, generates secure guest tokens, navigates to live HelpDesk with conversationId auto-join, (3) Universal HelpDesk - authenticated user chat with responsive mobile/desktop layouts and RBAC-based tool visibility. **Security**: Stable anonymous user IDs (`anon-${sessionId}`) prevent session hijacking, strict session ownership validation blocks cross-user access, security logging tracks attempted hijacks. Anonymous users receive deferred ticket creation (no database foreign key violations). Guest tokens enable unauthenticated access to specific conversations only. **Platform Workspace System (Nov 2025)**: 4-layer FK defense for anonymous escalations - (1) Fail-fast startup seeding with platform workspace creation, (2) Process termination on seed failure (exit code 1), (3) Concurrency lock serializes runtime seeding to prevent race conditions, (4) Runtime fallback re-seeds workspace if missing. Anonymous conversations use `customerId: null` (FK allows null) and `workspaceId: 'autoforce-platform-workspace'`. Authenticated users use real `customerId` and their workspace. Multi-auth support: Checks both Custom Auth (`req.session?.userId`) and Replit Auth (`req.user?.claims?.sub`) for proper authenticated/anonymous detection. E2E tested with full escalation flow validation.
-   **ScheduleOS™ Smart AI Approval Workflow (99% AI, 1% Human Governance)**: Complete autonomous scheduling system with confidence-based approval thresholds. AI generates shift assignments via Gemini 2.0 Flash Exp analyzing employee availability, skills, and workload balancing. Auto-approves schedules with ≥95% confidence (immediate shift assignment), requires human review for <95% confidence via proposal drawer UI. Legal disclaimer gating for <100% confidence proposals. Transaction-safe shift application prevents partial updates. Full audit trail with proposal persistence (`schedule_proposals` table) tracking AI responses, confidence scores, approver metadata, and post-approval edits for future learning. Frontend drawer component displays confidence factors, assignment reasoning, and approval controls.
-   **Schedule Migration via Gemini Vision (NEW)**: Multimodal AI-powered schedule extraction from external apps (Deputy, WhenIWork, GetSling). Upload PDFs or screenshots → Gemini Vision API performs OCR and table extraction → AI learns scheduling patterns from historical data → Preview extracted shifts with confidence scores → One-click import. Includes automatic pattern recognition (discovers soft constraints like employee preferences, workload balancing rules, client preferences). Billing via `scheduleos_migration` feature key. Service: `server/services/scheduleMigration.ts`, Endpoint: `POST /api/scheduleos/migrate-schedule`, UI: `ScheduleMigrationDialog` component integrated into ScheduleOS Smart page.
-   **Enhanced Constraint System (UPGRADE)**: Weighted constraint optimization framework distinguishing hard constraints (non-negotiable legal compliance: availability, double-booking prevention, rest periods, time-off requests) from soft constraints (penalty-weighted preferences: experience, workload balance, commute distance, employee preferences, clopening avoidance). Predictive metrics integration for reliability scoring, late history penalties, and absenteeism risk consideration. Enhanced confidence scoring reflects constraint satisfaction levels.
-   **AI-Powered Employee Scoring System (Nov 2025)**: Comprehensive weighted scoring algorithm (`server/services/automation/employeeScoring.ts`) for intelligent shift matching. **8 Scoring Factors**: Skills match (0.25), certifications (0.15), performance (0.15), reliability (0.10), distance (0.10), pay margin (0.10), OT risk (0.10), contractor status (0.05). **Hard Filters**: Availability verification, credential validation, OT threshold enforcement, distance limits. **Security Architecture**: Scoring service returns `fullEmployee` objects to prevent unvetted candidates reaching Gemini AI. AI Fill endpoint (POST `/api/shifts/:id/ai-fill`) uses vetted employee list exclusively—no re-fetching that could bypass filters. **Database Schema**: Normalized child tables (`employee_skills`, `employee_metrics`, `employee_certifications`) for rich employee profiles. Service calculates composite scores (0-1), formats top candidates for Gemini final selection, logs detailed scoring breakdowns for transparency.
-   **Fill Request Marketplace (Nov 2025)**: External contractor matching system for shifts with no qualified internal employees. Endpoint: POST `/api/shifts/:id/fill-request`. **Workflow**: Creates `shift_request` record → searches `contractor_pool` (filtered by distance/availability) → scores contractors (simplified algorithm) → sends `shift_offers` to top 3 matches (24hr expiry, 10% rate markup) → tracks offer status (pending/accepted/declined/expired). **Database Tables**: `contractor_pool` (external workers with skills, certs, rates, max distance), `contractor_skills`, `contractor_certifications`, `contractor_metrics`, `shift_requests` (marketplace search records), `shift_offers` (offers sent to contractors). **Known Technical Debt**: Contractor scoring uses simplified logic (should reuse employeeScoring service), no transaction safety for request+offers creation (risk of orphaned records on partial failure), fullEmployee field not strongly typed to Drizzle select type.
-   **Universal Responsive Schedule with Drag-and-Drop (Nov 2025)**: Full-featured scheduling system with desktop/mobile responsive architecture. **Desktop**: Interactive drag-and-drop via @dnd-kit/core - employees draggable from sidebar to grid cells, auto-creates shift draft with prefilled times (default 8hr), opens modal for review/save. Visual feedback (50% opacity drag, primary/10 drop zones). **Mobile**: Touch-optimized interface (`universal-schedule-mobile.tsx`) with day/week/list views, AI Fill integration, post orders, RBAC enforcement. Drag disabled on mobile (sensors={isTouchDevice ? [] : sensors}) for stable touch UX. **Responsive Routing**: Single `/schedule` route serves desktop or mobile component based on device detection (`isMobile` hook), inside ProtectedRoute wrapper for auth. **Components**: `DraggableEmployee` (useDraggable hook, maintains click-to-select), `DroppableSlot` (wraps grid cells, isOver visual feedback), `onDragEnd` handler (prefills shift form, opens modal, toast confirmation). **Employee Form**: Compact mobile-responsive design with RBAC sections (Basic Info, Contact, Employment Details), max-h-[90vh] scroll, h-9 inputs. **Week Filtering**: Both desktop and mobile fetch shifts with weekStart/weekEnd params for correct data scoping.
