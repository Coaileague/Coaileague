**MOBILE RULES — ENFORCE ON EVERY COMPONENT**

```
TYPOGRAPHY:
- Minimum font size: 16px for body text (prevents iOS auto-zoom)
- Minimum font size: 14px for labels and captions
- No text smaller than 12px anywhere
- Line height minimum 1.5 for body text
- No text truncation without visible ellipsis
- No text overflow outside its container
- No text bleeding over images or other elements
- Heading sizes scale down proportionally on mobile

SPACING AND LAYOUT:
- All fields full width on mobile (max-width: 100%)
- No side-by-side fields on screens under 480px
- Padding minimum 16px on all card and container sides
- No negative margins that cause overflow
- No fixed-width elements wider than viewport
- All flex containers wrap on mobile — no overflow scroll unless intentional

BUTTONS AND TAP TARGETS:
- Minimum height: 48px for all buttons
- Minimum width: 48px for all icon buttons
- Minimum tap target: 44x44px for any interactive element
- No buttons with text that overflows the button boundary
- No overlapping buttons
- Button text never truncated — button grows to fit text
- Sufficient spacing between adjacent buttons — minimum 8px
- Destructive buttons (delete, deactivate) visually distinct

FORMS AND INPUTS:
- Input height minimum 48px on mobile
- Input font size minimum 16px (prevents iOS zoom)
- Label above every input — never placeholder-only
- Error messages below field — never overlapping
- Error text in red with sufficient contrast
- Required field indicator visible
- Character counter visible when maxLength set
- Dropdown uses native select on mobile
- Date fields use native mobile date picker

NAVIGATION:
- Bottom navigation bar minimum 64px height
- Bottom navigation above safe-area-inset-bottom
- No content hidden behind bottom navigation
- Active tab clearly indicated
- Tab icons minimum 24px
- Tab labels minimum 10px and visible

MODALS AND SHEETS:
- Modals maximum 95vw width on mobile
- Modals maximum 90vh height with internal scroll
- Modal close button minimum 44x44px tap target
- Bottom sheets slide up from above bottom navigation
- Bottom sheets have handle indicator at top
- Bottom sheet content scrollable
- No modal content overflowing the modal boundary

CARDS AND LISTS:
- Cards full width on mobile
- Card content never overflows card boundary
- List items minimum 56px height for touch
- List item actions accessible on mobile
- Swipe actions labeled and visible
- Badge and chip text never overflows

IMAGES AND MEDIA:
- All images max-width: 100%
- Images maintain aspect ratio
- No images overflowing their container
- Avatar images perfect circle with proper clip
- Icon sizes consistent — 16px, 20px, 24px scales

SCROLLING:
- Vertical scroll only — no unintentional horizontal scroll
- Scroll containers have -webkit-overflow-scrolling: touch on iOS
- Fixed elements account for safe area insets
- No content hidden below the fold without scroll indicator

SPECIAL COMPONENTS:
- Signature pad: full-screen modal on mobile with clear instructions
- Flashcard: full-width card with flip animation, large text
- Quiz questions: one per screen, full width, large answer buttons
- Document viewer: full-screen with pinch zoom support
- PDF viewer: full-screen with mobile-optimized controls
- Camera upload: shows camera icon prominently, handles permissions gracefully
- GPS capture: clear permission prompt, graceful fallback
```
