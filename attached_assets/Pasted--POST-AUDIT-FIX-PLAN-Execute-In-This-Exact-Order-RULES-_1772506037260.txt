## TIER 1: LAUNCH BLOCKERS (Fix These Before Any Client Goes Live)

### Fix 1: Invoice tax calculation is a stub
- **Why it blocks launch:** Clients get invoices with wrong tax amounts. That's a billing error on day one.
- **Location:** Identified in GAPS_AUDIT.md
- **Action:** Find the tax calculation stub. Replace it with real state tax logic. Texas state sales tax is 6.25% but security services may be exempt — check. The `is_tax_exempt` field on clients (GAP-014) was already added. Make sure the tax calculation uses it. Verify with a test invoice showing correct tax for a taxable client and $0 tax for an exempt client.

### Fix 2: Invoice adjustment persistence is a stub
- **Why it blocks launch:** If a manager corrects a time entry after an invoice was generated, the invoice stays wrong. In production that means billing a client for 8 hours when the guard only worked 6.
- **Location:** Identified in GAPS_AUDIT.md — this is GAP-016
- **Action:** When a time entry is edited after being linked to a draft invoice, auto-void the draft invoice and regenerate it with the corrected amounts. If the invoice was already sent, create a credit memo or adjustment entry in the ledger. Verify with a test: create invoice → edit linked time entry hours → confirm invoice updates or adjustment is created.

### Fix 3: Ledger write atomicity
- **Why it blocks launch:** If the ledger write fails after payroll processes, you have payroll records with no matching ledger entry. The books don't balance.
- **Action:** Move the ledger write INSIDE the payroll transaction. If the ledger write fails, the entire payroll transaction rolls back. Same for billing — the invoice creation and ledger write should be in the same transaction. Verify by checking that every invoice and every payroll run has a matching ledger entry.

### Fix 4: Seeder overlap fix
- **Why it blocks launch:** Dev/sandbox testing produces false double-booking alerts that waste debugging time and erode confidence in the system.
- **Action:** Add overlap checking to all 4 seeder files. Before assigning an employee to a shift, check if they're already assigned to an overlapping shift. If yes, skip that assignment. This is sandbox-only but affects every developer and every test run.

Claim it in the app." This is the autonomous staffing feature (4 credits per assignment).

### Fix 8: Route-level RBAC audit
