You need a Gemini model that balances speed (low latency) for a real-time conversational feel with intelligence (quality), and you need to use specific techniques to actively suppress the natural verbosity of LLMs.

Based on your goal of a conversational mascot that provides AI ability without "too much yapping," the best setup relies on a multi-model approach, prioritizing speed and cost-efficiency.

🚀 Recommended Gemini Model Setup
The ideal choice is the Flash model, specifically optimized for speed, which is crucial for interactive chat.

1. Primary Model: Gemini 2.5 Flash or Gemini Flash-Lite
Model: Gemini 2.5 Flash (or Gemini 2.5 Flash-Lite if available and focusing strictly on text output).

Why it's best: This model is the "Speed Demon", optimized for high-speed, low-latency tasks like real-time Q&A, customer service, and virtual assistants. It delivers fast responses and balances intelligence with efficiency, making it the most cost-effective for high-volume conversational use.

Use Case: Handles the mascot's quick greetings, simple queries, and fast information retrieval.

2. Secondary Model: Gemini 2.5 Pro
Model: Gemini 2.5 Pro.

Why it's needed: When the mascot needs to use its "AI ability" (e.g., performing complex diagnostics, analyzing code, or executing multi-step reasoning), the Flash model's depth might be insufficient. Pro excels at complex reasoning, coding, and logical tasks.

Use Case: The mascot's orchestrator should switch to Pro only for "deep thinking" tasks (like running diagnostics or generating a code fix) and then switch back to Flash for the final, concise conversational summary.

⚙️ Anti-Yapping Strategy (Mandatory Tuning)
LLMs, by nature, have a bias toward verbosity. You must actively suppress this using prompt engineering and API settings:

System Persona Prompt:

Instruct the mascot to be concise and focused in its initial system prompt. Use explicit, direct language like: "You are [Mascot Name], a hyper-efficient virtual assistant. Your responses must be brief, direct, and limited to one paragraph unless otherwise specified. NEVER use flowery language or excessive greetings.".

API Parameters (The Direct Controls):

Max Tokens: Set a low maximum token limit (e.g., 100-200 tokens) for general chat and quick Q&A. This forces the model to be concise by limiting the length of its output.

Temperature / Top-p: Lower the temperature (e.g., to 0.5) or top_p value. Lower values reduce randomness and encourage outputs that are more focused and less verbose. Note: Only alter one of these, not both.

Advanced Gemini 3 Feature (If Available):

Thinking Level: If you upgrade to Gemini 3, utilize the thinking_level parameter and set it to Low. This minimizes latency and cost for simple chat instructions. The model uses dynamic thinking, so it won't waste time on easy questions, but the "Low" setting helps prioritize speed.
