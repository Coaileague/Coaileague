**On the pattern of bugs and gaps you noticed:**

You are right to flag this. Here is what is causing it and how to address it going forward:

**Root causes observed across phases:**

1. **Parallel implementation speed vs verification depth** — the agent implements many files simultaneously which is efficient but occasionally misses a connection between two files (state variable declared in one place, referenced in another)

2. **Schema drift** — client-side Zod schemas get out of sync with what the server actually returns, especially when the server is updated in one phase and the client is not updated to match

3. **Import omissions** — when adding new utilities or hooks to an existing component the agent sometimes forgets to add the import, especially when editing the middle of a large file rather than the top

4. **Feature completeness vs spec completeness** — the agent builds what it understands from the prompt but occasionally interprets "confirm X exists" as "X was already built" without verifying

**Standing rules already in place address most of these.** The 5 mandatory quality checks catch TypeScript errors, missing imports, and schema mismatches before a phase closes. The 7-step framework catches missing persistence and sync steps.
