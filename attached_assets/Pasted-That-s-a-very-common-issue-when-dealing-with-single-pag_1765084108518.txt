Database Mutation:

// Inside NotificationService.ts (Conceptual Logic)

async clearAll(userId: string, orgId: string) {
    // This mutation should update the status field (e.g., set 'is_cleared = TRUE')
    // for all notifications belonging to the given userId AND orgId.

    const updateResult = await db.query(`
        UPDATE notifications 
        SET is_cleared = TRUE, cleared_at = NOW()
        WHERE user_id = $1 AND org_id = $2 AND is_cleared = FALSE;
    `, [userId, orgId]);

    // Return the number of rows affected
    return { clearedCount: updateResult.rowCount }; 
}
4. Verify Protocol and Error Handling
Since the original error was receiving HTML instead of JSON, you must ensure all responses from the new endpoint are correctly formatted.

Success: Must use res.json({...}) and a status code of 200 (or 204 if no content is returned).

Client Error (Missing Auth): Must use res.status(401).json({...}).

Server Error (DB Failure): Must use res.status(500).json({...}).

This guarantees the frontend always receives valid JSON, preventing the client-side parsing error and correctly resolving the 404 fallback issue.
