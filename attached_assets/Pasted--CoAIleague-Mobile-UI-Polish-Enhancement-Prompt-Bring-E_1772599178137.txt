# CoAIleague Mobile UI Polish & Enhancement Prompt
## Bring Every Screen to Life — Enterprise-Grade, Not Prototype-Grade

**Date:** March 3, 2026
**Context:** The platform is functionally complete. The architecture, data flow, and features work. What's missing is visual polish, personality, and the feeling that this is a $899-$9,999/mo enterprise product — not a developer's side project. Every screen should feel like it belongs in the same family as the Invoices page (which is already good).

**IMPORTANT RULES:**
- Do NOT change any routing, data flow, API calls, or business logic
- Do NOT break any existing functionality
- Do NOT touch Trinity scheduling logic or credit/billing services
- ONLY enhance visual presentation, spacing, typography, color, micro-interactions
- Keep the existing dark theme — just make it richer and more intentional
- All changes must maintain mobile responsiveness

## CRITICAL LAYOUT BUGS — FIX BEFORE ANY POLISH

These are functional bugs that prevent users from interacting with the app. Fix all of these FIRST.

### BUG 1: Bottom Nav Overlapping Content (Multiple Pages)
The bottom navigation bar (Home, Schedule, Clock, Mail, More) and the floating action button (FAB) are overlapping tappable content on nearly every page. This makes buttons, toggles, and menu items physically impossible to tap.

**Root cause:** Pages are not accounting for the bottom nav bar height + FAB size in their scroll padding. Every scrollable page needs `padding-bottom` of at least `calc(bottom-nav-height + FAB-size + safe-area-inset-bottom + 16px)` — typically around 140-160px total.

**Affected pages (confirmed from screenshots):**
- Profile/Account sheet: "Logout" and "Cancel" buttons hidden behind bottom nav
- More menu: "LOG OUT" text literally cut off mid-word, HELPDESK row unreachable behind FAB
- Settings → Automation: "Shift Reminders" toggle covered by FAB — cannot be tapped
- Settings → Quick Toggles: Bottom toggles partially obscured by FAB
- Employees list: Last employee card clipped by FAB
- Shift Marketplace: Second shift card's "Pick Up" button hidden behind FAB
- Invoices: AR Aging chart data covered by FAB
- My Paychecks: Pay History card corner covered by FAB
- Availability: Saturday card partially cut off

**Fix:** Add a global bottom padding/margin to ALL scrollable page containers. The value should be at minimum 80px (nav bar ~56px + 24px breathing room), plus additional space if the FAB is present on that page. Approach:

```css
/* Apply to every main scrollable content area */
.page-content {
  padding-bottom: 160px; /* nav bar (56) + FAB (56) + badge (20) + spacing (28) */
}

/* For bottom sheets specifically (profile, More menu) */
.bottom-sheet-content {
  padding-bottom: calc(env(safe-area-inset-bottom) + 80px);
}
```

Alternatively, if using a layout wrapper component, add the padding there once and it propagates everywhere.

### BUG 2: FAB Positioning
The FAB with the red "8" badge sits in the bottom-right corner and overlaps interactive elements on every page. Two options:

**Option A (Recommended):** Move the FAB higher — position it above the bottom nav bar, not overlapping it. Use `bottom: calc(nav-bar-height + 16px)` so it floats in clear space.

**Option B:** Make the FAB contextual — only show it on pages where a primary creation action exists (Schedule → Create Shift, Messages → New Message, Reports → New Report). Hide it on pages where it doesn't serve a purpose (Settings, Paychecks, More menu). This also cleans up the visual clutter.

### BUG 3: Safety Checklist Checkboxes Not Rendering
On the Safety Checklist page, all checkboxes render as plain white/gray squares with no visible check state. Users cannot tell which items are checked vs unchecked. This is a component rendering bug — likely the custom checkbox CSS or the checked state prop isn't being applied.

**Fix:** Inspect the checkbox component. Ensure:
- Unchecked state: Dark outlined square with transparent/dark fill
- Checked state: Blue/teal filled square with white checkmark icon
- Tappable area: The entire row should be tappable (not just the tiny checkbox)
- Visual feedback: Row background should subtly change when checked (e.g., slight green tint or opacity reduction on the label text)

### BUG 4: More Menu "LOG OUT" Text Truncated
At the very bottom of the More menu (Image 17), "LOG OUT" is visibly cut off — you can see "LOG OU..." before it disappears behind the bottom nav. This is the same padding issue as Bug 1 but specifically bad here because:
1. Users NEED to be able to log out
2. If an examiner or prospect is evaluating the app and can't find logout, it looks broken
3. It's a security concern — officers sharing devices need reliable logout access

## GLOBAL DESIGN SYSTEM UPGRADES

### Color Palette Enhancement
The current palette is too monochrome — almost everything is the same dark blue-gray card on a slightly darker background. Add depth with these additions:

```
EXISTING (keep):
  Background:       #0a0f1a (dark navy)
  Card:             #111827 (slightly lighter)
  Primary Accent:   #3b82f6 (blue)
  Success:          #10b981 (green)
  Warning:          #f59e0b (amber)
  Danger:           #ef4444 (red)

ADD:
  Card Elevated:    #1a2332 (for cards that need to pop)
  Card Interactive: #162033 with 1px border #2a3a52 on hover
  Subtle Gradient:  linear-gradient(135deg, #1a2332 0%, #111827 100%) for hero cards
  Accent Teal:      #06b6d4 (already used in availability — use more broadly)
  Accent Purple:    #8b5cf6 (for AI/Trinity features)
  Accent Amber:     #f59e0b (for financial/billing features)
  Text Primary:     #f1f5f9
  Text Secondary:   #94a3b8
  Text Muted:       #64748b
  Border Subtle:    #1e293b
  Border Active:    #3b82f6
```

### Card System
Every card in the app should follow one of these 3 levels:

**Level 1 — Container Card** (sections, forms):
```css
background: #111827;
border: 1px solid #1e293b;
border-radius: 12px;
padding: 16px;
```

**Level 2 — Interactive Card** (employee cards, shift cards, notification cards):
```css
background: #1a2332;
border: 1px solid #2a3a52;
border-radius: 12px;
padding: 16px;
transition: border-color 0.2s, transform 0.1s;
/* On press/hover: border-color: #3b82f6; transform: scale(0.98); */
```

**Level 3 — Hero/Summary Card** (stats, KPI displays):
```css
background: linear-gradient(135deg, #1a2332 0%, #0f172a 100%);
border: 1px solid #2a3a52;
border-radius: 16px;
padding: 20px;
/* Add subtle inner glow for important stats */
box-shadow: inset 0 1px 0 rgba(255,255,255,0.05);
```

### Typography Scale
```
Page Title:      20px, font-weight 700, text-primary
Section Header:  14px, font-weight 600, text-secondary, uppercase, letter-spacing 0.05em
Card Title:      16px, font-weight 600, text-primary
Body:            14px, font-weight 400, text-primary
Caption/Meta:    12px, font-weight 400, text-muted
Stat Number:     28px, font-weight 700, text-primary
Stat Label:      12px, font-weight 500, text-secondary
```
