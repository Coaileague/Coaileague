### MODULE: Use of Force Doctrine

```
KNOWLEDGE FILE: use_of_force_security.md

CONTENTS:
- Use of Force Continuum (Security-Specific):
  1. Professional Presence (uniform, posture, demeanor)
  2. Verbal Commands (clear, direct, de-escalation focused)
  3. Soft Hand Control (escort holds, guided compliance)
  4. Hard Hand Control (restraint techniques — ONLY if authorized by contract + licensing)
  5. Intermediate Weapons (OC spray, baton — ONLY if licensed + authorized)
  6. Deadly Force (firearm — ONLY Level III/IV, ONLY per Penal Code Chapter 9)

- Key Legal Principles:
  - Security officers are NOT law enforcement
  - Citizen's arrest limitations by state
  - Duty to retreat vs stand your ground (varies by state)
  - Liability differences: company vs individual
  - Report documentation requirements after any use of force
  - When to call law enforcement vs handle independently

- TABC-Specific (Texas Alcoholic Beverage Commission):
  - 51% establishments (bars, clubs) — special rules
  - Security at licensed premises — cannot be employed by the establishment 
    as both security AND other roles simultaneously
  - Intoxicated person handling procedures
  - ID verification authority and limitations
  - Ejection procedures and documentation
  - Coordination with TABC enforcement

TRINITY USES THIS WHEN:
  - Training compliance verification
  - Incident report review (was use of force appropriate per doctrine?)
  - Post orders generation (site-specific force authorization levels)
  - Contract proposals (specifying authorized force levels per site type)
  - Legal risk assessment on incident reports
  - Flagging incidents that may need legal counsel review
```

### MODULE: Tax Knowledge by State

```
KNOWLEDGE FILE: state_tax_security.md

CONTENTS:

PAYROLL TAXES (per state):
  Federal (all states):
    - Social Security: 6.2% (employer) + 6.2% (employee) up to wage base $168,600 (2025)
    - Medicare: 1.45% each, no cap. Additional 0.9% employee above $200K
    - FUTA: 6.0% on first $7,000/employee (effectively 0.6% after state credit)
    
  Texas:
    - State income tax: NONE
    - SUTA (unemployment): 0.31% - 6.31% on first $9,000/employee (rate varies by experience)
    - Workers comp: NOT state-run, private insurance required
      - Armed security rate: ~$8-15 per $100 payroll
      - Unarmed security rate: ~$3-7 per $100 payroll
      - PPO/executive protection: ~$12-20 per $100 payroll
    
  California:
    - State income tax: 1% - 13.3% (progressive brackets)
    - SDI (State Disability): 1.1% on first $153,164
    - SUTA: 1.5% - 6.2% on first $7,000
    - ETT (Employment Training Tax): 0.1% on first $7,000
    
  Florida:
    - State income tax: NONE
    - SUTA: varies on first $7,000
    
  New York:
    - State income tax: 4% - 10.9%
    - NYC additional: 3.078% - 3.876% (if operating in NYC)
    - SDI: 0.511% up to $0.14/day
    - PFL (Paid Family Leave): 0.455%

SERVICE TAXES:
  Texas:
    - Security guard services ARE taxable under Texas Tax Code
    - Sales tax rate: 6.25% state + up to 2% local = max 8.25%
    - Must charge sales tax on invoices to clients
    - Exception: government contracts may be exempt
    
  California:
    - Security services generally NOT subject to sales tax
    - (Services vs tangible goods distinction)
    
  Florida:
    - Security services ARE taxable
    - Rate: 6% state + up to 1.5% local
    
  New York:
    - Security services ARE taxable
    - Rate: 4% state + up to 4.875% local

TRINITY USES THIS WHEN:
  - Calculating payroll (correct withholdings per state)
  - Generating invoices (add service tax where required)
  - Estimating labor costs for proposals (total burden rate)
  - P&L calculations (true cost per employee including all taxes + insurance)
  - Multi-state operations (different rules per operating state)
  - Financial forecasting
  - Recommending pricing (must account for tax burden)
```

## 2C: Knowledge Base Storage

```sql
-- ═══════════════════════════════════════════════════
-- TRINITY KNOWLEDGE BASE
-- ═══════════════════════════════════════════════════
CREATE TABLE IF NOT EXISTS trinity_knowledge_base (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  
  -- Scope
  scope VARCHAR(20) NOT NULL,
  -- Scopes: 'global' (all orgs), 'state' (state-specific), 'org' (org-specific)
  workspace_id UUID REFERENCES workspaces(id),     -- NULL for global/state scope
  state_code VARCHAR(2),                            -- NULL for global/org scope
  
  -- Content
  module_key VARCHAR(100) NOT NULL UNIQUE,          -- e.g., 'texas_security_1702', 'pricing_economics'
  title VARCHAR(300) NOT NULL,
  category VARCHAR(50) NOT NULL,
  -- Categories: regulation, licensing, tax, pricing, use_of_force, labor_law,
  --             insurance, contract_law, tabc, compliance, industry_standards
  content TEXT NOT NULL,                             -- The actual knowledge content (markdown)
  
  -- Versioning
  version INTEGER NOT NULL DEFAULT 1,
  effective_date DATE,                               -- When this knowledge became valid
  expiration_date DATE,                              -- When it expires (tax rates change annually)
  
  -- Metadata
  source VARCHAR(300),                               -- Where this knowledge came from
  last_verified_at TIMESTAMP,                        -- When was this last fact-checked
  
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_kb_scope ON trinity_knowledge_base(scope, state_code);
CREATE INDEX idx_kb_category ON trinity_knowledge_base(category);
CREATE INDEX idx_kb_module ON trinity_knowledge_base(module_key);

-- ═══════════════════════════════════════════════════
-- ORG-SPECIFIC KNOWLEDGE (Learned from uploads + operations)
-- ═══════════════════════════════════════════════════
CREATE TABLE IF NOT EXISTS trinity_org_knowledge (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  workspace_id UUID NOT NULL REFERENCES workspaces(id),
  
  -- What Trinity learned
  knowledge_type VARCHAR(50) NOT NULL,
  -- Types: uploaded_document, financial_pattern, operational_pattern,
  --        client_relationship, employee_pattern, pricing_history,
  --        expense_category, insurance_policy, contract_terms
  
  title VARCHAR(300) NOT NULL,
  summary TEXT NOT NULL,                             -- Trinity's summary of what she learned
  source_document_id UUID REFERENCES document_vault(id),  -- If learned from an upload
  source_data_query TEXT,                            -- If learned from platform data
  
  -- The actual knowledge (structured)
  knowledge_data JSONB NOT NULL,
  -- Examples:
  -- For insurance: {provider, policy_number, coverage_type, annual_premium, 
  --                 per_employee_rate, expiration_date, deductible}
  -- For workers_comp: {rate_per_100, classification_codes, carrier, annual_audit_date}
  -- For pricing: {client, bill_rate, pay_rate, margin, last_adjusted, contract_term}
  -- For pattern: {pattern_type, description, confidence, data_points, recommendation}
  
  -- Confidence and freshness
  confidence_score DECIMAL(3,2) DEFAULT 0.80,        -- 0.00 to 1.00
  last_validated_at TIMESTAMP,
  expires_at TIMESTAMP,                              -- Knowledge may become stale
  
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_org_knowledge ON trinity_org_knowledge(workspace_id, knowledge_type);
```

## 5B: Trinity Event Handlers

Trinity subscribes to events and decides what action to take:

```typescript
// server/trinity/eventHandlers.ts

trinityEventBus.on('invoice.overdue', async (event) => {
  const { invoice, client, days_overdue, amount } = event.data;
  
  if (days_overdue === 30) {
    // First reminder — gentle
    await trinity.executeAction({
      tool: 'draft_email',
      inputs: {
        purpose: 'payment_reminder',
        recipientContext: client,
        talking_points: [`Invoice #${invoice.number}`, `$${amount} outstanding`, `30 days past due`],
      },
      requiresApproval: true,
      approvalRole: 'manager',
    });
  }
  
  if (days_overdue === 45) {
    // Second notice — formal
    await trinity.executeAction({
      tool: 'draft_email',
      inputs: {
        purpose: 'formal_collection_notice',
        recipientContext: client,
        talking_points: [`Final notice before escalation`, `Service may be affected`],
      },
      requiresApproval: true,
      approvalRole: 'owner',
    });
  }
  
  if (days_overdue === 60) {
    // Alert owner — recommend service suspension discussion
    await trinity.executeAction({
      tool: 'send_notification',
      inputs: {
        recipients: ['owner'],
        title: `Client ${client.name}: $${amount} overdue 60 days`,
        body: `Recommend scheduling a call to discuss payment or service adjustment. Trinity can draft a service suspension notice if needed.`,
        type: 'urgent',
      },
    });
  }
});

trinityEventBus.on('document.uploaded', async (event) => {
  const { document, category } = event.data;
  
  // Auto-analyze uploaded documents to learn from them
  const learnableCategories = ['insurance', 'contract', 'license', 'financial', 'compliance'];
  
  if (learnableCategories.includes(category)) {
    await trinity.executeAction({
      tool: 'analyze_document',
      inputs: { documentId: document.id, documentType: category },
      creditCost: 10,
    });
  }
});

trinityEventBus.on('contract.expiring', async (event) => {
  const { contract, client, days_until_expiry } = event.data;
  
  if (days_until_expiry === 90) {
    // Start renewal process early
    const profitability = await trinity.executeTool('calculate_profitability', {
      entityType: 'client', entityId: client.id, period: 'last_12_months',
    });
    
    if (profitability.margin_percent < 25) {
      // Recommend rate increase with renewal
      await trinity.executeAction({
        tool: 'analyze_pricing',
        inputs: { clientId: client.id },
      });
    }
    
    await trinity.executeAction({
      tool: 'send_notification',
      inputs: {
        recipients: ['owner', 'manager'],
        title: `Contract Renewal: ${client.name} expires in 90 days`,
        body: `Current margin: ${profitability.margin_percent}%. ${profitability.margin_percent < 25 ? 'Rate increase recommended.' : 'Margins healthy.'} Trinity can draft a renewal proposal.`,
        type: 'info',
      },
    });
  }
});
```
