**PART 1 — ACTIONABLE NOTIFICATION ARCHITECTURE**

Every notification classified into one of these types and rendered accordingly:

**Type 1 — Informational**
No action required. Display only.
- Render as info card — no buttons
- Trinity also posts summary to Org Operations Briefing Channel if org-wide significance

**Type 2 — Navigational**
User needs to see something. Deep link button navigates to exact record.
- Example: "56 shift entries auto-approved — View Schedule →"
- Lands on specific record not just the tab

**Type 3 — Actionable**
User must approve, deny, or verify. Inline action buttons inside notification card. User never leaves panel.
- Approve / Deny with confirmation micro-interaction
- Action fires API call immediately on tap
- Card updates in real time
- Examples: timesheet approval, shift swap, expense approval, document signature
- On denial — Trinity auto-generates internal report with denial reason and routes to supervisor via notification AND email

**Type 4 — Document Pipeline**
Requires document signing, HR form, or report generation.
- Button opens document pipeline inline or navigates directly
- Trinity simultaneously sends email via Resend with document attached or linked for formal routing
- On manager denial — internal report generated, routed back to supervisor with denial reason via notification panel AND email trail
- Examples: onboarding documents, incident reports, disciplinary forms, contract signing

**Type 5 — Credit or Billing Action**
Requires upgrade, credit purchase, or billing decision.
- Direct link to billing or upgrade page
- Current credit balance shown in notification context before user acts

**Type 6 — Trinity Advisory**
Trinity has analyzed data and is advising action. Human verification always required.
- Trinity's reasoning summary displayed
- Confidence score displayed
- Always "Verify & Approve" never just "Approve"
- Trinity simultaneously posts advisory summary to Org Operations Briefing Channel so leadership is aware
- For anything touching money or client relations — verification step is mandatory and never skippable regardless of confidence score
