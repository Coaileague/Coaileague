# PART 1: DEACTIVATE/REACTIVATE SYSTEM

## 1.1 Concept

**Simple, unified terminology for access control:**

```
DEACTIVATE = Remove access (soft delete, data preserved)
REACTIVATE = Restore access (data intact, user can log in again)

NOT using: revoke, suspend, freeze, lock, ban, disable, archive
JUST using: deactivate / reactivate
```

## 1.2 Who Can Deactivate What

```
ROLE                    CAN DEACTIVATE
────                    ──────────────
Org Owner/Admin    →    Employees, Clients
Support Staff      →    Organizations, End Users
System (automated) →    Organizations (non-payment, maintenance, audit freeze)
```

## 1.3 What Happens on Deactivate

```
DEACTIVATE EMPLOYEE:
1. Set employee.status = 'deactivated'
2. Set employee.deactivatedAt = NOW()
3. Set employee.deactivatedBy = actorId
4. Invalidate all active sessions for this user
5. Emit WebSocket: 'employee:deactivated' → All org members see it
6. User immediately loses access (can't log in, API calls rejected)
7. Remove from active schedules (optional: reassign shifts)
8. Data preserved for audit/records

DEACTIVATE CLIENT:
1. Set client.status = 'deactivated'
2. Emit WebSocket: 'client:deactivated'
3. Client portal access revoked immediately
4. Contracts/invoices preserved but read-only

DEACTIVATE ORGANIZATION:
1. Set workspace.status = 'deactivated'
2. Set workspace.deactivatedAt, deactivatedBy, deactivationReason
3. Invalidate ALL sessions for ALL users in org
4. Emit WebSocket: 'workspace:deactivated' → Everyone kicked out
5. Data frozen for audit (can't modify, only view by support)
6. Billing paused
```

## 1.4 What Happens on Reactivate

```
REACTIVATE:
1. Set status = 'active'
2. Clear deactivatedAt, deactivatedBy
3. User can log in again
4. Emit WebSocket: 'employee:reactivated' / 'client:reactivated' / 'workspace:reactivated'
5. Access restored immediately
```

## 1.7 Real-Time Sync

```typescript
// When deactivating, IMMEDIATELY sync across all systems:

async function deactivateEmployee(employeeId: string, actorId: string, reason?: string) {
  return await db.transaction(async (tx) => {
    // 1. Update database
    const employee = await tx.update(employees)
      .set({
        status: 'deactivated',
        deactivatedAt: new Date(),
        deactivatedBy: actorId,
        deactivationReason: reason,
      })
      .where(eq(employees.id, employeeId))
      .returning();
    
    // 2. Invalidate sessions (immediate logout)
    await tx.delete(sessions)
      .where(eq(sessions.userId, employee[0].userId));
    
    // 3. Emit WebSocket (all devices see it)
    emitToWorkspace(io, employee[0].workspaceId, 'employee:deactivated', {
      employeeId,
      deactivatedBy: actorId,
      timestamp: new Date().toISOString(),
    });
    
    // 4. Emit to the user's devices (they get kicked)
    if (employee[0].userId) {
      emitToUser(io, employee[0].userId, 'session:terminated', {
        reason: 'Account deactivated',
      });
    }
    
    return employee[0];
  });
}
```

# PART 3: HELPAI BOT ORCHESTRATION SYSTEM

## 3.1 Concept

```
HELPAI = Supervisor Bot (always listening)
         │
         ├──► ReportBot (generates reports on command)
         ├──► ClockBot (handles time tracking commands)
         ├──► MeetingBot (schedules/manages meetings)
         ├──► ScheduleBot (shift management)
         └──► [Future bots...]

FLOW:
1. User types message in ANY chat room
2. HelpAI listens for keywords/commands or toggle triggers
3. HelpAI determines which bot to summon
4. HelpAI sends instructions to sub-bot
5. Sub-bot executes task
6. Sub-bot reports success/failure to HelpAI
7. HelpAI relays result to user
8. Sub-bot goes to rest mode
```

## 3.2 Why This Approach

```
OLD WAY (Room-based):
- Bot only works in specific rooms
- User must know which room to use
- Confusing for users

NEW WAY (HelpAI Orchestration):
- Works in ANY room
- User just talks naturally
- HelpAI figures out what they need
- More intuitive, less commands to remember
```

## 3.3 Architecture

```typescript
// server/services/bots/helpai-orchestrator.ts

interface SubBot {
  name: string;
  triggers: string[];  // Keywords that summon this bot
  execute: (context: BotContext) => Promise<BotResult>;
}

const subBots: SubBot[] = [
  {
    name: 'ReportBot',
    triggers: ['report', 'generate report', 'show me', 'analytics', 'stats'],
    execute: async (ctx) => {
      // Generate report
      return { success: true, message: 'Report generated', data: report };
    },
  },
  {
    name: 'ClockBot',
    triggers: ['clock in', 'clock out', 'punch in', 'punch out', 'time'],
    execute: async (ctx) => {
      // Handle time tracking
      return { success: true, message: 'Clocked in at 9:00 AM' };
    },
  },
  {
    name: 'MeetingBot',
    triggers: ['schedule meeting', 'meeting', 'calendar', 'book'],
    execute: async (ctx) => {
      // Schedule meeting
      return { success: true, message: 'Meeting scheduled' };
    },
  },
  {
    name: 'ScheduleBot',
    triggers: ['shift', 'schedule', 'assign', 'coverage', 'who is working'],
    execute: async (ctx) => {
      // Handle scheduling
      return { success: true, message: 'Schedule updated' };
    },
  },
];

class HelpAIOrchestrator {
  async processMessage(message: string, context: BotContext): Promise<void> {
    // 1. Check if message triggers any bot
    const matchedBot = this.findMatchingBot(message);
    
    if (!matchedBot) {
      // No bot needed, HelpAI handles directly or ignores
      return;
    }
    
    // 2. Summon the sub-bot
    await this.emitStatus(context, `Summoning ${matchedBot.name}...`);
    
    // 3. Execute the bot's task
    try {
      const result = await matchedBot.execute(context);
      
      // 4. Report result
      await this.emitResult(context, matchedBot.name, result);
      
    } catch (error) {
      // 5. Report failure
      await this.emitResult(context, matchedBot.name, {
        success: false,
        message: `${matchedBot.name} encountered an error: ${error.message}`,
      });
    }
    
    // 6. Bot goes to rest mode (nothing to do, just completes)
  }
  
  private findMatchingBot(message: string): SubBot | null {
    const lowerMessage = message.toLowerCase();
    
    for (const bot of subBots) {
      for (const trigger of bot.triggers) {
        if (lowerMessage.includes(trigger)) {
          return bot;
        }
      }
    }
    
    return null;
  }
}
```

  const baseX = window.innerWidth - 350;  // Right side

## 5.2 Cost Tracking Implementation

```typescript
// server/services/ai/cost-tracker.ts

interface AIUsage {
  workspaceId: string;
  model: 'gemini' | 'claude' | 'gpt-4' | 'gpt-3.5';
  inputTokens: number;
  outputTokens: number;
  cost: number;
  feature: string;  // 'trinity', 'report', 'scheduling', etc.
  actorType: 'END_USER' | 'SYSTEM' | 'AI_AGENT';
  actorId: string;
}

// Cost per 1K tokens (adjust to actual pricing)
const COST_PER_1K_TOKENS = {
  'gemini': { input: 0.0005, output: 0.0015 },
  'claude': { input: 0.008, output: 0.024 },
  'gpt-4': { input: 0.03, output: 0.06 },
  'gpt-3.5': { input: 0.0005, output: 0.0015 },
};

async function trackAIUsage(usage: AIUsage): Promise<void> {
  const pricing = COST_PER_1K_TOKENS[usage.model];
  
  const cost = 
    (usage.inputTokens / 1000) * pricing.input +
    (usage.outputTokens / 1000) * pricing.output;
  
  // Debit from org's credits
  await db.transaction(async (tx) => {
    // 1. Log the usage
    await tx.insert(aiUsageLogs).values({
      workspaceId: usage.workspaceId,
      model: usage.model,
      inputTokens: usage.inputTokens,
      outputTokens: usage.outputTokens,
      cost: cost,
      feature: usage.feature,
      actorType: usage.actorType,
      actorId: usage.actorId,
      createdAt: new Date(),
    });
    
    // 2. Debit credits
    await tx.insert(creditTransactions).values({
      workspaceId: usage.workspaceId,
      amount: -cost,
      type: 'ai_usage',
      description: `${usage.model} - ${usage.feature}`,
      actorType: usage.actorType,  // Must be uppercase!
      actorId: usage.actorId,
    });
    
    // 3. Update workspace balance
    await tx.update(workspaces)
      .set({ 
        creditBalance: sql`credit_balance - ${cost}`,
      })
      .where(eq(workspaces.id, usage.workspaceId));
  });
}
```

## 5.3 Wrap All AI Calls

// Every AI call must go through this wrapper

async function callAI(
  model: string,
  prompt: string,
  context: { workspaceId: string; feature: string; actorId: string }
): Promise<AIResponse> {
  const startTime = Date.now();
  
  // Make the AI call
  const response = await aiClient.complete(model, prompt);
  
  // Track usage and cost
  await trackAIUsage({
    workspaceId: context.workspaceId,
    model: model,
    inputTokens: response.usage.inputTokens,
    outputTokens: response.usage.outputTokens,

  });

}
