## PHASE 5 ŌĆö HANDLE CROSS-DOMAIN REFERENCES

Many tables reference tables in other domains via foreign keys. This is expected and fine.

### Rule for cross-domain imports:
Import from the domain file directly, not from the monolith:

```typescript
// In scheduling domain ŌĆö referencing employees from workforce domain
import { employees } from '../workforce';
import { sites } from '../clients';

export const shifts = pgTable('shifts', {
  id: uuid('id').primaryKey(),
  employeeId: uuid('employee_id').references(() => employees.id),
  siteId: uuid('site_id').references(() => sites.id),
  // rest of columns unchanged
});
```

### Cross-domain reference map (check these specifically):
- `shifts` ŌåÆ references `employees` (workforce) and `sites` (clients)
- `clock_events` ŌåÆ references `employees` (workforce) and `shifts` (scheduling)
- `payroll_runs` ŌåÆ references `employees` (workforce) and `pay_periods` (payroll)
- `timesheets` ŌåÆ references `employees` (workforce) and `pay_periods` (payroll)
- `trinity_jobs` ŌåÆ references `organizations` (orgs)
- `ai_usage_logs` ŌåÆ references `organizations` (orgs)
- `messages` ŌåÆ references `users` (auth) and `chat_rooms` (comms)
- `notifications` ŌåÆ references `users` (auth)
- `audit_log` ŌåÆ references `users` (auth) and `organizations` (orgs)
- `document_signatures` ŌåÆ references `documents` (compliance) and `employees` (workforce)
- `invoices` ŌåÆ references `organizations` (orgs) and `billing_plans` (billing)

For any reference not listed above ŌĆö import from the appropriate domain file.
