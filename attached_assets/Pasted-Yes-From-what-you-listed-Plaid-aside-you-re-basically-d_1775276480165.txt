What the agent can do for you with scripts

Yes, absolutely. The agent can do most of the prep work so you only do the manual console/env steps.

Have it build these:

1. scripts/prod/get-statewide-ids.ts

Purpose:

look up the Statewide workspace ID

look up the Statewide owner user ID

print the exact env var values you need to paste


Expected output:

GRANDFATHERED_TENANT_ID=...
GRANDFATHERED_TENANT_OWNER_ID=...

2. scripts/prod/validate-required-env.ts

Checks whether these are set:

GRANDFATHERED_TENANT_ID

GRANDFATHERED_TENANT_OWNER_ID

ALLOWED_ORIGINS

TWILIO_PHONE_NUMBER_SID

Plaid vars if payroll scope is enabled


3. scripts/prod/check-grandfathered-statewide.ts

Verifies:

Statewide workspace matches the env var IDs

Statewide is exempt from billing logic

Statewide billing page does not show paid-tenant prompts

no trial countdown

no downgrade/lockout flags


4. scripts/prod/check-inbound-email-readiness.ts

This can’t change DNS for you, but it can:

verify the app expects Resend inbound

verify the inbound webhook routes exist

verify subdomain parsing works

verify all 6 base tenant addresses are routed


5. scripts/prod/generate-db-role-sql.ts

Have the agent generate a safe SQL file for:

creating app_db_user

granting base permissions

revoking update/delete on immutable tables


That way you are not hand-editing SQL live.

6. scripts/prod/post-config-smoke.ts

Run this after you set env vars and DNS:

verify CORS origin is correct

verify Statewide exemption is active

verify health endpoint passes

verify billing page renders for Statewide

verify inbound email routes are ready

verify no missing prod envs remain
