**PASS 10 — SILENT FAILURE ELIMINATION AUDIT**

Find every place where an operation can fail without the user or system knowing.

**Patterns to search for:**

```typescript
// PATTERN 1 — Empty catch
try { await someOperation() } catch (e) {}

// PATTERN 2 — Console only catch on critical path
.catch(console.error)

// PATTERN 3 — Undefined field access
const total = invoice.totalAmount // field doesn't exist → undefined
Number(undefined) // → NaN

// PATTERN 4 — Missing await
db.insert(table).values(data) // not awaited — DB write silently skipped

// PATTERN 5 — Race condition SELECT-then-UPDATE
const record = await db.select()... 
// another request modifies record here
await db.update()... // updates based on stale read

// PATTERN 6 — Event inside transaction
await db.transaction(async tx => {
  await tx.insert(...)
  platformEventBus.publish(...) // fires even if tx rolls back
})

// PATTERN 7 — Missing rollback on multi-step operation
await db.insert(documents)...
await storage.upload(file)... // fails here — DB record exists with no file

// PATTERN 8 — NaN in financial calculation
const tax = parseFloat(rate) * amount // NaN if rate is null

// PATTERN 9 — Status transition without guard
await db.update(invoices).set({ status: 'paid' }) // no check that it was 'sent' first

// PATTERN 10 — Fire and forget notification
sendEmail(...) // not awaited, not caught
```

**For every instance found on a critical path — fix it:**
- Empty catches on financial operations → log to `trinity_credit_failures`
- Missing awaits on DB writes → add await
- Race conditions → add SELECT FOR UPDATE or atomic UPDATE WHERE
- Events inside transactions → move publish after transaction commit
- Multi-step without rollback → wrap in transaction
- NaN calculations → add `|| 0` fallback
- Missing status guards → add pre-flight check
- Fire and forget → wrap in try/catch with error logging

Report gap count. Fix before Pass 11.
