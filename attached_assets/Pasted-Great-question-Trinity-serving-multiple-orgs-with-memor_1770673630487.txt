Great question. Trinity serving multiple orgs with memory is a critical architectural piece. Here's how it works and what to check:

HOW AI MEMORY WORKS FOR MULTI-TENANT SYSTEMS
Three Types of Memory Trinity Needs:
1. Working Memory (Session Context)

Current conversation context
Active scheduling task state
Temporary calculations
Lives in RAM, cleared after session
Size: 8-32K tokens per active session

2. Episodic Memory (What Happened)

Past conversations with timestamps
Scheduling decisions and outcomes
Client feedback and issues
Staff performance notes
Size: 50-200MB per org (compressed)
Storage: Vector database + structured DB

3. Semantic Memory (What Trinity Knows)

Client preferences and patterns
Staff skills, reliability scores, preferences
Org-specific rules and policies
Seasonal patterns, recurring events
Size: 10-50MB per org
Storage: Vector embeddings + key-value store

## STORAGE RECOMMENDATIONS

| Memory Type | Storage Method | Size Per Org | Retention |
|-------------|---------------|--------------|-----------|
| Working Memory | Redis/RAM | 32KB active | Session only |
| Chat History | PostgreSQL + pgvector | 50-100MB | 90 days hot, archive after |
| Client Patterns | PostgreSQL | 10-20MB | Indefinite |
| Staff Profiles | PostgreSQL | 5-10MB | Indefinite |
| Vector Embeddings | pgvector or Pinecone | 50-200MB | Rolling 1 year |
| Learned Patterns | PostgreSQL JSON | 5-10MB | Indefinite, versioned |

**Total per org: ~150-350MB active, scalable**

For 100 orgs: ~15-35GB
For 1000 orgs: ~150-350GB

export class TrinityMemoryService {
  // Retrieve relevant context before any Trinity operation
  async getContextForSession(workspaceId: string, query: string): Promise<TrinityContext> {
    const [
      recentConversations,
      relevantClients,
      relevantStaff,
      orgContext,
      applicablePatterns
    ] = await Promise.all([
      this.getRecentConversations(workspaceId, 10),
      this.semanticSearchClients(workspaceId, query),
      this.semanticSearchStaff(workspaceId, query),
      this.getOrgContext(workspaceId),
      this.getApplicablePatterns(workspaceId, query)
    ]);
    
    return {
      recentConversations,
      relevantClients,
      relevantStaff,
      orgContext,
      applicablePatterns
    };
  }
  
  // Save conversation with embedding for future semantic search
  async saveConversation(params: SaveConversationParams): Promise<void>;
  
  // Update client intelligence after scheduling
  async updateClientIntelligence(workspaceId: string, clientId: string, update: Partial<ClientIntelligence>): Promise<void>;
  
  // Update staff intelligence after shift completion
  async updateStaffIntelligence(workspaceId: string, staffId: string, update: Partial<StaffIntelligence>): Promise<void>;
  
  // Learn and store new patterns
  async learnPattern(workspaceId: string, pattern: LearnedPattern): Promise<void>;
  
  // Semantic search using vector embeddings
  async semanticSearch(workspaceId: string, query: string, table: string, limit: number): Promise<any[]>;
}

async processRequest(workspaceId: string, userId: string, message: string) {
  // 1. Load memory context
  const memoryContext = await trinityMemory.getContextForSession(workspaceId, message);
  
  // 2. Build system prompt with memory
  const systemPrompt = this.buildSystemPromptWithMemory(memoryContext);
  
  // 3. Process with AI
  const response = await this.callAI(systemPrompt, message);
  
  // 4. Save conversation to memory
  await trinityMemory.saveConversation({
    workspaceId,
    userId,
    userMessage: message,
    assistantResponse: response,
    topic: this.extractTopic(message)
  });
  
  return response;
}
