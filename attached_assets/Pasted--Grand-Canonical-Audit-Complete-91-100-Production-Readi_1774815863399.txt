**SUBAGENT 1 — Float Precision in Invoice Calculations**

Financial calculations must use exact decimal arithmetic. Float imprecision causes penny errors that compound across thousands of invoices.

```bash
# Find all floating point math on financial values

  server/ --include="*.ts" | grep -v node_modules | \
  grep -v "FinancialCalculator\|formatCurrency\|display\|label\|string\|test" | \
  grep -i "amount\|price\|rate\|pay\|invoice\|total\|subtotal\|wage\|gross\|net" | head -30
```

For each violation found:
- Replace raw float math with `FinancialCalculator` service methods
- Verify all money stored as INTEGER cents in DB not DECIMAL dollars
- Verify all math uses integer arithmetic: `Math.round(amount * 100)` not `amount * rate`
- Run: `SELECT * FROM invoices WHERE subtotal != ROUND(subtotal, 2)` — must return zero rows

```typescript

// RIGHT:
```

**SUBAGENT 2 — Disputed Invoice Locking at Service Layer**

Disputed invoices must be locked at the service layer not just the route layer. Currently the dispute flag may only be checked in the route handler — if anything calls the service directly the lock is bypassed.

```bash
# Find the invoice service or mutation functions
grep -rn "updateInvoice\|patchInvoice\|invoiceService\." \
  server/ --include="*.ts" | grep -v node_modules | head -20

# Find where disputed status is checked
grep -rn "disputed\|DISPUTED\|status.*disputed\|disputed.*status" \
  server/ --include="*.ts" | grep -v node_modules | head -20
```

Fix: Add guard in the invoice update service function itself:

```typescript
// In invoice service — updateInvoice function
export async function updateInvoice(id: string, workspaceId: string, updates: any) {
  // Fetch current state
  const current = await db.query(
    'SELECT status FROM invoices WHERE id = $1 AND workspace_id = $2',
    [id, workspaceId]
  );
  
  if (!current.rows[0]) throw new NotFoundError('Invoice not found');
  
  // Service-layer lock on disputed invoices
  if (current.rows[0].status === 'disputed') {
    // Only allow: adding dispute notes, resolving dispute
    const allowedUpdates = ['dispute_notes', 'dispute_resolution', 'status'];
    const attemptedFields = Object.keys(updates);
    const blockedFields = attemptedFields.filter(f => !allowedUpdates.includes(f));
    
    if (blockedFields.length > 0) {
      throw new ConflictError(
        `Invoice is in dispute. Cannot modify: ${blockedFields.join(', ')}. ` +
        `Resolve dispute first.`
      );
    }
  }
  
  // Also lock voided invoices
  if (current.rows[0].status === 'void' || current.rows[0].status === 'voided') {
    throw new ConflictError('Voided invoices cannot be modified.');
  }
  
  // Proceed with update
  return db.query('UPDATE invoices SET ... WHERE id = $1', [...]);
}
```

```bash
# Must throw ConflictError regardless of how the service is called
```

**SUBAGENT 3 — Net Pay Floor in Garnishments**

When garnishments are applied an officer's net pay must never go below $0. Currently there may be no floor check meaning garnishments could theoretically produce negative net pay.

```bash
# Find payroll calculation code
grep -rn "garnishment\|garnishments\|netPay\|net_pay\|calculateNetPay" \
  server/ --include="*.ts" | grep -v node_modules | head -20

# Find where net pay is calculated
grep -rn "grossPay\|gross_pay\|deduction\|totalDeductions" \
  server/ --include="*.ts" | grep -v node_modules | head -20
```

Fix: Add explicit floor at zero:

```typescript
// In payroll calculation service
function calculateNetPay(grossPay: number, deductions: number[], garnishments: number[]): number {
  const totalDeductions = deductions.reduce((sum, d) => sum + d, 0);
  const totalGarnishments = garnishments.reduce((sum, g) => sum + g, 0);
  
  // Disposable earnings (federal minimum for garnishment calculation)
  const disposableEarnings = grossPay - totalDeductions;
  
  // CCPA limits garnishment to 25% of disposable earnings
  // or amount exceeding 30x federal minimum wage — whichever is less
  const federalMinWage = 7.25;
  const ccpaLimit = Math.min(
    disposableEarnings * 0.25,
    Math.max(0, disposableEarnings - (30 * federalMinWage))
  );
  
  // Apply garnishment but never exceed CCPA limit
  const actualGarnishment = Math.min(totalGarnishments, Math.max(0, ccpaLimit));
  
  // NET PAY FLOOR AT ZERO — never negative
  const netPay = Math.max(0, grossPay - totalDeductions - actualGarnishment);
  
  return netPay;
}
```

**SUBAGENT 4 — License Hard-Block on Clock-In**

Currently an officer with an expired license may receive a warning but still clock in. The correct behavior is a hard block — expired license means the officer cannot clock in and the system must notify their supervisor immediately.

```bash
# Find clock-in route handler
grep -rn "clock.*in\|clockIn\|clock_in" \
  server/routes/ --include="*.ts" | grep -v node_modules | head -20

# Find license check in clock-in flow
grep -rn "license.*expir\|expir.*license\|licenseCheck\|checkLicense" \
  server/ --include="*.ts" | grep -v node_modules | head -10
```

Fix the clock-in handler:

```typescript
// In time-entries route or service — clock-in handler
async function processClockIn(employeeId: string, workspaceId: string, shiftId: string) {
  
  // Step 1: Check license expiry — HARD BLOCK
  const employee = await db.query(`
    SELECT e.*, ol.expiry_date as license_expiry, ol.license_number
    FROM employees e
    LEFT JOIN officer_licenses ol ON ol.employee_id = e.id 
      AND ol.is_primary = true
      AND ol.status = 'active'
    WHERE e.id = $1 AND e.workspace_id = $2
  `, [employeeId, workspaceId]);
  
  const emp = employee.rows[0];
  if (!emp) throw new NotFoundError('Employee not found');
  
  // Hard block: expired license
  if (emp.license_expiry && new Date(emp.license_expiry) < new Date()) {
    // Notify supervisor immediately
    await notificationDeliveryService.send({
      workspaceId,
      type: 'officer_license_expired_clockin_blocked',
      data: {
        officerName: `${emp.first_name} ${emp.last_name}`,
        employeeNumber: emp.employee_number,
        licenseExpiry: emp.license_expiry,
        attemptedClockIn: new Date().toISOString()
      },
      targetRoles: ['supervisor', 'org_manager', 'org_owner']
    });
    
    throw new ValidationError(
      `Clock-in blocked: Your security license expired on ` +
      `${new Date(emp.license_expiry).toLocaleDateString()}. ` +
      `Your supervisor has been notified. ` +
      `Please renew your license before your next shift.`
    );
  }
  
  // Hard block: suspended officer
  if (emp.status === 'suspended') {
    throw new ValidationError(
      'Clock-in blocked: Your account is currently suspended. ' +
      'Please contact your supervisor.'
    );
  }
  
  // Hard block: terminated officer
  if (emp.status === 'terminated') {
    throw new ValidationError(
      'Clock-in blocked: This account is no longer active.'
    );
  }
  
  // Hard block: incomplete Tier 1 onboarding
  const incompleteBlocking = await db.query(`
    SELECT COUNT(*) FROM onboarding_tasks 
    WHERE employee_id = $1 
    AND tier = 1 
    AND status != 'completed'
    AND is_blocking = true
  `, [employeeId]);
  
  if (parseInt(incompleteBlocking.rows[0].count) > 0) {
    throw new ValidationError(
      'Clock-in blocked: You have incomplete required onboarding tasks. ' +
      'Please complete all Tier 1 tasks before clocking in.'
    );
  }
  
  // All checks passed — proceed with clock-in
  // ... rest of clock-in logic
}
```

**SUBAGENT 7 — Webhook Retry Count (4 vs spec's 5)**

The webhook delivery service retries 4 times but the spec requires 5 attempts with exponential backoff.

```bash
# Find webhook retry configuration
grep -rn "maxAttempts\|maxRetries\|retry.*count\|attempt.*5\|attempt.*4" \
  server/services/webhookDeliveryService.ts | head -10
```

Fix:
```typescript
// In webhookDeliveryService.ts
const MAX_RETRY_ATTEMPTS = 5; // Was 4 — update to 5

// Exponential backoff schedule:
// Attempt 1: Immediate
// Attempt 2: 5 minutes
// Attempt 3: 30 minutes  
// Attempt 4: 2 hours
// Attempt 5: 24 hours
// After attempt 5: mark as permanently_failed, notify org_owner

const RETRY_DELAYS_MINUTES = [0, 5, 30, 120, 1440];

function getNextRetryAt(attemptNumber: number): Date {
  const delayMinutes = RETRY_DELAYS_MINUTES[attemptNumber] || 1440;
  return new Date(Date.now() + delayMinutes * 60 * 1000);
}

// After 5th failure: notify org_owner
if (attemptNumber >= MAX_RETRY_ATTEMPTS) {
  await notificationDeliveryService.send({
    workspaceId,
    type: 'webhook_permanently_failed',
    data: { webhookUrl, eventType, totalAttempts: MAX_RETRY_ATTEMPTS }
  });
  
  await db.query(
    'UPDATE webhook_deliveries SET status = $1 WHERE id = $2',
    ['permanently_failed', deliveryId]
  );
}
```
