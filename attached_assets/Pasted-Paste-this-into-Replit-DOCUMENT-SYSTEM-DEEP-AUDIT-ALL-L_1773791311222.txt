**PASS 5 — 7-STEP PIPELINE AUDIT**

Fix every gap found in this pass before Pass 6.

Every document operation must follow the 7-step universal pipeline:

```
Step 1 — TRIGGER: What initiates the document operation?
Step 2 — FETCH: What data is retrieved to process the document?
Step 3 — VALIDATE: What validation runs before mutation?
Step 4 — PROCESS: What transformation or preparation happens?
Step 5 — MUTATE: What DB write or external API call executes?
Step 6 — CONFIRM: What confirms the operation succeeded?
Step 7 — NOTIFY: Who is notified and how?
```

Audit each of these document operations against the 7-step pipeline:

**Document upload:**
- Step 1: Upload triggered by user or Trinity
- Step 2: Fetch workspace settings, document category config, required field rules
- Step 3: Validate file type, file size, required metadata (name, category, assignedTo)
- Step 4: Process file — virus scan if configured, generate preview thumbnail, extract metadata
- Step 5: Write to storage, write DB record with all assignment fields
- Step 6: Confirm DB record created, confirm file is retrievable at stored URL
- Step 7: Notify assigned entity (employee or client), notify org admin, emit `document_uploaded` to platformEventBus

**Document signing request:**
- Step 1: Manager or Trinity triggers signature request
- Step 2: Fetch document, fetch signer contact info, fetch workspace e-signature settings
- Step 3: Validate document is in correct state for signing, validate signer has email
- Step 4: Prepare signature request payload for provider
- Step 5: Call e-signature provider API, update document status to `pending_signature`
- Step 6: Confirm provider accepted the request, confirm signer received email
- Step 7: Notify org owner that signature was requested, emit `document_signature_requested`

**Document signed (webhook callback):**
- Step 1: Webhook fires from e-signature provider
- Step 2: Fetch local document record by provider document ID
- Step 3: Validate webhook signature, validate document is in `pending_signature` state
- Step 4: Process signed document — download signed PDF from provider
- Step 5: Update document status to `signed`, store signed PDF, write audit trail
- Step 6: Confirm signed document is stored and retrievable
- Step 7: Notify org owner and signer, route to document safe, emit `document_signed`

For every step that is missing or incomplete — implement it.
