## PHASE 1 — ESTABLISH CURRENT STATE (READ ONLY)

### Step 1A — Generate complete current table list
Run this and output every table currently in the database, alphabetically:
```sql
SELECT table_name, 
       pg_size_pretty(pg_total_relation_size(quote_ident(table_name))) as size,
       (SELECT COUNT(*) FROM information_schema.columns 
        WHERE table_name = t.table_name) as column_count
FROM information_schema.tables t
WHERE table_schema = 'public'
ORDER BY table_name;
```
Save this output. Label it: **CURRENT TABLE LIST**

### Step 1B — Get original table list from git history
Run:
```bash
git log --oneline | head -20
```
Find the commit BEFORE the refactor started. Then:
```bash
git show [pre-refactor-commit-hash]:path/to/schema/file > /tmp/original_schema.sql
```
Or if the schema is tracked differently:
```bash
git diff [pre-refactor-commit-hash] HEAD -- . | grep "^-.*CREATE TABLE" > /tmp/dropped_tables.txt
cat /tmp/dropped_tables.txt
```
Save this output. Label it: **ORIGINAL TABLE LIST**

### Step 1C — Generate the diff
Compare ORIGINAL vs CURRENT and produce three lists:

**List A — TABLES THAT WERE DROPPED** (in original, not in current)
**List B — TABLES THAT WERE ADDED/RENAMED** (in current, not in original)  
**List C — TABLES THAT SURVIVED** (in both)

## PHASE 4 — FOREIGN KEY & RELATIONSHIP INTEGRITY

Run this to find broken foreign key references (tables referenced that no longer exist):
```sql
SELECT
    tc.table_name as referencing_table,
    kcu.column_name as referencing_column,
    ccu.table_name AS referenced_table,
    ccu.column_name AS referenced_column,
    tc.constraint_name
FROM information_schema.table_constraints AS tc
JOIN information_schema.key_column_usage AS kcu
    ON tc.constraint_name = kcu.constraint_name
JOIN information_schema.constraint_column_usage AS ccu
    ON ccu.constraint_name = tc.constraint_name
WHERE tc.constraint_type = 'FOREIGN KEY'
AND ccu.table_name NOT IN (
    SELECT table_name FROM information_schema.tables
    WHERE table_schema = 'public'
)
ORDER BY tc.table_name;
```

List every broken foreign key relationship found. These are features that will fail at runtime even if the code looks fine.
