# COAILEAGUE - CLIENT CONTRACT & EMPLOYEE PACKET WORKFLOW SYSTEM

## OVERVIEW

Build an online contract management and employee onboarding system that:
1. Converts the SPS General Contract for Services into a fillable, signable online form
2. Creates state-required employee onboarding packets
3. Integrates with Trinity AI for automated shift building and decision making
4. Stores signed documents with audit trails
5. Triggers workflows upon signature completion

 * SIGNATURE ENDPOINT:
 * 
 * POST /api/contracts/:id/sign
 * 
 * Body:
 * {
 *   signatureData: string, // Base64 image
 *   signerName: string,
 *   signerTitle: string,
 *   signerEmail: string,
 *   agreedToTerms: boolean,
 *   agreedToElectronicSignature: boolean
 * }
 * 
 * Captures from request:
 * - IP address (req.ip)
 * - User agent (req.headers['user-agent'])
 * - Timestamp (server time)
 * 
 * Actions:
 * 1. Validate contract exists and is in 'sent' status
 * 2. Save signature data
 * 3. Generate signed PDF with signature embedded
 * 4. Update contract status to 'signed'
 * 5. Send confirmation email to both parties
 * 6. Trigger post-signature workflow (see below)
 */

## PART 4: TRINITY INTEGRATION

### 4.1 Contract Data for Trinity Decision Making

```typescript
/**
 * TRINITY USES CONTRACT DATA FOR:
 * 
 * 1. SHIFT BUILDING
 *    Input: Contract schedule requirements
 *    Output: Proposed shifts matching requirements
 *    
 * 2. EMPLOYEE ASSIGNMENT OPTIMIZATION
 *    Consider:
 *    - Employee qualifications (armed/unarmed license)
 *    - Employee pay rate vs contract bill rate (profitability)
 *    - Employee location vs site location (minimize travel)
 *    - Employee availability
 *    - Employee preferences
 *    
 * 3. PROFITABILITY ANALYSIS
 *    Calculate per shift:
 *    - Revenue: hours × contract rate
 *    - Cost: hours × employee rate + burden
 *    - Margin: revenue - cost
 *    - Flag low-margin shifts
 *    
 * 4. SCHEDULING CONFLICTS
 *    Detect and resolve:
 *    - Double-booked employees
 *    - Uncovered shifts
 *    - Overtime situations
 *    
 * 5. COMPLIANCE MONITORING
 *    Alert when:
 *    - Employee license expiring
 *    - Firearms qualification due
 *    - Insurance renewal needed
 */

interface TrinitySchedulingInput {
  contractId: string;
  dateRange: { start: string; end: string };
  
  requirements: Array<{
    siteId: string;
    siteAddress: string;
    coordinates: { lat: number; lng: number };
    dayOfWeek: number;
    startTime: string;
    endTime: string;
    officersRequired: number;
    requiresArmed: boolean;
    billRatePerHour: number;
  }>;
  
  availableEmployees: Array<{
    employeeId: string;
    name: string;
    homeAddress: string;
    coordinates: { lat: number; lng: number };
    isArmedCertified: boolean;
    payRatePerHour: number;
    availability: Array<{ dayOfWeek: number; available: boolean }>;
    preferences: {
      maxDistanceMiles: number;
      preferredSites: string[];
      avoidSites: string[];
    };
  }>;
}

interface TrinitySchedulingOutput {
  proposedSchedule: Array<{
    shiftId: string;
    siteId: string;
    date: string;
    startTime: string;
    endTime: string;
    assignedEmployeeId: string;
    
    financials: {
      billableHours: number;
      revenue: number;
      employeeCost: number;
      margin: number;
      marginPercentage: number;
    };
    
    logistics: {
      employeeDistanceToSiteMiles: number;
      estimatedTravelMinutes: number;
    };
  }>;
  
  unfilledShifts: Array<{
    requirement: any;
    reason: string; // 'no_qualified_employees', 'no_available_employees', etc.
  }>;
  
  warnings: Array<{
    type: string;
    message: string;
    shiftId?: string;
    employeeId?: string;
  }>;
  
  summary: {
    totalShifts: number;
    filledShifts: number;
    unfilledShifts: number;
    totalRevenue: number;
    totalCost: number;
    totalMargin: number;
    averageMarginPercentage: number;
  };
}
```
