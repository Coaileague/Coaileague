# Autoforce — Replit Setup Guide (End‑User Pays All)

This guide gives copy‑paste steps, code stubs, and checks so your **autonomous** schedule→invoice→payroll features work on Replit. It assumes **Node.js + Express + Postgres** and **Stripe Billing**. Users connect **their own** QuickBooks/Gusto/Paychex; your app orchestrates via OAuth + webhooks.

---

## 0) TL;DR Launch Steps

1. **Fork Replit** and set Environment Secrets (see §2).
2. **Create Postgres** (Neon/Supabase/Railway) and run **SQL migrations** (see §3).
3. **Install** dependencies: `npm i` (see §4 for `package.json`).
4. **Stripe Billing**: create product + prices; set webhooks (see §6).
5. **Register OAuth apps** for QBO/Gusto/Paychex; set redirect URIs (see §5).
6. **Paste server skeleton** (Express app, queues, health, OAuth routes, webhooks) from §4–§7.
7. **Seed demo org** + sample data (see §9).
8. **Verify health** endpoint shows `safe_to_run: true` (see §8).
9. **Run chaos tests** (see §10). If clean, invite pilots.

---

## 1) Architecture Snapshot

* **DB**: Postgres with multi‑tenant tables (`org_id` everywhere)
* **Auth**: JWT; Support impersonation via `x-impersonate-org-id` logged to `audit_logs`
* **Integrations**: OAuth 2.0 (QuickBooks, Gusto, Paychex), webhooks to mirror status
* **Automation**: Job queues (BullMQ) with idempotency keys on all writes
* **Billing**: Stripe Checkout + metered usage (user pays partners separately)
* **Reports**: REST endpoints (Payroll, Billable Hours, Invoices) + CSV/PDF exports
* **Oversight**: 1% approval queue with rules + audit trail

## 3) SQL Migrations (copy‑paste)

Run this in your Postgres (psql/GUI):

```sql
-- Tenants
CREATE TABLE organizations (
  id uuid PRIMARY KEY,
  name text NOT NULL,
  timezone text DEFAULT 'America/Chicago',
  stripe_customer_id text,
  stripe_subscription_id text,
  billing_status text DEFAULT 'inactive', -- inactive|active|past_due|canceled
  created_at timestamptz DEFAULT now()
);

-- OAuth connections
CREATE TABLE connections (
  id uuid PRIMARY KEY,
  org_id uuid NOT NULL REFERENCES organizations(id),
  provider text NOT NULL CHECK (provider IN ('qbo','gusto','paychex')),
  access_token text NOT NULL,
  refresh_token text,
  scope text,
  expires_at timestamptz,
  external_company_id text, -- e.g., QBO realmId / Gusto company id
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
CREATE INDEX connections_org_provider_idx ON connections(org_id, provider);

-- Core
CREATE TABLE clients (
  id uuid PRIMARY KEY,
  org_id uuid NOT NULL REFERENCES organizations(id),
  name text NOT NULL,
  qbo_customer_id text,
  created_at timestamptz DEFAULT now()
);
CREATE INDEX clients_org_idx ON clients(org_id);

CREATE TABLE workers (
  id uuid PRIMARY KEY,
  org_id uuid NOT NULL REFERENCES organizations(id),
  name text NOT NULL,
  gusto_employee_id text,
  paychex_employee_id text,
  created_at timestamptz DEFAULT now()
);
CREATE INDEX workers_org_idx ON workers(org_id);

CREATE TABLE shifts (
  id uuid PRIMARY KEY,
  org_id uuid NOT NULL REFERENCES organizations(id),
  worker_id uuid NOT NULL REFERENCES workers(id),
  client_id uuid NOT NULL REFERENCES clients(id),
  start_at timestamptz NOT NULL,
  end_at timestamptz NOT NULL,
  hours numeric(7,2) GENERATED ALWAYS AS ((extract(epoch from (end_at - start_at))/3600.0)) STORED,
  rate numeric(10,2) NOT NULL,
  approved_bool boolean DEFAULT false,
  flagged_reason text,
  created_at timestamptz DEFAULT now()
);
CREATE INDEX shifts_org_date_idx ON shifts(org_id, start_at);

-- Invoices mirror
CREATE TABLE invoices (
  id uuid PRIMARY KEY,
  org_id uuid NOT NULL REFERENCES organizations(id),
  client_id uuid NOT NULL REFERENCES clients(id),
  provider text NOT NULL DEFAULT 'qbo',
  external_id text,
  issue_date date,
  due_date date,
  status text,
  total numeric(12,2),
  paid_total numeric(12,2),
  created_at timestamptz DEFAULT now()
);
CREATE INDEX invoices_org_dates_idx ON invoices(org_id, issue_date);

CREATE TABLE invoice_line_items (
  id uuid PRIMARY KEY,
  invoice_id uuid NOT NULL REFERENCES invoices(id),
  shift_id uuid REFERENCES shifts(id),
  hours numeric(7,2),
  rate numeric(10,2),
  amount numeric(12,2)
);

-- Payroll mirror
CREATE TABLE payroll_runs (
  id uuid PRIMARY KEY,
  org_id uuid NOT NULL REFERENCES organizations(id),
  provider text NOT NULL CHECK (provider IN ('gusto','paychex')),
  external_id text,
  period_start date,
  period_end date,
  gross_total numeric(12,2),
  status text, -- draft|submitted|paid|failed
  created_at timestamptz DEFAULT now()
);
CREATE INDEX payroll_runs_org_period_idx ON payroll_runs(org_id, period_start, period_end);

CREATE TABLE payroll_earnings (
  id uuid PRIMARY KEY,
  payroll_run_id uuid NOT NULL REFERENCES payroll_runs(id),
  worker_id uuid NOT NULL REFERENCES workers(id),
  hours numeric(7,2),
  rate numeric(10,2),
  gross numeric(12,2)
);

-- Audit
CREATE TABLE audit_logs (
  id uuid PRIMARY KEY,
  org_id uuid,
  actor_id uuid,
  actor_role text,            -- user|support|webhook|system
  action text,                -- e.g., REPORT_VIEW|OAUTH_CONNECT|WEBHOOK
  entity text,                -- invoice|payroll|shift
  entity_id text,
  payload jsonb,
  created_at timestamptz DEFAULT now()
);
CREATE INDEX audit_logs_org_time_idx ON audit_logs(org_id, created_at DESC);
```

## 5) OAuth Quick Setup Notes

* **QuickBooks**: create app in Intuit Developer → set Redirect URI to `${APP_BASE_URL}/oauth/callback/qbo` and subscribe to **Accounting** scopes; enable webhooks; capture `realmId`.
* **Gusto**: register app; request payroll scopes; capture **company_id** after connect.
* **Paychex**: partner onboarding often required; stub now, wire later.
* **Storage**: save `access_token`, `refresh_token`, `expires_at`, `external_company_id` in `connections`.
* **Refresh**: cron job or on‑demand refresh before calls; pause automations if refresh fails.

**Usage metering**

* Nightly job counts **processed shifts** per org and posts usage to `price_shift_1`.

## 7) Job Queue & Idempotency

* Use **BullMQ** with Redis for write jobs (QBO invoice create/send, Gusto payroll create/submit).
* **Idempotency**: Compute hash of (org_id, provider, action, payload) → store in Redis for 48h; skip repeats.
* **Retries**: exponential backoff with jitter; max 5.
* **Dead‑letter**: failed jobs go to a DLQ; show banner in Oversight.

## 10) Chaos & Acceptance Tests

**Chaos**

* Drop partner API for 10m → verify retries + no duplicates (idempotency keys).
* Reorder webhooks (payment before invoice) → eventual consistency maintained.
* DST overnight shift (1:30–02:30 fallback) → correct hours.

**Acceptance**

* 3 parallel pay cycles with **0 monetary variance** post‑approval.
* Reports match partner totals; CSV export parity.
* P95 latency: shift→invoice < 10m.

## 11) Oversight Rules (1% layer)

* Flags: overlaps, >12h, missing mapping, rate mismatch, negative totals, invoice↔payroll variance > $0.01.
* Confidence scoring: auto‑approve clean; queue flagged.
* Approve triggers: QBO **send**; Gusto/Paychex **submit**.
* Audit: record actor, rule version, timestamps.

## 12) Report Queries (examples)

**Billable hours (by client, by week)**

```sql
SELECT c.name AS client,
       DATE_TRUNC('week', s.start_at) AS week,
       SUM(s.hours) AS hours,
       SUM(s.hours * s.rate) AS amount
FROM shifts s
JOIN clients c ON c.id = s.client_id
WHERE s.org_id = $1 AND s.approved_bool = TRUE
  AND s.start_at >= $2 AND s.end_at <= $3
GROUP BY client, week
ORDER BY week DESC, client;
```

**Payroll summary (by pay period)**

```sql
SELECT pr.period_start, pr.period_end,
       SUM(pe.hours) AS hours, SUM(pe.gross) AS gross
FROM payroll_runs pr
JOIN payroll_earnings pe ON pe.payroll_run_id = pr.id
WHERE pr.org_id = $1
  AND pr.period_start >= $2 AND pr.period_end <= $3
GROUP BY pr.period_start, pr.period_end
ORDER BY pr.period_start DESC;
```

**Invoice aging**

```sql
SELECT i.id, c.name AS client, i.issue_date, i.due_date, i.total,
       (i.total - COALESCE(i.paid_total,0)) AS balance,
       CASE WHEN i.status = 'paid' THEN '0'
            WHEN CURRENT_DATE - i.due_date <= 30 THEN '0-30'
            WHEN CURRENT_DATE - i.due_date <= 60 THEN '31-60'
            WHEN CURRENT_DATE - i.due_date <= 90 THEN '61-90'
            ELSE '90+' END AS aging_bucket
FROM invoices i
JOIN clients c ON c.id = i.client_id
WHERE i.org_id = $1 AND i.issue_date BETWEEN $2 AND $3;
```

## 15) Graduation to Autonomous Mode (Org-Level)

**Goal:** For the first *N* automations per org (configurable), run with **guided approvals (1% or higher)** and/or **user-guided steps**. When the org demonstrates a stable success score above a threshold, **graduate to Autonomous Mode** (no prompts). Org can opt to keep approvals on.

### Config (feature flags / per‑org settings)

* `FF_AUTONOMY_GRADUATION_ENABLED = true`
* `org_settings.autonomy.minimum_trials = 50`
* `org_settings.autonomy.min_success_score = 0.95`
* `org_settings.autonomy.min_approval_rate = 0.98`
* `org_settings.autonomy.sample_rate = 0.01`
* `org_settings.autonomy.require_approvals_until_graduated = true`
* `org_settings.notifications.email_recipients = ["ops@org.com"]`

### Data Model Additions

```sql
ALTER TABLE organizations ADD COLUMN autonomy_state text DEFAULT 'shadow'; -- shadow|copilot|autonomous|paused
ALTER TABLE organizations ADD COLUMN autonomy_trials int DEFAULT 0;
ALTER TABLE organizations ADD COLUMN autonomy_successes int DEFAULT 0;
ALTER TABLE organizations ADD COLUMN autonomy_score numeric(5,4) DEFAULT 0.0000;
ALTER TABLE organizations ADD COLUMN autonomy_last_review timestamptz;

CREATE TABLE autonomy_events (
  id uuid PRIMARY KEY,
  org_id uuid NOT NULL REFERENCES organizations(id),
  action_type text NOT NULL,                -- invoice|payroll|schedule
  result text NOT NULL,                     -- success|flagged|failed|rolled_back
  confidence numeric(5,4) DEFAULT 1.0000,   -- 0..1 (validator output)
  created_at timestamptz DEFAULT now()
);
CREATE INDEX autonomy_events_org_time_idx ON autonomy_events(org_id, created_at DESC);
```

### Graduation Algorithm (nightly job)

1. Compute windowed stats (last 50–200 automations):

   * `success_rate = successes / trials`
   * `approval_rate = approvals_without_changes / approvals_total`
   * `error_rate`, `rollback_rate`, `flag_rate`
2. If `trials ≥ minimum_trials` **AND** `success_rate ≥ min_success_score` **AND** `approval_rate ≥ min_approval_rate` **AND** circuit healthy:

   * Set `organizations.autonomy_state = 'autonomous'`
   * Set `org_settings.autonomy.require_approvals_until_graduated = false`
   * Keep random sampling at `sample_rate` for post-checks.
3. If error/rollback spike occurs → demote to `copilot` and notify.

### Oversight Behavior by State

* **shadow**: simulate only; create drafts; no external mutations.
* **copilot**: create drafts + require approval (1% minimum, plus all flagged).
* **autonomous**: auto-execute; no approval prompts; continue 1% random sampling.
* **paused**: no execution; health or billing issue.

### Endpoints

```
GET  /orgs/:id/autonomy
PATCH /orgs/:id/autonomy { state?:'shadow'|'copilot'|'autonomous'|'paused', sample_rate?:number, require_approvals?:boolean }
GET  /orgs/:id/autonomy/score  -- returns trials, successes, score, last_review
```

### Notifications

* **On Graduation:**

  * Email: "Your org has graduated to Autonomous Mode — no more prompts. Sampling remains at 1%."
  * Workspace banner for 24h with dismiss.
* **On Demotion:**

  * Email + banner: reason (error spike, circuit breaker), next steps.
* **Run Updates (optional):**

  * "Invoice batch for ACME created → sent"; "Payroll draft submitted".

**Email template (subject/body)**

> Subject: ✅ Autoforce: Your org is now in Autonomous Mode
> Body: Based on your last {{trials}} automations with a {{success_rate}} success rate and {{approval_rate}} approval rate, we’ve enabled Autonomous Mode. You can opt back into approvals in Settings → Autonomy.

### UI/UX

* **Settings → Autonomy** panel:

  * Status badge: Shadow / Co‑Pilot / Autonomous / Paused
  * Toggle: "Require approvals before submission" (default off after graduation)
  * Slider/input: Sampling rate (0–10%)
  * Read‑only metrics: trials, success rate, approval rate, last demotion reason

### Acceptance Tests

* Create 60 simulated clean runs → verify state flips to `autonomous` with emails sent.
* Introduce 3 failures in 20 runs → verify demotion back to `copilot` and banner appears.
* Ensure user toggle can force approvals even in `autonomous`.

## 16) Notifications & Notification Center (Email + Push + In‑App)

**Objective:** Deliver real‑time visibility without forcing email. Support **in‑app notifications, web push, email**, optional **Slack/SMS**, and a user **Notification Center** with preferences. All tied to automation events (runs, flags, approvals, graduation/demotion, billing, webhooks).

### Channels (now vs later)

* **In‑App Notification Center (NOW)**: bell icon, unread badge, inbox list, toasts.
* **Web Push (NOW)**: VAPID-based Web Push for desktop/mobile browsers (permissions opt‑in). Store subscriptions per device.
* **Email (NOW)**: for critical + daily digests.
* **Slack / Teams (LATER)**: per‑org webhook URL.
* **SMS (LATER)**: Twilio stub; only for CRITICAL alerts if opted in.

### Event Categories & Severity

* **Categories:** `automation_update`, `approval_required`, `flagged_item`, `graduation`, `demotion`, `billing_status`, `integration_status`, `report_ready`.
* **Severity:** `info`, `warning`, `critical` (critical also emails by default).

### UX Spec

* **Bell icon** with unread count; dropdown shows last 10 notifications; link to **Notification Center** page.
* **Notification Center page**: filters (category, severity, unread), search, bulk mark‑as‑read, pagination.
* **Toasts**: real‑time for key events (e.g., “Payroll draft submitted”).
* **Deep links**: each item links to the relevant entity (invoice, payroll run, oversight queue).
* **Workspace banners**: for state changes (Paused, Reconnect, Graduated/Demoted) until dismissed.

### Preferences & Quiet Hours

* Per‑user, per‑category channel prefs: in_app, push, email (checkbox matrix).
* **Quiet Hours** (local timezone): suppress push/email except critical.
* **Digest** options: daily/weekly summaries for non‑critical categories.

### Delivery & Escalation Logic

* **CRITICAL** notifications spawn an **escalation_incident** if a policy is attached to the org.
* Step through **escalation_targets** in order: send SMS/Voice/Push/Slack; wait `step_timeout_seconds`; if no **ack**, go to next target.
* **Acknowledgement**:

  * Web: signed link hits `/api/escalations/:id/ack` and records `acked_by_user_id`.
  * Voice: DTMF menu ("Press 1 to acknowledge, 2 to call back ops").
  * SMS: reply `ACK <incidentId>` (simple parser).
* When acked → set incident `status='acked'`, stop further steps; create follow‑up **in‑app** notification.
* After root cause fixed → mark `resolved` and notify involved targets.

**Client Registration (e.g., src/push.js)**

```js
export async function registerPush(vapidPublicKey) {
  if (!('serviceWorker' in navigator) || !('PushManager' in window)) return { ok:false, reason:'unsupported' };
  const reg = await navigator.serviceWorker.register('/sw.js');
  const permission = await Notification.requestPermission();
  if (permission !== 'granted') return { ok:false, reason:'denied' };

  const sub = await reg.pushManager.subscribe({
    userVisibleOnly: true,
    applicationServerKey: urlBase64ToUint8Array(vapidPublicKey)
  });

  const resp = await fetch('/api/notifications/push/subscribe', {
    method: 'POST', headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(sub)
  });
  return { ok: resp.ok };
}

function urlBase64ToUint8Array(base64String) {
  const padding = '='.repeat((4 - base64String.length % 4) % 4);
  const base64 = (base64String + padding).replace(/-/g, '+').replace(/_/g, '/');
  const rawData = atob(base64);
  const outputArray = new Uint8Array(rawData.length);
  for (let i = 0; i < rawData.length; ++i) outputArray[i] = rawData.charCodeAt(i);
  return outputArray;
}
```
