**STEP 5 — ADD APIERROR TYPED ERROR HANDLING TO EVERY QUERY**

Currently when an API call fails — network error, 500, 401, 403, 404 — the error that reaches the `useAsyncData` hook's `isError` state is an untyped `unknown`. The Phase 3 `ErrorBoundary` catches render failures but does not distinguish between a Zod parse failure on a bad response shape and a 401 that requires a redirect to login.

Create `client/src/lib/apiError.ts`:

```typescript
export class ApiError extends Error {
  constructor(
    public status: number,
    public code: string,
    message: string
  ) {
    super(message);
    this.name = 'ApiError';
  }
}

export class ApiContractViolation extends Error {
  constructor(
    public endpoint: string,
    public validationError: string
  ) {
    super(`API response contract violation on ${endpoint}: ${validationError}`);
    this.name = 'ApiContractViolation';
  }
}

export async function apiFetch<T>(
  url: string,
  schema: { parse: (data: unknown) => T },
  options?: RequestInit
): Promise<T> {
  const response = await fetch(url, options);
  
  if (response.status === 401) {
    throw new ApiError(401, 'UNAUTHORIZED', 'Session expired');
  }
  
  if (response.status === 403) {
    throw new ApiError(403, 'FORBIDDEN', 'Access denied');
  }
  
  if (!response.ok) {
    const body = await response.json().catch(() => ({}));
    throw new ApiError(
      response.status,
      body.code ?? 'API_ERROR',
      body.error ?? `Request failed with status ${response.status}`
    );
  }
  
  const json = await response.json();
  
  try {
    return schema.parse(json);
  } catch (validationError) {
    throw new ApiContractViolation(
      url,
      validationError instanceof Error 
        ? validationError.message 
        : String(validationError)
    );
  }
}
```

Update every `queryFn` that was updated in Step 3 to use `apiFetch` instead of raw `fetch`. This gives every query typed error handling that distinguishes auth failures from contract violations from server errors.

Update the `useAsyncData` hook from Phase 3 to handle `ApiError` specifically:

```typescript

  window.location.href = '/login';
}
```
