## PART 4: SPECIFIC ISSUES TO LOOK FOR

### Semantic Issues
- [ ] Data meaning changes between systems
- [ ] Same concept named differently in different places
- [ ] Inconsistent date/time formats
- [ ] Inconsistent status codes
- [ ] Orphaned records (data without proper relationships)

### Canonical Issues
- [ ] Multiple sources of truth for same data
- [ ] Data duplicated across tables/systems
- [ ] Updates not propagating to all copies
- [ ] Conflicting data between systems

### Systematic Issues
- [ ] Workflows that require manual steps that should be automated
- [ ] Missing validation that causes bad data
- [ ] Error handling that fails silently
- [ ] Processes that don't complete atomically

### Race Conditions
- [ ] Concurrent edits causing data loss
- [ ] Timing-dependent bugs
- [ ] Sync order issues
- [ ] Lock contention problems

### Communication Failures
- [ ] Systems not receiving updates they need
- [ ] Real-time updates that aren't real-time
- [ ] Events not triggering proper notifications
- [ ] Webhooks/callbacks failing silently
