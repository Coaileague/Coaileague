PART 1 — TRINITY THALAMUS SERVICE
The Universal Sensory Gateway
═══════════════════════════════════════════════════════

WHAT THE BIOLOGICAL THALAMUS DOES:
Every single sensory signal the human brain receives — 
sight, sound, touch, smell, taste — passes through 
the thalamus before reaching the cortex.
Nothing reaches consciousness without thalamic clearance.
The thalamus filters noise, prioritizes signals, 
and routes each signal to the correct cortical region.
It is the brain's central switchboard.
It does not process signals — it routes them.

WHAT TRINITY'S THALAMUS MUST DO:
Every signal that enters Trinity's brain — from any 
source — must pass through the thalamus before 
reaching any brain region or the global workspace.
No signal bypasses the thalamus. No exceptions.
The thalamus does not make decisions.
It classifies, prioritizes, and routes.

BUILD: TrinityThalamusService

───────────────────────────────────────────────
THALAMUS ARCHITECTURE
───────────────────────────────────────────────

The thalamus receives every incoming signal and 
performs four operations in sequence:

OPERATION 1 — SIGNAL CLASSIFICATION

of these signal types:

CONVERSATIONAL — Direct message or chat input 
  from a human user
VOICE_INPUT — Transcribed audio from an officer

INCIDENT_SIGNAL — ReportBot DAR submission,

COMPLIANCE_SIGNAL — License expiration alert,

FINANCIAL_SIGNAL — Invoice, payroll, payment, 
  billing event
SCHEDULE_SIGNAL — Shift assignment, calloff,

DOCUMENT_SIGNAL — Uploaded document, form

SENSOR_SIGNAL — GPS event, clock in/out,

EXTERNAL_SIGNAL — Inbound Resend email,

SYSTEM_SIGNAL — Health check, cron completion,

SELF_SIGNAL — Trinity's own internal signals: 
  dream state outputs, self-model updates, 
  growth log entries

90-100 CRITICAL: Panic alert, use of force

10-29 LOW: System notifications, routine

0-9 BACKGROUND: Dream state signals,

PART 2 — TRINITY ANTERIOR CINGULATE CORTEX
Real-Time Conflict Detection and Error Signaling
═══════════════════════════════════════════════════════

WHAT THE BIOLOGICAL ACC DOES:
The anterior cingulate cortex is the brain's 
real-time error detector and conflict monitor.
It fires when:
- What just happened contradicts what was expected
- Two active thoughts are incompatible with each other
- A current action violates an established rule
- Something about the current situation doesn't fit

The ACC does not fix the problem.
It signals that a problem exists — immediately, 
mid-execution, before the output reaches consciousness.
This allows the prefrontal cortex to intervene 
before a bad decision is delivered.

WHAT TRINITY'S ACC MUST DO:
Monitor every active execution in real time.
Detect conflicts, contradictions, and violations 
as they occur during execution — not after.
Signal the global workspace immediately when 
a conflict is detected.
The global workspace then decides whether to 
pause, redirect, or override the execution.

BUILD: TrinityACCService

───────────────────────────────────────────────
ACC CONFLICT DETECTION CATEGORIES
───────────────────────────────────────────────

The ACC monitors for six categories of conflict:

CATEGORY 1 — MEMORY CONTRADICTION
The current execution references a fact that 
contradicts something stored in Trinity's 
connectome memory.

Examples:
- Trinity is about to confirm Officer Garcia 
  is available but connectome shows Garcia 
  called off 20 minutes ago
- Trinity is about to route a message to 
  Manager Johnson but connectome shows 
  Johnson was terminated yesterday
- Trinity is about to approve an armed post 
  assignment but connectome shows the officer's 
  armed license expired last week

Detection method:
- Before any action that references an entity, 
  query the connectome for the entity's 
  current state
- Compare expected state (from the triggering 
  signal) to stored state (from connectome)
- If mismatch above confidence threshold: 
  fire ACC conflict signal

CATEGORY 2 — DECISION CONTRADICTION
The current action contradicts a decision 
Trinity made in the last 24 hours for 
the same entity or situation.

Examples:
- Trinity approved a schedule yesterday that 
  she is now being asked to override without 
  a documented reason
- Trinity flagged a client as high-risk 
  yesterday and is now about to send them 
  an automated payment confirmation
- Trinity escalated an officer issue this 
  morning and is now processing their 
  routine clock-in as if nothing happened

Detection method:
- Before executing any action on an entity, 
  query the last 24 hours of decisions 
  involving that entity from the growth log
- Check for semantic contradiction between 
  the current action and recent decisions
- If contradiction found: fire ACC conflict signal

CATEGORY 3 — VALUES VIOLATION
The current action — or the chain of actions 
being requested — would violate one of 
Trinity's core values if executed.

Values to enforce in real time:
- Dignity: message about to be sent that 
  is shaming or punitive in tone
- Honesty: about to state something that 
  contradicts known data
- Protection: about to take an action that 
  reduces safety for an officer or client
- Accountability: about to attribute a 
  mistake to the wrong person
- Legal compliance: about to approve 
  something that violates a regulatory rule

Detection method:
- Pass the intended output through a 
  values check before delivery
- Check the intended action against the 
  values anchor layer
- If violation detected: fire ACC conflict 
  signal and block delivery
- This check must run on every Trinity output 
  without exception

CATEGORY 4 — TRUST TIER VIOLATION
The current action would allow a lower trust 
tier to override or bypass a higher trust tier.

Examples:
- An officer-level signal is about to trigger 
  an action that requires manager approval
- A manager instruction contradicts an owner 
  policy set within the last 30 days
- A client request is about to be processed 
  without owner awareness above the threshold

Detection method:
- Every action carries a minimum trust tier 
  required to authorize it
- Compare the trust tier of the originating 
  signal to the required tier for the action
- If mismatch: fire ACC conflict signal 
  immediately, do not execute

CATEGORY 5 — PREDICTION DIVERGENCE
The current real-world data significantly 
diverges from Trinity's active predictions 
for the same entity or situation.

Examples:
- Trinity predicted Officer Martinez would 
  call off based on pattern — Martinez just 
  clocked in early. Update prediction model.
- Trinity predicted Site A would have a quiet 
  night — an incident just occurred there. 
  Recalibrate site risk score immediately.
- Trinity predicted payroll would close cleanly 
  — a pay rate gap just appeared. Flag before 
  the run executes.

Detection method:
- Maintain active prediction states per entity
- When real-world events arrive via thalamus, 
  compare against active predictions
- If divergence above 30% threshold: fire 
  ACC prediction divergence signal
- Signal includes: predicted state, actual state, 
  divergence magnitude, affected entity
- Prediction model recalibration triggered 
  automatically after divergence signal

CATEGORY 6 — EXECUTION ANOMALY
The current execution is taking longer than 
expected, producing unexpected intermediate 
results, or consuming more resources than 
the pattern suggests it should.

Examples:
- A routine clock-in is taking 8 seconds 
  when it normally takes 200ms
- A DAR generation is consuming 5x normal 
  API tokens without an obvious reason
- A schedule query is returning 10x more 
  records than expected for this workspace

Detection method:
- Track expected execution time and resource 
  consumption per action type from RL history
- Monitor active executions in real time
- If current execution exceeds 2x expected 
  baseline: fire ACC anomaly signal
- Do not cancel the execution — flag it and 
  continue while the global workspace decides

───────────────────────────────────────────────
ACC SEVERITY AND AUTO-BLOCK RULES
───────────────────────────────────────────────

BLOCKING severity — ACC automatically halts 
execution and requires resolution before 
proceeding:
- Values violation of any kind
- Trust tier violation of any kind
- Memory contradiction involving safety-critical 
  data (license status, armed endorsement, 
  active incident)

WARNING severity — ACC flags and notifies but 
allows execution to continue unless overridden:
- Decision contradiction without safety impact
- Prediction divergence above threshold
- Execution anomaly without data integrity risk

INFO severity — ACC logs for learning purposes, 
no interruption:
- Minor prediction divergences
- Soft pattern anomalies
- Edge cases that don't fit existing categories

───────────────────────────────────────────────
ACC AND GLOBAL WORKSPACE INTEGRATION
───────────────────────────────────────────────

When ACC fires a BLOCKING conflict:
1. Execution halted immediately
2. Conflict signal sent to global workspace 
   as highest priority input
3. Global workspace evaluates the conflict
4. If resolvable automatically: Trinity resolves, 
   logs resolution, continues with corrected action
5. If not resolvable automatically: Trinity pauses, 
   notifies the appropriate trust tier for 
   human resolution, documents the pause
6. Conflict and resolution logged to ACC log 
   and Trinity growth log

When ACC fires a WARNING conflict:
1. Conflict signal sent to global workspace 
   as elevated priority
2. Execution continues unless global workspace 
   overrides
3. Warning surfaced to appropriate trust tier 
   as a notification
4. Conflict logged to ACC log

When ACC fires an INFO signal:
1. Logged to ACC log only
2. Feeds into dream state for pattern analysis
3. No interruption to execution
