🧠 The "Guru" Root Cause Analysis
Radix UI's ScrollArea relies on complex internal Refs to calculate scrollbar sizes. When you use a standard React conditional like {showPanel && <ScrollArea ... />}:

The condition becomes false.

React starts unmounting the component.

Radix attempts to measure the DOM node one last time to clean up its event listeners (safelyDetachRef).

The Crash: Because the node is already being destroyed by React, Radix gets a "Ghost Ref" (partially null), tries to update state to handle it, and triggers a re-render during the unmount phase. React blocks this (Maximum update depth) to prevent a memory leak.

🛠️ The Permanent "Visual Persistence" Fix
To fix this forever, we must stop unmounting the ScrollArea. Instead, we must hide it. This keeps the Ref alive and happy even when the user can't see the panel.
