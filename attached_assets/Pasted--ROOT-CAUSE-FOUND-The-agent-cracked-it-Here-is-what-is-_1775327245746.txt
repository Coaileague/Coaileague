Root cause confirmed. showSplash is true on first visit because the localStorage key is not set. SplashScreen renders fixed inset-0 z-99999 covering all page content. Fix this immediately.

```typescript
// Find the showSplash logic — likely something like:
const [showSplash, setShowSplash] = useState(() => {

```

The fix has two parts:

PART 1 — Do not show splash on public routes. Find where showSplash state is initialized and add a public route check:

```typescript
const [showSplash, setShowSplash] = useState(() => {
  // Never show splash on public marketing routes
  const publicPaths = ['/', '/pricing', '/login', '/register', 
    '/about', '/contact', '/sms-consent', '/privacy', '/terms'];
  const isPublic = publicPaths.some(p => 
    window.location.pathname === p || 
    window.location.pathname.startsWith(p)
  );
  if (isPublic) return false;
  return !localStorage.getItem('splash_shown');
});
```

PART 2 — Make sure splash dismisses correctly and sets localStorage:

```typescript
const dismissSplash = () => {
  localStorage.setItem('splash_shown', 'true');
  setShowSplash(false);
};
```

PART 3 — Conditionally render SplashScreen only when needed:

```tsx
// In App.tsx or wherever SplashScreen renders:
{showSplash && !isPublicRoute && (
  <SplashScreen onDismiss={dismissSplash} />
)}
```
