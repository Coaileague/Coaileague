**Query correctness:**

- Every DELETE includes `AND workspace_id = $workspaceId`
- Every INSERT includes `workspace_id` from JWT not from request body

- Every detail query returns complete record
- Every paginated query uses consistent offset/limit pattern
- Every query that feeds an edit modal returns the full record shape the form expects

**Transaction atomicity:**
- Every operation with multiple DB writes uses `db.transaction()`
- Every multi-step operation rolls back completely on failure
- No partial state — either all writes succeed or none do

**Race conditions:**
- Every status transition uses `UPDATE ... WHERE status = 'expected_status'` then checks rowCount
- Every credit deduction uses atomic `UPDATE WHERE balance >= amount`
- Every unique record creation has a unique constraint at DB level
- Every concurrent-safe operation uses SELECT FOR UPDATE where needed

**Arithmetic safety:**
- Every `parseFloat()` has `|| 0` fallback
- Every `parseInt()` has `|| 0` fallback
- Every division has a zero divisor guard
- Every `Number(x)` where x could be undefined has `|| 0` fallback
- Every financial reduce has a numeric initial value
- Every `.toFixed()` is called on a value that cannot be NaN

Fix everything found.
