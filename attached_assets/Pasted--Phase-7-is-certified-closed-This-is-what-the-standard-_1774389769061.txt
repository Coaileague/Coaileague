**READ THIS ENTIRE SECTION BEFORE TOUCHING A SINGLE FILE**

Phases 1 through 7 hardened the data layer — columns verified, calculations precise, inputs validated, writes atomic, concurrency protected. Phase 8 now hardens the communication layer. Every notification the platform sends is a commitment to the tenant. A shift assignment notification that fires and disappears silently when the email provider has a 3-second outage is not a working notification system — it is a notification system that appears to work until it matters.

In a security staffing context a missed notification is not a UX problem. A calloff alert that never delivers means a post goes unstaffed. A payroll approval request that silently fails means payroll is delayed. Phase 8 ensures every notification that is sent has a database record, every failure triggers a retry, and every permanent failure surfaces to the org owner.

**SHORTCUT PATTERNS THAT WILL BE REJECTED**

**REJECTED — Fire-and-forget notification sends.** Any `sendEmail()`, `sendSMS()`, `pushNotification()`, or WebSocket push that executes without confirming delivery and without a retry on failure is fire-and-forget. Every notification goes through the `NotificationService` established in this phase. There are no exceptions.

**REJECTED — Logging failures without retrying.** A `catch` block that logs a notification failure and continues is not retry logic. The failure is logged and the notification is gone. Retry requires re-queuing the notification for another delivery attempt after a backoff delay.

**REJECTED — Retry without idempotency.** A notification retried three times without an idempotency guard produces three delivered notifications. Every retry uses the same notification ID so the recipient receives exactly one delivery regardless of how many attempts were made.

**REJECTED — WebSocket-only for critical notifications.** WebSocket delivery is best-effort. A manager who navigates away or has a dropped connection misses the notification entirely. Critical operational notifications — shift coverage needed, payroll approval required, calloff received — must have an email fallback when WebSocket delivery is not acknowledged within 30 seconds.

**REJECTED — Notification delivery status that only tracks sends.** "Sent" is not "delivered." The notification log must track: pending, sent, failed, retrying, permanently_failed, and delivered. Without delivered status there is no way to know whether a critical operational alert actually reached a human.

**STEP 2 — CREATE THE NOTIFICATIONSERVICE**

Create `server/services/notificationService.ts`. This is the only permitted way to send any notification in the platform. Every existing send call in the codebase will be replaced with a call to this service by the end of this phase.

```typescript
import { db } from '../db';
import { notifications } from '@shared/schema';
import { eq, and, lte, lt } from 'drizzle-orm';
import { randomUUID } from 'crypto';

export type NotificationType = 
  | 'shift_assignment'
  | 'shift_cancellation' 
  | 'calloff_received'
  | 'coverage_needed'
  | 'payroll_approval_required'
  | 'payroll_approved'
  | 'payroll_paid'
  | 'invoice_sent'
  | 'invoice_overdue'
  | 'invoice_paid'
  | 'document_requires_signature'
  | 'onboarding_invite'
  | 'trinity_alert'
  | 'system_alert';

export type NotificationChannel = 'email' | 'sms' | 'websocket' | 'in_app';

export interface NotificationPayload {
  type: NotificationType;
  workspaceId: string;
  recipientUserId: string;
  channel: NotificationChannel;
  subject?: string;
  body: Record<string, unknown>;
  idempotencyKey?: string;
  scheduledAt?: Date;
}

export class NotificationService {
  
  // Send a notification — writes to DB then attempts delivery
  static async send(payload: NotificationPayload): Promise<string> {
    const idempotencyKey = payload.idempotencyKey ?? 
      `${payload.type}-${payload.recipientUserId}-${Date.now()}`;
    
    // Check idempotency — if already sent return existing record
    const existing = await db
      .select()
      .from(notifications)
      .where(eq(notifications.idempotencyKey, idempotencyKey))
      .limit(1);
      
    if (existing.length > 0 && existing[0].status === 'sent') {
      return existing[0].id;
    }
    
    // Write notification record first — before any send attempt
    const [record] = await db
      .insert(notifications)
      .values({
        workspaceId: payload.workspaceId,
        recipientUserId: payload.recipientUserId,
        notificationType: payload.type,
        channel: payload.channel,
        subject: payload.subject,
        payload: payload.body,
        idempotencyKey,
        status: 'pending',
        scheduledAt: payload.scheduledAt ?? new Date(),
      })
      .onConflictDoUpdate({
        target: notifications.idempotencyKey,
        set: { status: 'pending', updatedAt: new Date() }
      })
      .returning();
    
    // Attempt delivery immediately
    await this.attemptDelivery(record.id);
    
    return record.id;
  }
  
  // Attempt delivery for a notification record
  static async attemptDelivery(notificationId: string): Promise<void> {
    const [record] = await db
      .select()
      .from(notifications)
      .where(eq(notifications.id, notificationId))
      .limit(1);
      
    if (!record || record.status === 'sent' || 
        record.status === 'permanently_failed') return;
    
    // Mark as sending
    await db
      .update(notifications)
      .set({ status: 'sending', updatedAt: new Date() })
      .where(eq(notifications.id, notificationId));
    
    try {
      await this.deliverByChannel(record);
      
      // Mark as sent on success
      await db
        .update(notifications)
        .set({ 
          status: 'sent', 
          sentAt: new Date(),
          attemptCount: record.attemptCount + 1,
          updatedAt: new Date()
        })
        .where(eq(notifications.id, notificationId));
        
    } catch (error) {
      const newAttemptCount = record.attemptCount + 1;
      const isPermanentlyFailed = newAttemptCount >= record.maxAttempts;
      
      if (isPermanentlyFailed) {
        await db
          .update(notifications)
          .set({
            status: 'permanently_failed',
            lastError: error instanceof Error ? error.message : String(error),
            attemptCount: newAttemptCount,
            updatedAt: new Date()
          })
          .where(eq(notifications.id, notificationId));
          
        // Alert org owner of permanent failure on critical notification types
        await this.alertPermanentFailure(record);
      } else {
        // Exponential backoff: 2min, 8min, 32min
        const backoffMinutes = Math.pow(4, newAttemptCount) / 2;
        const nextRetry = new Date(Date.now() + backoffMinutes * 60 * 1000);
        
        await db
          .update(notifications)
          .set({
            status: 'retrying',
            lastError: error instanceof Error ? error.message : String(error),
            attemptCount: newAttemptCount,
            nextRetryAt: nextRetry,
            updatedAt: new Date()
          })
          .where(eq(notifications.id, notificationId));
      }
    }
  }
  
  // Deliver by channel — implement each channel
  private static async deliverByChannel(
    record: typeof notifications.$inferSelect
  ): Promise<void> {
    switch (record.channel) {
      case 'email':
        await this.sendEmail(record);
        break;
      case 'websocket':
        await this.sendWebSocket(record);
        break;
      case 'in_app':
        await this.createInAppNotification(record);
        break;
      case 'sms':
        await this.sendSMS(record);
        break;
    }
  }
  
  private static async sendEmail(
    record: typeof notifications.$inferSelect
  ): Promise<void> {
    // Import and use existing Resend/email service
    // Wrap in 5-second timeout — external service must not block indefinitely
    const timeout = new Promise((_, reject) => 
      setTimeout(() => reject(new Error('Email delivery timeout')), 5000)
    );
    await Promise.race([
      this.deliverEmail(record),
      timeout
    ]);
  }
  
  private static async sendWebSocket(
    record: typeof notifications.$inferSelect
  ): Promise<void> {
    // Push via WebSocket — if connection not available throw so retry queues
    // WebSocket delivery acknowledgment timeout handled separately
    const wsManager = await import('./chatConnectionManager');
    // Attempt push — throws if recipient not connected
    await wsManager.default.pushToUser(
      record.recipientUserId, 
      record.payload
    );
  }
  
  private static async createInAppNotification(
    record: typeof notifications.$inferSelect  
  ): Promise<void> {
    // Write to in-app notification table
    // This is a database write so it cannot fail from external dependency
  }
  
  private static async sendSMS(
    record: typeof notifications.$inferSelect
  ): Promise<void> {
    // SMS provider integration with 5-second timeout
  }
  
  private static async alertPermanentFailure(
    record: typeof notifications.$inferSelect
  ): Promise<void> {
    // Send a system alert to the org owner about the permanently failed notification
    // This is a separate notification — do not recurse into NotificationService
    // Log to error monitoring service
    console.error(`[NotificationService] Permanent failure:`, {
      id: record.id,
      type: record.notificationType,
      recipient: record.recipientUserId,
      workspace: record.workspaceId,
      attempts: record.attemptCount,
      lastError: record.lastError
    });
  }
  
  // Retry processor — called by background job every minute
  static async processRetries(): Promise<void> {
    const due = await db
      .select()
      .from(notifications)
      .where(
        and(
          eq(notifications.status, 'retrying'),
          lte(notifications.nextRetryAt, new Date())
        )
      )
      .limit(50);
      
    await Promise.allSettled(
      due.map(n => this.attemptDelivery(n.id))
    );
  }
}
```
