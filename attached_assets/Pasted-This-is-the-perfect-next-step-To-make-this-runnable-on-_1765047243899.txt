1. The Tool & Action Registry
Purpose: This acts as the "API of APIs." It stores the definitions of every capability your AI has (e.g., "Send Email," "Query SQL," "Update CRM").

File: models.py (Add to your existing models)

from sqlalchemy import Column, String, Integer, JSON, Boolean, DateTime
from sqlalchemy.sql import func
from database import Base # Assuming you have a Base class

class ToolDefinition(Base):
    __tablename__ = 'tool_definitions'

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String, unique=True, nullable=False) # e.g., "stripe_refund_payment"
    description = Column(String, nullable=False) # Critical for the LLM to understand purpose
    
    # The JSON schema required to call this tool (OpenAI Function Calling format)
    parameters_schema = Column(JSON, nullable=False) 
    
    # Endpoint or Python path to execute
    execution_handler = Column(String, nullable=False) 
    
    is_active = Column(Boolean, default=True)
    requires_approval = Column(Boolean, default=False) # Safety guardrail
    created_at = Column(DateTime(timezone=True), server_default=func.now())

import json

class ToolRegistryService:
    def __init__(self, db_session):
        self.db = db_session

    def register_tool(self, name, description, schema, handler, requires_approval=False):
        """Adds a new capability to the AI's repertoire."""
        new_tool = ToolDefinition(
            name=name,
            description=description,
            parameters_schema=schema,
            execution_handler=handler,
            requires_approval=requires_approval
        )
        self.db.add(new_tool)
        self.db.commit()
        return new_tool

    def get_tools_for_context(self, context_tags):
        """
        Retrieves relevant tools based on workflow context.
        (Simple implementation returns all active tools)
        """
        return self.db.query(ToolDefinition).filter(ToolDefinition.is_active == True).all()

    def validate_arguments(self, tool_name, arguments):
        """

        """

        if not tool:
            raise ValueError(f"Tool {tool_name} not found.")

3. How they connect to your previous services
This is the flow you can set up in your main application file (main.py):

Ingestion: Client sends a request "Upload my invoices."

AI Engine:

Calls ContextResolver to find where invoices are usually stored.

Calls ToolRegistry to find the "Google Drive Upload" tool.

Generates a plan.

WorkflowLedger: Records the plan as "Pending."

RealTimeBridge: If the tool is external (like Stripe or Slack), this bridge handles the actual API traffic.

Summary of What You Have Now:
✅ WorkflowLedger: Tracks state.

✅ CommitmentManager: Handles promises/transactions.

✅ ContextResolver: Fetches data.

✅ RealTimeBridge: Connects APIs.

✅ ToolRegistry (New): Defines capabilities.
