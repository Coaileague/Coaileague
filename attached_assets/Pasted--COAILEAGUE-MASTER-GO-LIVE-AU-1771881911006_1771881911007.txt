## 1.1 — Multi-Tenant Data Isolation (THE #1 PRODUCTION KILLER)

If Org A can see Org B's data, the platform is dead on arrival. Test EVERY data access path.

**FOR EVERY API ENDPOINT, VERIFY:**

```
[ ] Query includes workspace_id / org_id filter
[ ] User can only access data from their own workspace
[ ] A user in Org A cannot craft a request to access Org B's data
[ ] Even if they guess/know another org's ID
```

**CROSS-TENANT BLEED POINTS — CHECK EACH ONE:**

```
[ ] API routes — every GET/POST/PUT/DELETE scoped to requesting org
[ ] WebSocket rooms — can a user join another org's chat room?
[ ] Real-time updates — do push events leak to wrong org?
[ ] File uploads — files stored with org isolation (not shared bucket)?
[ ] Reports — report generation scoped to requesting org?
[ ] Search — search index scoped to user's org?
[ ] Trinity AI — Trinity's context includes ONLY the requesting org's data?
[ ] Email notifications — emails reference correct org branding/info?
[ ] Admin panels — platform admin view properly isolates org data?
```

**TEST THIS SPECIFICALLY:**
1. Create two test orgs with different data
2. Log in as user from Org A
3. Try to access Org B's employees, clients, shifts, invoices, time entries, chat messages, documents
4. If ANY return Org B's data → **STOP EVERYTHING. Fix before launch.**
