## CRITICAL ISSUES IDENTIFIED

1. **"Rendered more hooks than during the previous render"** - React hooks violation causing page crashes
2. **"Duplicate notifications for New Update Available"** - Event listeners firing multiple times

**THESE ARE UNIVERSAL ISSUES** affecting all pages on desktop and mobile, breaking workflows.

# PART 1: REACT HOOKS RULES VIOLATION HUNT

## 1.1 Understanding the Error

```
"Rendered more hooks than during the previous render"

This error means:
- Hooks are being called CONDITIONALLY
- Hooks are called AFTER early returns
- Hooks are called INSIDE loops
- Hooks are called INSIDE nested functions
- The NUMBER of hooks changes between renders

React requires hooks to be called:
- At the TOP LEVEL of the component
- In the SAME ORDER every render
- NEVER conditionally
```

## 1.2 Find ALL Hooks Violations

```bash
# VIOLATION 3: Hooks inside loops
```

## 1.3 Common Violation Patterns

### Pattern 1: Early Return Before Hooks (MOST COMMON)

```tsx
// ❌ BAD - Hooks after conditional return
function ClientsPage() {
  const { user } = useAuth();
  
  if (!user) {
    return <Redirect to="/login" />; // Early return
  }
  
  // These hooks won't run if user is null!
  const [clients, setClients] = useState([]); // VIOLATION
  const { data } = useQuery(['clients']); // VIOLATION
  
  return <div>{/* ... */}</div>;
}

// ✅ GOOD - All hooks before any returns
function ClientsPage() {
  const { user } = useAuth();
  const [clients, setClients] = useState([]);
  const { data } = useQuery(['clients'], { enabled: !!user });
  
  if (!user) {
    return <Redirect to="/login" />;
  }
  
  return <div>{/* ... */}</div>;
}
```

### Pattern 2: Conditional Hook Calls

```tsx
// ❌ BAD - Hook inside condition
function UserProfile({ userId }) {
  if (userId) {
    const { data } = useQuery(['user', userId]); // VIOLATION
  }
  
  return <div>{/* ... */}</div>;
}

// ✅ GOOD - Hook always called, condition in options
function UserProfile({ userId }) {
  const { data } = useQuery(
    ['user', userId], 
    () => fetchUser(userId),
    { enabled: !!userId } // Conditional execution, not conditional hook
  );
  
  return <div>{/* ... */}</div>;
}
```

### Pattern 3: Hooks in Loops

```tsx
// ❌ BAD - Hooks in map/loop
function ItemList({ items }) {
  return items.map(item => {
    const [expanded, setExpanded] = useState(false); // VIOLATION
    return <Item key={item.id} expanded={expanded} />;
  });
}

// ✅ GOOD - Extract to separate component
function ItemList({ items }) {
  return items.map(item => (
    <ItemWithState key={item.id} item={item} />
  ));
}

function ItemWithState({ item }) {
  const [expanded, setExpanded] = useState(false); // OK - top level of component
  return <Item expanded={expanded} />;
}
```

### Pattern 4: Hooks in Event Handlers

```tsx
// ❌ BAD - Hook in callback
function Form() {
  const handleSubmit = () => {
    const [loading, setLoading] = useState(false); // VIOLATION
    setLoading(true);
  };
  
  return <form onSubmit={handleSubmit} />;
}

// ✅ GOOD - Hook at top level
function Form() {
  const [loading, setLoading] = useState(false);
  
  const handleSubmit = () => {
    setLoading(true);
  };
  
  return <form onSubmit={handleSubmit} />;
}
```

### Pattern 5: Dynamic Hook Count

```tsx
// ❌ BAD - Variable number of hooks
function DynamicFields({ fields }) {
  const values = fields.map(field => {
    return useState(field.defaultValue); // VIOLATION - different count each render
  });
}

// ✅ GOOD - Single state object
function DynamicFields({ fields }) {
  const [values, setValues] = useState(
    fields.reduce((acc, f) => ({ ...acc, [f.name]: f.defaultValue }), {})
  );
}
```

# PART 2: DUPLICATE EVENTS & NOTIFICATIONS HUNT

## 2.1 Understanding Duplicate Events

```
Duplicate "New Update Available" notifications indicate:

1. useEffect running multiple times without cleanup
2. Event listeners added multiple times
3. WebSocket handlers registered multiple times
4. Service worker events firing multiple times
5. Context providers mounted multiple times
6. Subscriptions not cleaned up on unmount
```

```bash
  done
```

## 2.3 Common Duplication Patterns

### Pattern 1: useEffect Without Cleanup

```tsx
// ❌ BAD - No cleanup, listener added every render
useEffect(() => {
  window.addEventListener('resize', handleResize);
  // Missing cleanup!
}, []);

// ✅ GOOD - Cleanup on unmount
useEffect(() => {
  window.addEventListener('resize', handleResize);
  return () => {
    window.removeEventListener('resize', handleResize);
  };
}, []);
```

### Pattern 2: WebSocket Without Cleanup

```tsx
// ❌ BAD - Socket handlers accumulate
useEffect(() => {
  socket.on('notification', handleNotification);
  // No cleanup - handlers pile up!
}, []);

// ✅ GOOD - Remove handler on cleanup
useEffect(() => {
  socket.on('notification', handleNotification);
  return () => {
    socket.off('notification', handleNotification);
  };
}, []);
```

### Pattern 3: Service Worker Multiple Registrations

```tsx
// ❌ BAD - Registers every time component mounts
useEffect(() => {
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.register('/sw.js');
    navigator.serviceWorker.addEventListener('message', handleMessage);
    // Registers multiple times on remount!
  }
}, []);

// ✅ GOOD - Check if already registered, cleanup
useEffect(() => {
  let registration;
  
  const setup = async () => {
    if ('serviceWorker' in navigator) {
      registration = await navigator.serviceWorker.register('/sw.js');
      navigator.serviceWorker.addEventListener('message', handleMessage);
    }
  };
  
  setup();
  
  return () => {
    navigator.serviceWorker.removeEventListener('message', handleMessage);
  };
}, []);
```

### Pattern 4: Interval/Timeout Without Cleanup

```tsx
// ❌ BAD - Intervals pile up
useEffect(() => {
  setInterval(() => {
    checkForUpdates();
  }, 30000);
  // No cleanup - multiple intervals running!
}, []);

// ✅ GOOD - Clear on cleanup
useEffect(() => {
  const interval = setInterval(() => {
    checkForUpdates();
  }, 30000);
  
  return () => clearInterval(interval);
}, []);
```

### Pattern 5: Subscription Without Unsubscribe

```tsx
// ❌ BAD - Subscriptions accumulate
useEffect(() => {
  const unsubscribe = store.subscribe(handleChange);
  // Never unsubscribes!
}, []);

// ✅ GOOD - Unsubscribe on cleanup
useEffect(() => {
  const unsubscribe = store.subscribe(handleChange);
  return () => unsubscribe();
}, []);
```

### Pattern 6: Context Provider Remounting

```tsx
// ❌ BAD - Provider inside component that remounts
function App() {
  const [key, setKey] = useState(0);
  
  return (
    <div key={key}> {/* Forces remount */}
      <NotificationProvider> {/* Recreated on key change */}
        <Content />
      </NotificationProvider>
    </div>
  );
}

// ✅ GOOD - Providers at stable level
function App() {
  return (
    <NotificationProvider> {/* Stable, doesn't remount */}
      <Content />
    </NotificationProvider>
  );
}
```

## 4.3 Standard Component Template

```tsx
/**
 * CORRECT COMPONENT STRUCTURE
 * 
 * 1. All hooks at the top (useState, useEffect, useQuery, etc.)
 * 2. Derived state / computations
 * 3. Event handlers
 * 4. Early returns for loading/error states
 * 5. Main render
 */

function StandardComponent({ id }) {
  // ========================================
  // 1. ALL HOOKS FIRST - ALWAYS RUN
  // ========================================
  
  // Auth/Context hooks
  const { user } = useAuth();
  const { workspace } = useWorkspace();
  
  // State hooks
  const [isOpen, setIsOpen] = useState(false);
  const [searchQuery, setSearchQuery] = useState('');
  
  // Query hooks (with enabled flag for conditional fetching)
  const { data, isLoading, error } = useQuery(
    ['resource', id],
    () => fetchResource(id),
    { enabled: !!id && !!user } // Conditional execution, NOT conditional hook
  );
  
  // Mutation hooks
  const { mutate, isLoading: isMutating } = useMutation(updateResource);
  
  // Effect hooks
  useEffect(() => {
    // Effect logic
    return () => {
      // Cleanup - ALWAYS include for subscriptions/listeners
    };
  }, [dependencies]);
  
  // Ref hooks
  const inputRef = useRef(null);
  
  // Memo hooks
  const filteredData = useMemo(() => {
    return data?.filter(item => item.name.includes(searchQuery));
  }, [data, searchQuery]);
  
  // Callback hooks
  const handleSubmit = useCallback(() => {
    mutate({ id, data: formData });
  }, [id, formData, mutate]);
  
  // ========================================
  // 2. DERIVED STATE (no hooks)
  // ========================================
  
  const hasData = data && data.length > 0;
  const canEdit = user?.role === 'admin';
  
  // ========================================
  // 3. EVENT HANDLERS (no hooks)
  // ========================================
  
  const handleClick = () => {
    setIsOpen(true);
  };
  
  // ========================================
  // 4. EARLY RETURNS (AFTER all hooks)
  // ========================================
  
  if (isLoading) {
    return <LoadingSpinner />;
  }
  
  if (error) {
    return <ErrorMessage error={error} />;
  }
  
  if (!user) {
    return <Navigate to="/login" />;
  }
  
  // ========================================
  // 5. MAIN RENDER
  // ========================================
  
  return (
    <div>
      {/* Component JSX */}
    </div>
  );
}
```

## 4.4 useEffect Cleanup Template

```tsx
/**
 * ALWAYS include cleanup for:
 * - Event listeners
 * - Subscriptions
 * - Timers (setTimeout, setInterval)
 * - WebSocket handlers
 * - Abort controllers for fetch
 */

// Event listener cleanup
useEffect(() => {
  const handleResize = () => setWidth(window.innerWidth);
  window.addEventListener('resize', handleResize);
  return () => window.removeEventListener('resize', handleResize);
}, []);

// WebSocket cleanup
useEffect(() => {
  const handleMessage = (data) => setMessages(prev => [...prev, data]);
  socket.on('message', handleMessage);
  return () => socket.off('message', handleMessage);
}, []);

// Timer cleanup
useEffect(() => {
  const timer = setTimeout(() => setVisible(false), 3000);
  return () => clearTimeout(timer);
}, []);

// Interval cleanup
useEffect(() => {
  const interval = setInterval(() => fetchUpdates(), 30000);
  return () => clearInterval(interval);
}, []);

// Fetch abort cleanup
useEffect(() => {
  const controller = new AbortController();
  
  fetch('/api/data', { signal: controller.signal })
    .then(res => res.json())
    .then(data => setData(data))
    .catch(err => {
      if (err.name !== 'AbortError') {
        setError(err);
      }
    });
  
  return () => controller.abort();
}, []);

// Subscription cleanup
useEffect(() => {
  const unsubscribe = store.subscribe(state => setState(state));
  return () => unsubscribe();
}, []);
```
