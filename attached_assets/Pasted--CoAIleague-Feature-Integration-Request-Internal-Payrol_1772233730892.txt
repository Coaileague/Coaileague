### 1.1 What This Feature Does

**Payroll Module:**
- Calculate gross pay from time records (hours × rate)
- Calculate overtime (hours > 40 × 1.5 rate)
- Apply configured deductions
- Generate pay stubs (PDF)
- Create payroll register reports
- Export data for external tax filing service
- Track payroll history

**Invoicing Module:**
- Generate invoices from billable hours + contract rates
- Add custom line items
- Calculate taxes
- Send invoices via email
- Display in client portal
- Track payment status
- Generate AR/aging reports
- Optional: Stripe integration for payment collection

### 1.2 Why We Need It

| Current State | Problem | Solution |
|---------------|---------|----------|
| QB Payroll | $125/mo + $6/employee = $245/mo | Internal calculation = $0 |
| QB Invoicing | Per-invoice fees + % of payments | Internal generation = $0 |
| QB Lock-in | Dependent on external system | Full control |
| Data Duplication | Hours in CoAIleague AND QB | Single source of truth |

**Annual Savings: ~$2,400-3,000**

### 1.3 What We Still Need Externally

| Function | Why External | Cheapest Option |
|----------|--------------|-----------------|
| Tax Filing (941, W2, state) | IRS/state compliance complex | Patriot ($17/mo) or Check API |
| Direct Deposit | ACH network access | Gusto Embedded or Stripe |
| Payment Processing | PCI compliance | Stripe (2.9%) |

## 2. DATA MODEL REQUIREMENTS

### 2.1 New Tables Required

| Table Name | Purpose | Primary Key |
|------------|---------|-------------|
| payroll_runs | Each payroll execution | payroll_run_id |
| payroll_items | Line items per employee per run | payroll_item_id |
| pay_stubs | Generated pay stub records | pay_stub_id |
| deduction_configs | Configured deductions per org | deduction_id |
| invoices | Invoice headers | invoice_id |
| invoice_line_items | Invoice details | line_item_id |
| payments | Payment records | payment_id |
| billing_rates | Rate overrides beyond contract | rate_id |

### 2.2 Table Schemas

```sql
-- PAYROLL_RUNS
CREATE TABLE payroll_runs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id),
  workspace_id UUID NOT NULL,
  
  pay_period_start DATE NOT NULL,
  pay_period_end DATE NOT NULL,
  pay_date DATE NOT NULL,
  
  status VARCHAR(20) NOT NULL DEFAULT 'draft', -- draft, processing, completed, exported
  
  total_gross DECIMAL(12,2) NOT NULL DEFAULT 0,
  total_deductions DECIMAL(12,2) NOT NULL DEFAULT 0,
  total_net DECIMAL(12,2) NOT NULL DEFAULT 0,
  total_employer_cost DECIMAL(12,2) NOT NULL DEFAULT 0,
  
  exported_at TIMESTAMP,
  export_file_url TEXT,
  
  created_at TIMESTAMP DEFAULT NOW(),
  created_by UUID NOT NULL REFERENCES users(id),
  updated_at TIMESTAMP DEFAULT NOW(),
  updated_by UUID NOT NULL REFERENCES users(id),
  
  UNIQUE(org_id, pay_period_start, pay_period_end)
);

-- PAYROLL_ITEMS
CREATE TABLE payroll_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id),
  workspace_id UUID NOT NULL,
  payroll_run_id UUID NOT NULL REFERENCES payroll_runs(id) ON DELETE CASCADE,
  employee_id UUID NOT NULL REFERENCES employees(id),
  
  regular_hours DECIMAL(6,2) NOT NULL DEFAULT 0,
  overtime_hours DECIMAL(6,2) NOT NULL DEFAULT 0,
  
  regular_rate DECIMAL(8,2) NOT NULL,
  overtime_rate DECIMAL(8,2) NOT NULL,
  
  gross_pay DECIMAL(10,2) NOT NULL,
  
  -- Deductions (expandable)
  deduction_federal DECIMAL(8,2) DEFAULT 0,
  deduction_state DECIMAL(8,2) DEFAULT 0,
  deduction_fica DECIMAL(8,2) DEFAULT 0,
  deduction_medicare DECIMAL(8,2) DEFAULT 0,
  deduction_other DECIMAL(8,2) DEFAULT 0,
  deduction_notes TEXT,
  
  total_deductions DECIMAL(10,2) NOT NULL,
  net_pay DECIMAL(10,2) NOT NULL,
  
  -- Employer costs
  employer_fica DECIMAL(8,2) DEFAULT 0,
  employer_medicare DECIMAL(8,2) DEFAULT 0,
  employer_futa DECIMAL(8,2) DEFAULT 0,
  employer_suta DECIMAL(8,2) DEFAULT 0,
  employer_other DECIMAL(8,2) DEFAULT 0,
  total_employer_cost DECIMAL(10,2) NOT NULL,
  
  pay_stub_id UUID REFERENCES pay_stubs(id),
  
  created_at TIMESTAMP DEFAULT NOW(),
  created_by UUID NOT NULL,
  updated_at TIMESTAMP DEFAULT NOW(),
  updated_by UUID NOT NULL
);

-- INVOICES
CREATE TABLE invoices (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id),
  workspace_id UUID NOT NULL,
  client_id UUID NOT NULL REFERENCES clients(id),
  contract_id UUID REFERENCES contracts(id),
  
  invoice_number VARCHAR(50) NOT NULL,
  
  billing_period_start DATE,
  billing_period_end DATE,
  invoice_date DATE NOT NULL,
  due_date DATE NOT NULL,
  
  subtotal DECIMAL(12,2) NOT NULL DEFAULT 0,
  tax_rate DECIMAL(5,4) DEFAULT 0,
  tax_amount DECIMAL(10,2) DEFAULT 0,
  total DECIMAL(12,2) NOT NULL DEFAULT 0,
  
  status VARCHAR(20) NOT NULL DEFAULT 'draft', -- draft, sent, viewed, partial, paid, overdue, void
  
  sent_at TIMESTAMP,
  sent_to_email TEXT,
  viewed_at TIMESTAMP,
  
  payment_link TEXT, -- Stripe link if enabled
  
  notes TEXT,
  terms TEXT,
  
  pdf_url TEXT,
  
  created_at TIMESTAMP DEFAULT NOW(),
  created_by UUID NOT NULL REFERENCES users(id),
  updated_at TIMESTAMP DEFAULT NOW(),
  updated_by UUID NOT NULL REFERENCES users(id),
  
  UNIQUE(org_id, invoice_number)
);

-- INVOICE_LINE_ITEMS
CREATE TABLE invoice_line_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id),
  invoice_id UUID NOT NULL REFERENCES invoices(id) ON DELETE CASCADE,
  
  description TEXT NOT NULL,
  quantity DECIMAL(10,2) NOT NULL,
  unit_price DECIMAL(10,2) NOT NULL,
  amount DECIMAL(12,2) NOT NULL,
  
  -- Source tracking
  source_type VARCHAR(20), -- time_record, service_item, manual
  source_id UUID,
  
  sort_order INT DEFAULT 0,
  
  created_at TIMESTAMP DEFAULT NOW()
);

-- PAYMENTS
CREATE TABLE payments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id),
  workspace_id UUID NOT NULL,
  invoice_id UUID NOT NULL REFERENCES invoices(id),
  client_id UUID NOT NULL REFERENCES clients(id),
  
  amount DECIMAL(12,2) NOT NULL,
  payment_date DATE NOT NULL,
  payment_method VARCHAR(30), -- check, ach, credit_card, cash, stripe
  
  reference_number TEXT,
  stripe_payment_id TEXT,
  
  notes TEXT,
  
  created_at TIMESTAMP DEFAULT NOW(),
  created_by UUID NOT NULL REFERENCES users(id)
);
```

## 5. SEVEN-STEP EXECUTION PROCESS

### 5.1 Action: RUN PAYROLL

| Step | What Happens | Validation | Failure Handling |
|------|--------------|------------|------------------|
| 1. TRIGGER | User clicks "Run Payroll" for date range | | |
| 2. VALIDATE | Date range valid? No overlapping runs? User has permission? | Check existing runs, check role | Error: "Payroll already exists for this period" |
| 3. PROCESS | Pull time records, calculate hours, apply rates, calculate deductions | Verify all employees have rates | Flag employees with missing data |
| 4. PERSIST | Save payroll_run, payroll_items, generate pay_stubs | Transaction ensures all-or-nothing | Rollback on failure |
| 5. PROPAGATE | Update dashboards, financial reports, Trinity | Push to real-time | Queue if offline |
| 6. NOTIFY | Alert payroll admin of completion, optionally notify employees | Check notification prefs | Log if notification fails |
| 7. CONFIRM | Show summary: total gross, deductions, net, employee count | | Return success with summary |

### 5.2 Action: GENERATE INVOICE

| Step | What Happens | Validation | Failure Handling |
|------|--------------|------------|------------------|
| 1. TRIGGER | User clicks "Generate Invoice" for client/period | | |
| 2. VALIDATE | Client active? Contract active? Hours exist? User has permission? | Check client status, contract | Error: "No billable hours found" |
| 3. PROCESS | Pull billable hours, apply rates, add line items, calculate totals | Verify rates configured | Flag missing rates |
| 4. PERSIST | Save invoice, line_items, generate PDF | Transaction | Rollback on failure |
| 5. PROPAGATE | Update client record, AR reports, client portal, Trinity | Push updates | Queue if needed |
| 6. NOTIFY | (If auto-send) Email invoice to client | Verify email address | Flag for manual send |
| 7. CONFIRM | Show invoice preview with total | | Return invoice_id |

### 5.3 Action: RECORD PAYMENT

| Step | What Happens | Validation | Failure Handling |
|------|--------------|------------|------------------|
| 1. TRIGGER | User enters payment OR Stripe webhook | | |
| 2. VALIDATE | Invoice exists? Not already fully paid? Amount valid? | Check invoice status | Error: "Invoice already paid" |
| 3. PROCESS | Create payment record, calculate remaining balance | Handle partial payments | |
| 4. PERSIST | Save payment, update invoice status (partial/paid) | Transaction | Rollback on failure |
| 5. PROPAGATE | Update AR reports, client record, client portal, Trinity | Push updates | |
| 6. NOTIFY | Notify billing staff, optionally send receipt to client | | |
| 7. CONFIRM | Show payment confirmation | | |

## 7. RACE CONDITION PREVENTION

| Scenario | Risk | Prevention |
|----------|------|------------|
| Two users run payroll for same period | Duplicate payroll | UNIQUE constraint on org_id + period |
| Invoice generated while hours being edited | Wrong totals | Lock time records during generation OR snapshot |
| Payment recorded twice (double-click) | Over-payment | Idempotency key on payment creation |
| Payroll run while employee being deactivated | Include/exclude confusion | Snapshot employees at run start |
| Invoice sent while being edited | Wrong version sent | Lock invoice during send |
