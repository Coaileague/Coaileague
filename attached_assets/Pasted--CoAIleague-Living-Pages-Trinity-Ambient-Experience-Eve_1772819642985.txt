## FROM BRYAN — THE VISION

I do not want dead pages. I do not want pages that are just cards, buttons, and basic text with nothing happening. Every single page on this platform — mobile and desktop — needs to feel alive, helpful, and intelligent. 

The reference is what Homebase does on their mobile dashboard — the moment you open it you feel something. Your money is there. Your shift is there. It feels like it knows you.

We are going further than that. Trinity is our AI and she needs to be ambient — always present, always helpful, always aware of what the user is looking at. Not buried in a link. Not hidden behind a menu. She follows the user everywhere, knows the context of every page, and proactively helps.

This is the standard I want. Nothing less.

---

## ⛔ STANDING RULES

1. **NO DROP TABLE** — Not one. Ever.
2. **NO SCHEMA CHANGES** — Frontend and AI wiring only.
3. **READ BEFORE WRITE** — Understand the current page before touching it.
4. **ONE PAGE SECTION AT A TIME** — Build, verify mobile + desktop, checkpoint, next.
5. **TRINITY READS, NEVER GUESSES** — All Trinity widgets pull from real data. No hardcoded content.
6. **UNIVERSAL COMPONENTS ONLY** — Every element uses the universal system. No custom one-off implementations.

## PART 1 — THE TRINITY AMBIENT WIDGET (FAB)

This is the most important element on the entire platform. It lives on every single page.

### What It Is
A floating action button that is Trinity's presence on every page. It is context-aware — it knows which page the user is on, what data is visible, and what questions the user might have. It can analyze what's on screen and answer questions about it in plain language.

Think of it exactly like QuickBooks' Intuit Assist widget — but smarter because Trinity knows the entire CoAIleague platform.

### Behavior
```
Default state:    Small FAB, bottom right, Trinity logo/icon, subtle pulse animation
Hover/tap:        Expands to show quick action chips
Open state:       Full chat panel slides up (mobile) or pops out (desktop)
Scrolling:        Hides when scrolling down, reappears when scrolling up or stops
Conflict avoid:   Moves left if bottom nav or other FABs are nearby
```

### Visual Specs
```
Size:             56px circle (default), 44px when scrolling hides it partially
Background:       Brand gradient (cyan → purple)
Icon:             Trinity logo or AI sparkle icon
Shadow:           0 4px 20px rgba(0,212,255,0.3) — glowing cyan shadow
Pulse:            Subtle scale(1) → scale(1.05) → scale(1) every 3s when idle
Badge:            Red dot when Trinity has a proactive suggestion waiting
Position:         Fixed, bottom: 24px + safe area, right: 20px
Mobile:           Bottom: 80px (above bottom nav)
Desktop:          Bottom: 24px, right: 24px
```

### Context Awareness
When the user opens the Trinity widget, it automatically:
1. Detects the current page/route
2. Pulls the page title and primary data context
3. Generates a contextual greeting

```typescript
// Context detection per route
const trinityContext = {
  '/dashboard':     'Looking at your dashboard. I can explain any widget or help you navigate.',
  '/scheduling':    'You\'re on the schedule. I can help you spot conflicts or explain coverage gaps.',
  '/timesheets':    'Viewing timesheets. I can help you find discrepancies or explain overtime calculations.',
  '/payroll':       'On the payroll page. Ask me anything about how pay is calculated.',
  '/billing':       'Viewing billing. I can explain any charge, plan, or invoice.',
  '/compliance':    'Compliance page. I can tell you what\'s expiring soon and what needs action.',
  // ... all routes
};
```

### Screen Analysis Feature
When the user taps "Analyze this page" inside the Trinity widget:
1. Capture current viewport as screenshot using `html2canvas`
2. Send to Trinity with the page context
3. Trinity describes what she sees and offers insights or next steps
4. Response appears in the chat panel

### Quick Action Chips (shown before opening full chat)
These appear when the FAB is tapped once — contextual to the current page:

```
On Dashboard:     [ What do my earnings mean? ] [ What's my next shift? ] [ How do I clock in? ]
On Scheduling:    [ Any conflicts this week? ] [ Who's uncovered? ] [ Explain this shift ]
On Timesheets:    [ Any overtime? ] [ Explain my hours ] [ What's pending approval? ]
On Payroll:       [ How was this calculated? ] [ When do I get paid? ] [ Explain deductions ]
On Compliance:    [ What's expiring soon? ] [ What do I need to complete? ] [ Explain this requirement ]
```

### Trinity Widget Component
```typescript
// src/components/trinity/TrinityWidget.tsx
interface TrinityWidgetProps {
  route: string;
  userRole: 'employee' | 'manager' | 'admin' | 'org_owner';
  orgId: string;
  userId: string;
}
```

**Wire this to every page layout through UniversalLayout so it appears automatically everywhere.**

## PART 3 — GUIDED ONBOARDING ANIMATIONS

When a new org owner logs in for the first time, Trinity guides them through setup step by step with animated, helpful instructions. This is not a boring checklist — it's an interactive guided experience.

### Onboarding Flow

**Step 1 — Welcome**
```
Animated Trinity avatar appears
Greeting: "Welcome to CoAIleague! I'm Trinity, your AI workforce assistant.
           Let me help you set up your organization in just a few steps."
CTA: [ Let's Get Started → ]
```

**Step 2 — Spotlight Animation**
When Trinity guides a user to click something:
- A pulsing spotlight circle highlights the target element
- An animated arrow points to it
- Trinity speech bubble explains what to do
- Backdrop dims everything except the spotlight target

```typescript
// Spotlight component
interface SpotlightProps {
  targetSelector: string;    // CSS selector of element to highlight
  message: string;           // Trinity's instruction
  position: 'top' | 'bottom' | 'left' | 'right';
  onComplete: () => void;
}
```

**Step 3 — Progress Indicator**
```
○ ● ○ ○ ○ ○   Step 2 of 6: Add your first employee
─────────────────────────────────────
Animated progress bar fills as steps complete
Each completed step shows a green checkmark with a satisfying pop animation
```

**Step 4 — Celebration Moments**
When a setup step is completed:
- Brief confetti burst animation (CSS only, no heavy libraries)
- Trinity: "Great job! Your payroll is now configured."
- Smooth transition to next step

### Onboarding Steps to Guide
1. Organization profile — name, logo, address
2. Add first site/location
3. Add first employee
4. Configure pay period and rates
5. Connect Stripe for billing
6. Create first schedule

### Returning User Nudges
For users who started onboarding but didn't finish — Trinity shows a resume card:
```
"You're 60% set up. Finish adding your employees to unlock full scheduling."
[ Continue Setup → ]
```

## PART 6 — UNIVERSAL MODAL/DRAWER/SHEET HEADERS

Every modal, drawer, and sheet gets a consistent header with this exact structure:

```
╔══════════════════════════════════════════╗
║  ← Back  [Title]              [✕ Close] ║
║  [Subtitle if needed]                    ║
║  ─────────────────────────────────────── ║
```

```typescript
// All modals, drawers, sheets use this header
interface UniversalOverlayHeader {
  title: string;
  subtitle?: string;
  showBack?: boolean;
  onBack?: () => void;
  onClose: () => void;
  actions?: React.ReactNode;  // additional header actions
}
```

**Rules:**
- Title always present — no untitled overlays
- Close button always top right — no exceptions
- Back button on mobile when navigating within an overlay
- Subtitle optional but encouraged for context
- Actions slot for page-specific header buttons

```typescript

  // requestAnimationFrame counter from 0 to target
```
