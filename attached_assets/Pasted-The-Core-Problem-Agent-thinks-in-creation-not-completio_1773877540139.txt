The Core Problem
Agent thinks in creation not completion. It measures done by whether the code exists, not whether the code actually works end to end. So you get:

Route defined ✅ — but never called from the frontend ❌
Database table created ✅ — but never seeded with real data ❌
Component built ✅ — but wired to a dead endpoint ❌
Document record inserted ✅ — but no actual file behind it ❌
Event subscriber registered ✅ — but no publisher ever fires it ❌
Action registered ✅ — but nothing ever calls it ❌

716 actions. 72 orphaned subscribers. 15 financial events never published. Placeholder document paths. All the same root cause — creation without completion.

WHAT A COMPLETE FEATURE LOOKS LIKE
Every feature must pass all 8 checkpoints to be 
considered complete. If any checkpoint fails, 
the feature is INCOMPLETE regardless of how much 
code exists behind it.
═══════════════════════════════════════════════════════

CHECKPOINT 1 — UI ENTRY POINT
Does the button, form, link, or trigger actually 
exist and render in the frontend for the correct role?

CHECKPOINT 2 — FRONTEND TO API
Does the frontend call make it to the correct 
API route? Is the route mounted? Is it protected 
with the correct role guard? Does the request 
include workspace_id?

CHECKPOINT 3 — VALIDATION
Does the API validate the input correctly? 
Are numeric fields coerced before validation? 
Do invalid inputs return clean error messages 
back to the frontend?

CHECKPOINT 4 — BUSINESS LOGIC
Does the service layer execute the correct 
business logic? Are edge cases handled? 
Are constraints enforced?

CHECKPOINT 5 — DATABASE WRITE
Does the data actually get written to the 
correct table with all required fields populated? 
Is workspace_id present? Are relationships 
correctly linked?

CHECKPOINT 6 — EVENT PUBLISHED
Does the correct platform event get published 
after the operation? Does Trinity get notified? 
Do any other subscribers receive the event?

CHECKPOINT 7 — RESPONSE AND CACHE
Does a success response return to the frontend? 
Does the frontend cache invalidate so the UI 
reflects the change immediately without a 
page refresh?

CHECKPOINT 8 — REAL DATA VISIBLE
Can a real user see the result of their action 
immediately after completing it? Is the data 
visible, correct, and properly formatted on screen?
