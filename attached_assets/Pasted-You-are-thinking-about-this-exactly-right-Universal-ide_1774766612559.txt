**1 — ORGANIZATION ID (Workspace)**

```
Format: ORG-[STATE]-[NNNNN]
Example: ORG-TX-00142

Generated: on workspace creation
Stored in: workspaces.org_id
Unique: platform-wide
Used for: 
  - Trinity voice identification
  - Platform-wide search
  - Document headers
  - SRA audit references
  - API keys prefix
  - Support tickets
  - Billing references
```

**On phone:** "Your organization ID is ORG-TX-00142. Please have this ready when calling."

**2 — CLIENT IDENTIFICATION NUMBER**

```
Format: CLT-[ORG_SHORT]-[NNNNN]  
Example: CLT-ACM-00891

Generated: when client is created
Stored in: clients.client_number
Unique: within workspace (globally unique with prefix)
Used for:
  - Trinity voice caller identification
  - Invoice references
  - Contract references
  - Support ticket routing
  - Document attachment
  - Portal login identifier
```

**On phone:** "Please enter or say your 3-letter organization code followed by your 5-digit client number."

Trinity hears "ACM zero zero eight nine one" → instantly pulls up client record → knows their posts, their account manager, their open invoices, their complaint history.

**3 — EMPLOYEE IDENTIFICATION NUMBER**

```
Format: EMP-[ORG_SHORT]-[NNNNN]
Example: EMP-ACM-00034

Generated: when officer is created
Stored in: employees.employee_number
Unique: within workspace
Used for:
  - Trinity voice identification
  - Clock-in verification (voice or keypad)
  - Pay stub references
  - Schedule references
  - Document attachment
  - Incident report linking
  - Onboarding tracking
  - Performance records
  - Background verification reference
```

**4 — CLOCK-IN PIN**

This is separate from employee number. Specifically for clock-in verification.

```
Format: 6-digit numeric PIN
Example: 847291

Created by: officer sets their own PIN on first login
Stored as: bcrypt hash in employees.clockin_pin_hash
Used for:
  - Voice clock-in verification
  - Kiosk clock-in
  - Emergency clock-in when internet unavailable
  - Never the same as their account password
  
Verification on voice call:
  "Please enter or say your 6-digit clock-in PIN"
  Trinity compares against hash — never stores raw PIN
```

**Why PIN not SSN:**
SSN over a phone call is a liability. A 6-digit PIN the officer creates is secure, easy to say, and if compromised only affects clock-in not their identity. SSN should never be spoken on any call.

**SYNC WITH ENTIRE PLATFORM**

For this to work IDs need to be added to every system:

```bash
# Tables needing human-readable ID columns added:
ALTER TABLE workspaces ADD COLUMN org_id VARCHAR(20) UNIQUE;
ALTER TABLE clients ADD COLUMN client_number VARCHAR(20) UNIQUE;
ALTER TABLE employees ADD COLUMN employee_number VARCHAR(20) UNIQUE;
ALTER TABLE users ADD COLUMN user_number VARCHAR(20) UNIQUE;
ALTER TABLE employees ADD COLUMN clockin_pin_hash VARCHAR(255);

# Auto-generation functions:
CREATE OR REPLACE FUNCTION generate_org_id(state_code TEXT)
RETURNS TEXT AS $$
DECLARE
  next_num INTEGER;
BEGIN
  SELECT COALESCE(MAX(
    CAST(SUBSTRING(org_id FROM 8) AS INTEGER)
  ), 0) + 1
  INTO next_num
  FROM workspaces
  WHERE org_id LIKE 'ORG-' || state_code || '-%';
  
  RETURN 'ORG-' || state_code || '-' || 
         LPAD(next_num::TEXT, 5, '0');
END;
$$ LANGUAGE plpgsql;

# Similar functions for CLT, EMP, USR formats

# Backfill existing records:
UPDATE workspaces SET org_id = generate_org_id('TX')
WHERE org_id IS NULL;

UPDATE employees SET employee_number = 
  'EMP-' || SUBSTRING(w.company_name FOR 3) || '-' || 
  LPAD(ROW_NUMBER() OVER()::TEXT, 5, '0')
FROM workspaces w WHERE w.id = employees.workspace_id
AND employees.employee_number IS NULL;
```

**Search indexes on all ID columns:**
```sql
CREATE INDEX idx_workspaces_org_id ON workspaces(org_id);
CREATE INDEX idx_clients_client_number ON clients(client_number);
CREATE INDEX idx_employees_employee_number ON employees(employee_number);
CREATE INDEX idx_users_user_number ON users(user_number);

-- GIN trigram for partial search
CREATE INDEX idx_employees_emp_num_trgm 
ON employees USING gin(employee_number gin_trgm_ops);
```

**Trinity search using IDs:**
```typescript
// Universal ID resolver in Trinity
async function resolveEntityByID(id: string, workspaceId: string) {
  if (id.startsWith('ORG-')) {
    return db.query('SELECT * FROM workspaces WHERE org_id = $1', [id]);
  }
  if (id.startsWith('CLT-')) {
    return db.query('SELECT * FROM clients WHERE client_number = $1 AND workspace_id = $2', [id, workspaceId]);
  }
  if (id.startsWith('EMP-')) {
    return db.query('SELECT * FROM employees WHERE employee_number = $1 AND workspace_id = $2', [id, workspaceId]);
  }
  if (id.startsWith('USR-')) {
    return db.query('SELECT * FROM users WHERE user_number = $1', [id]);
  }
  if (id.startsWith('DOC-')) {
    return db.query('SELECT * FROM document_vault WHERE document_number = $1 AND workspace_id = $2', [id, workspaceId]);
  }
  if (id.startsWith('INV-')) {
    return db.query('SELECT * FROM invoices WHERE invoice_number = $1 AND workspace_id = $2', [id, workspaceId]);
  }
  if (id.startsWith('CLK-')) {
    return db.query('SELECT * FROM time_entries WHERE reference_id = $1 AND workspace_id = $2', [id, workspaceId]);
  }
}
```
