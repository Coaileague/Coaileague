**SECTION 3 — TRAINING ROUTES**

Create `server/routes/trainingRoutes.ts` — mount at `/api/training` with `requireAuth` at mount level.

**`GET /api/training/modules`**
Returns all modules available to the workspace — platform defaults plus workspace custom modules. Include completion status per employee if `employeeId` query param provided.

**`GET /api/training/modules/:id`**
Returns module with sections. Does NOT return correct answers — only question text and options.

**`POST /api/training/attempts/start`**
```
Body: { moduleId, employeeId }
7-Step:
1. TRIGGER — Employee or manager initiates training
2. FETCH — Get module, check existing attempts, verify not already certified for current period
3. VALIDATE — Confirm workspace match, confirm employee is active, confirm no valid unexpired certificate exists (cannot retake until anniversary)
4. PROCESS — Calculate attempt number, set attempt_type
5. MUTATE — INSERT training_attempt record with started_at
6. CONFIRM — Return attempt ID with first section content
7. NOTIFY — No notification on start
```

**`POST /api/training/attempts/:id/submit-section`**
```
Body: { sectionId, answers: { questionId: selectedAnswer }[] }
7-Step:
1. TRIGGER — Employee submits section quiz
2. FETCH — Get section questions with correct answers, get attempt record
3. VALIDATE — Verify attempt belongs to employee and workspace, verify section not already submitted
4. PROCESS — Calculate section score, identify missed questions
5. MUTATE — Update attempt.section_scores with this section result, update current_section_index
6. CONFIRM — Return score and whether to advance or review
7. NOTIFY — No notification on section completion
Return: { sectionScore, passed, missedTopics, nextSection or 'final_exam' }
```

**`POST /api/training/attempts/:id/submit-final`**
```
Body: { answers: { questionId: selectedAnswer }[] }
7-Step:
1. TRIGGER — Employee submits final exam
2. FETCH — Get all final exam questions, get all section scores, get attempt record
3. VALIDATE — Verify all sections completed first, verify attempt is in progress
4. PROCESS — Calculate final exam score, calculate overall score (weighted: sections 40%, final 60%), determine passed
5. MUTATE:
   IF PASSED:
   - UPDATE attempt: passed=true, overall_score, completed_at
   - Generate certificate number: `CERT-[WORKSPACE-4]-[EMPLOYEE-4]-[MODULE-4]-[TIMESTAMP]`
   - Set expires_at = completed_at + module.certificateValidDays
   - Generate certificate PDF (see Section 4)
   - INSERT training_certificates record
   - INSERT document record linking certificate to employee document safe
   - UPDATE employee: training_completion_percentage, compliance_score
   IF FAILED:
   - UPDATE attempt: passed=false, overall_score, completed_at
   - Increment attempt_number
   - IF attempt_number >= module.maxAttemptsBeforeIntervention:
     - Set flagged_for_intervention = true
     - INSERT training_interventions record
     - UPDATE employee compliance_score -= 5
6. CONFIRM — Verify certificate stored if passed
7. NOTIFY:
   IF PASSED: notify employee + org owner, publish training_certificate_earned
   IF FAILED + INTERVENTION: notify manager + owner, publish training_intervention_required
   IF FAILED + NO INTERVENTION YET: notify employee of failed attempt, attempts remaining
```

**`GET /api/training/employees/:employeeId/transcript`**
Returns complete training history — all modules, all attempts, all certificates, all interventions. Workspace scoped.

**`GET /api/training/compliance-report`**
Returns workspace-wide report — per employee completion status, certificates valid/expired, upcoming renewals, intervention flags, total hours, compliance percentage.

**`POST /api/training/interventions/:id/complete`**
Logs completed intervention — who conducted it, duration, notes, outcome. Notifies employee they can retake.

**`GET /api/training/certificates/:certificateNumber/verify`**
Public endpoint — no auth required. Returns certificate validity, employee name (first name + last initial only), module name, issue date, expiry. Used for QR code verification.

**SECTION 5 — EMPLOYEE COMPLIANCE SCORE INTEGRATION**

Read the existing employee scoring system before touching it. Extend it — do not replace it.

**Compliance score calculation (add to existing scoring logic):**

The compliance score (0-100) is reduced by:
- Each required training module not completed: -5 points
- Each day a required training is overdue past due date: -1 point per day (configurable)
- Each unacknowledged post order or SOP past 24 hours of assignment: -3 points
- Each unsigned required document past 48 hours of assignment: -2 points
- Each training intervention flag not yet resolved: -10 points
- Each expired certificate not renewed within 30 days of expiry: -5 points

The compliance score recovers when:
- Training is completed: +5 points
- Document is signed: +2 points
- Post order is acknowledged: +3 points
- Intervention is resolved and test passed: +10 points

**Wire compliance score to scheduling:**
- Score 90-100: Preferred officer — appears first in auto-fill candidates
- Score 70-89: Standard — normal priority
- Score 50-69: Discouraged — appears last in auto-fill, manager sees amber warning badge
- Below 50: Flagged — manager sees red warning badge, Trinity includes in morning brief
- Score never blocks shift assignment — it only affects priority and shows warnings

**Add to employee list and profile:**
- Compliance score badge next to employee name
- Tooltip showing what is reducing the score with direct links to fix each item

**SECTION 6 — ANNIVERSARY RENEWAL SYSTEM**

Create `server/services/training/trainingRenewalService.ts`:

Daily cron — check all valid certificates:
- If `expires_at` is within 30 days — send reminder to employee: "Your [Module] certificate expires in [N] days. Complete renewal training to maintain your compliance score."
- If `expires_at` has passed — mark certificate `is_valid = false`, reduce employee compliance score, notify employee and manager
- If expired and not renewed within 30 days — add to morning brief as intervention item

On certificate expiry:
- Employee can start a new attempt (anniversary allows retake)
- Previous passing score displayed for reference
- If they pass renewal — new certificate issued, score recovered
