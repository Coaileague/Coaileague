Build a Diagnostics Subagent that runs overnight E2E audits on my LIVE published website (hosted via Replit) and generates a full diagnostic report with screenshots/video/logs.

Use Node.js + Playwright.

IMPORTANT: CAPTCHA handling
- The site may show CAPTCHA sometimes (not fully installed). The runner must NEVER hang.
- Add a CAPTCHA detector:
  - detect presence of common captcha elements: reCAPTCHA iframe, hcaptcha iframe, elements with id/class containing "captcha", "recaptcha", "hcaptcha"
  - detect “I’m not a robot” text
- If CAPTCHA is detected:
  - take a full-page screenshot
  - save HTML snapshot of the page
  - record it in the report as “CAPTCHA BLOCKER”
  - SKIP that step/page/workflow and continue with remaining tests

Core behavior
1) Crawl mode:
- Start at BASE_URL, discover internal links (same domain) up to MAX_PAGES (default 300)
- For each visited page:
  - capture full-page screenshot
  - record console errors (console + pageerror)
  - record failed network requests and any responses >= 400 (especially fetch/xhr)
  - detect broken images (naturalWidth==0 OR asset 4xx/5xx)
  - detect broken links (optional external check configurable)
  - detect “error UI” keywords: error, failed, exception, 404, 500, something went wrong
- Save artifacts per page

2) Workflow mode:
- Support workflows.json steps: goto, click, fill, waitForURL, waitForSelector, assertVisible, assertText, select, upload
- Example workflows:
  - Homepage load
  - Login with TEST_USERNAME/TEST_PASSWORD
  - Navigate to 5 key pages
  - Submit one form with dummy data
- Each step:
  - screenshot before/after important actions
  - on failure, save screenshot + trace + video (if enabled)
- Continue even if one workflow fails

Artifacts/Report
- Create /reports/<timestamp>/:
  - report.html (shareable)
  - summary.json
  - screenshots/
  - videos/ (optional)
  - traces/ (optional)
  - logs/ (console + network)
  - html_snapshots/ (on captcha/fail)
- HTML report groups failures:
  - CAPTCHA Blockers
  - Console Errors
  - Page Errors
  - Network/API Failures (4xx/5xx)
  - Broken Images
  - Broken Links
  - Workflow Step Failures
- Each issue includes:
  - URL, step name, timestamp
  - error message/stack
  - request URL/status for API failures
  - screenshot link (and video/trace link if present)

Reliability
- Add timeouts + retries (2 retries) for navigation/click
- Add “safe-click” guardrail to avoid destructive actions:
  - never click elements containing: delete, remove, logout, unsubscribe, cancel, destroy, pay, confirm payment
  - support allowlist selectors in config
- Never hang: global per-page timeout and per-workflow timeout; on timeout capture screenshot + continue.

Commands
- Add scripts:
  - npm run diag
  - npm run diag:crawl
  - npm run diag:workflows
  - npm run diag:nightly

Deliverables
- /diagnostics-runner/ folder containing runner code
- workflows.json example
- README.md with setup:
  - how to set env vars in Replit Secrets
  - how to run nightly
