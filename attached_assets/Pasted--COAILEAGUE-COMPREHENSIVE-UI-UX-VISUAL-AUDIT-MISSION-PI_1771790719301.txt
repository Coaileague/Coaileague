# PART 1: MODAL / SHEET / DIALOG AUDIT

## 1.1 The Problem You Showed Me

The screenshot shows a **Help Desk bottom sheet** that:
- Stretches across the ENTIRE viewport width on desktop
- Should be compact and centered (max-width ~400-500px)
- Options (Mute, Archive, Leave) have too much empty space
- Cancel button spans full width unnecessarily

**This is a UNIVERSAL pattern issue** - likely affects all modals/sheets/dialogs.

```bash
# Find all modal/dialog/sheet components

```

## 1.4 Standard Modal Sizing Guidelines

```typescript
/**
 * MODAL SIZE STANDARDS
 * 
 * Apply these max-widths consistently:
 */

const MODAL_SIZES = {
  // Small: Confirmations, simple actions
  sm: 'max-w-sm',      // 384px - "Are you sure?" dialogs
  
  // Medium: Forms, options, settings
  md: 'max-w-md',      // 448px - Most common size
  
  // Large: Complex forms, multi-step
  lg: 'max-w-lg',      // 512px - Detailed forms
  
  // XL: Data tables, previews
  xl: 'max-w-xl',      // 576px - Content-heavy modals
  
  // 2XL: Full editors, complex UIs
  '2xl': 'max-w-2xl',  // 672px - Maximum for modals
  
  // Full: Rare, usually mobile only
  full: 'max-w-full',  // 100% - Avoid on desktop
};

/**
 * SHEET SIZE STANDARDS (Bottom/Side Sheets)
 */

const SHEET_SIZES = {
  // Bottom sheets on mobile: full width, auto height
  bottomMobile: 'w-full max-h-[85vh]',
  
  // Bottom sheets on desktop: centered, constrained
  bottomDesktop: 'max-w-md mx-auto',
  
  // Side sheets: fixed width
  sideSheet: 'w-80 max-w-[90vw]', // 320px, but never more than 90%
};
```

## 1.5 Fix Pattern for Modals

```tsx
// BAD - No max-width, stretches full screen
<DialogContent className="w-full">
  {content}
</DialogContent>

// GOOD - Constrained width, responsive
<DialogContent className="w-full max-w-md mx-auto sm:max-w-lg">
  {content}
</DialogContent>

// BAD - Bottom sheet full width on desktop
<SheetContent side="bottom" className="w-full">
  {content}
</SheetContent>

// GOOD - Bottom sheet constrained on desktop
<SheetContent 
  side="bottom" 
  className="w-full sm:max-w-md sm:mx-auto sm:rounded-t-xl"
>
  {content}
</SheetContent>
```

## 2.2 Text Overflow Checklist

```
For EVERY text element:

TRUNCATION:
[ ] Long text truncates with ellipsis (if single line)
[ ] Multi-line text has line-clamp
[ ] Tooltips show full text on hover (for truncated)

CONTAINERS:
[ ] Text container has overflow handling
[ ] Flex items have min-w-0 (prevents flex blowout)
[ ] Grid items have overflow handling

RESPONSIVE:
[ ] Text doesn't overflow on mobile
[ ] Font sizes appropriate for viewport
[ ] Line heights comfortable
```

## 2.3 Common Text Overflow Fixes

```tsx
// BAD - Text can blow out flex container
<div className="flex">
  <span>{veryLongText}</span>
</div>

// GOOD - Truncate and prevent blowout
<div className="flex min-w-0">
  <span className="truncate">{veryLongText}</span>
</div>

// BAD - Button text can overflow
<Button>{dynamicText}</Button>

// GOOD - Button text truncates
<Button className="max-w-full">
  <span className="truncate">{dynamicText}</span>
</Button>

// Multi-line truncation
<p className="line-clamp-2">{description}</p>
<p className="line-clamp-3">{longerDescription}</p>
```

  onOpenChange: (open: boolean) => void;
  title?: string;
  description?: string;
  children: React.ReactNode;
  size?: 'sm' | 'md' | 'lg';
}

const SHEET_SIZES = {
  sm: 'max-w-sm',
  md: 'max-w-md', 
  lg: 'max-w-lg',
};

export function BottomSheet({ 
  open, 
  onOpenChange, 
  title, 
  description, 
  children,
  size = 'md'
}: BottomSheetProps) {
  return (
    <Sheet open={open} onOpenChange={onOpenChange}>
      <SheetContent 
        side="bottom"
        className={cn(
          "mx-auto rounded-t-2xl pb-safe",
          SHEET_SIZES[size],
          // On very small screens, go full width with small margin
          "w-[calc(100%-2rem)] sm:w-full"
        )}
      >
        {/* Drag handle indicator */}
        <div className="mx-auto mb-4 h-1.5 w-12 rounded-full bg-muted" />
        
        {(title || description) && (
          <div className="mb-6 text-center">
            {title && <SheetTitle>{title}</SheetTitle>}
            {description && (
              <SheetDescription>{description}</SheetDescription>
            )}
          </div>
        )}
        
        <div className="px-1">
          {children}
        </div>
      </SheetContent>
    </Sheet>
  );
}
