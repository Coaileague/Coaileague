**Org bank account connection:**

`POST /api/payroll/bank/link-token` — returns Plaid link token. Auth required, workspace-scoped.

`POST /api/payroll/bank/connect` — receives `publicToken` from Plaid Link. Exchanges for access token. Stores encrypted `plaidAccessToken` and `plaidItemId` in workspace settings. Never stores raw routing or account numbers. Writes audit log with `isSensitiveData: true`. Publishes `workspace_bank_connected` to platformEventBus.

`GET /api/payroll/bank/status` — returns `{ connected: boolean, accountMask: string | null }`. Never returns full account details.

`DELETE /api/payroll/bank/disconnect` — removes access token, writes audit log, publishes `workspace_bank_disconnected`.

**Employee direct deposit via Plaid:**

`POST /api/payroll/employees/:employeeId/bank/link-token` — employee-scoped link token.

`POST /api/payroll/employees/:employeeId/bank/connect` — exchanges token, stores encrypted `plaidAccessToken` and `plaidItemId` in `employee_bank_accounts` table. Add `plaid_access_token` (encrypted) and `plaid_item_id` columns if not present. Display as last-4 only everywhere.

**Payroll disbursement via Plaid Transfer:**

In `POST /api/payroll/runs/:id/process` — after pay stubs are created, execute Plaid transfers as the primary disbursement step:

For each employee WITH a connected Plaid account:
1. Call `plaidService.verifyBankAccount()` — confirm account valid and active
2. Call `plaidService.initiateTransfer()` — initiate ACH push to employee bank
3. Store `transferId` on pay stub record
4. Set pay stub status to `transfer_initiated`
5. Publish `{ type: 'payroll_transfer_initiated', workspaceId, employeeId, transferId, amount }` to platformEventBus

For each employee WITHOUT a connected Plaid account — generate NACHA entry for that employee only. Flag as `requires_manual_disbursement` in run summary.

Update payroll run status to `disbursing` after all transfers initiated.

**Transfer monitoring:**

Create `server/services/payroll/payrollTransferMonitor.ts` — cron every 2 hours:
- For every payroll run in `disbursing` status, call `getTransferStatus()` for each pending transfer
- `settled` → pay stub to `paid`, emit `payroll_transfer_settled`
- `failed` → pay stub to `failed`, emit `payroll_transfer_failed`, notify org owner with employee name, amount, and failure reason
- All transfers settled → run transitions to `paid`, emit `payroll_run_paid`
- If `getTransferStatus()` throws — write to `trinity_credit_failures`, notify org owner, increment `consecutive_poll_failures` counter — after 3 consecutive failures set pay stub to `poll_failed`

Add `POST /api/payroll/runs/:id/retry-failed-transfers` — retry failed transfers once. Manager role required.
