THE REAL PROBLEM: ID MISMATCH
Frontend sends: Room slug from URL (c0ea8807-...)
Server returns: Conversation UUID (2ba4e086-...)
Filter compares: c0ea8807-... !== 2ba4e086-... → REJECTED
URL: /chatrooms/c0ea8807-4b8a-4f2e-9c1d-abc123
                ↑ This is the SLUG

Server message: { conversationId: "2ba4e086-5d7f-4a3b-8e2c-def456", ... }
                                   ↑ This is the actual UUID

Hook filter: if (data.conversationId !== "c0ea8807-...") REJECT
                                          ↑ Comparing apples to oranges
