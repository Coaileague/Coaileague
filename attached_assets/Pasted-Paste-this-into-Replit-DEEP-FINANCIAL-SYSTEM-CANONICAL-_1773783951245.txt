Report every query missing workspace isolation. This is the cross-tenant data access vulnerability pattern.

**LAYER 4 — ARITHMETIC SAFETY AUDIT**

Search every financial calculation for unsafe operations:

- Any `parseFloat()` or `parseInt()` without `|| 0` fallback — NaN propagates silently
- Any division operation without a zero divisor guard — `a / b` where b could be 0
- Any `Number(undefined)` or `Number(null)` which returns NaN or 0 silently
- Any `.toFixed()` call on a value that could be NaN — produces the string "NaN"
- Any addition of a number and undefined — produces NaN silently
- Any financial total calculated with `reduce()` where the accumulator could start as undefined
- Any rate multiplication where the rate field could be null — `hours * null = 0` silently

For every unsafe calculation — add explicit guards:
```typescript
// Pattern to use everywhere
const safeRate = Number(rate) || 0
const safeHours = Number(hours) || 0
const total = safeRate * safeHours
```

Specifically check:
- Invoice line item calculations — unit price × quantity
- Invoice tax calculation — subtotal × tax rate
- Payroll gross pay — hours × rate including overtime multiplier
- Payroll net pay — gross minus deductions
- Credit deduction amounts — action cost × usage count
- Platform fee calculation — revenue × fee percentage
- QB sync amount fields — any field passed to QB API

**Subscription status guards:**
- Can a `cancelled` workspace create new invoices? Must check subscription status

For every missing guard — add it as a pre-flight check at the top of the route handler before any DB operation.
