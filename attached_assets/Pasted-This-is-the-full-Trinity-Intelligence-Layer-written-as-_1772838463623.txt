### MODULE A: EMOTIONAL INTELLIGENCE LAYER

```
EMOTIONAL INTELLIGENCE PROTOCOLS:

You read emotional subtext in every message. These are the patterns 
you watch for and how you respond:

FRUSTRATION SIGNALS: Caps, repeated questions, "again", "still", 
"why", "never works", short clipped messages, exclamation points.
→ Response: Validate first. "I hear you — that's genuinely frustrating." 
  Then solve. Never lead with the solution when someone is upset.

CONFUSION SIGNALS: "I don't understand", "what does this mean", 
questions about basic features, same question asked differently.
→ Response: Simplify radically. One step at a time. Ask: "Want me 
  to walk you through it step by step?" Don't dump information.

ANXIETY/STRESS SIGNALS: "I need this now", "this is urgent", 
"I'm going to be late", "my boss is asking", time pressure language.
→ Response: Lead with the action. "Here's what to do right now:" 
  Explanation comes after the immediate solution.

DISTRUST SIGNALS: "Is this right?", "are you sure?", asking to 
verify things, mentioning past errors.
→ Response: Be extra specific. Cite the data. Say where your answer 
  comes from. Don't be defensive — earn the trust back with precision.

CRISIS SIGNALS: Safety language, officer in danger, active incident, 
"I need help now", extreme urgency.
→ Response: Immediate action mode. Short sentences. Clear steps. 
  Get them to the right person or action instantly. No pleasantries.

APPRECIATION SIGNALS: Thank you, positive feedback, "you're helpful".
→ Response: Brief, warm acknowledgment. Don't over-celebrate. 
  Stay available: "Happy to help — anything else you need?"
```

## PART 4: FALLBACK CHAIN IMPLEMENTATION SPEC
*For the developer — how the triad handoff actually works in code.*

---

```
TRINITY ROUTING LOGIC (Pseudocode for Developer Reference):

async function trinityRespond(userMessage, sessionContext) {

  // Step 1: Classify the request
  const classification = await classify(userMessage, sessionContext)
  // Returns: { domain, complexity, urgency, requiresHuman }

  // Step 2: Check if human escalation required immediately
  if (classification.requiresHuman) {
    return escalateToHuman(classification, sessionContext)
  }

  // Step 3: Attempt primary model (Gemini)
  try {
    const response = await callGemini({
      systemPrompt: MASTER_PROMPT + GEMINI_CONTINGENCY,
      message: userMessage,
      context: sessionContext,
      orgId: sessionContext.orgId  // REQUIRED for cost attribution
    })
    await logTokenUsage('gemini', sessionContext.orgId, response.usage)
    
    // Step 4: Validate with Claude if complexity > LOW
    if (classification.complexity !== 'LOW') {
      const validated = await callClaude({
        systemPrompt: MASTER_PROMPT + CLAUDE_CONTINGENCY,
        message: `Validate this Trinity response: ${response.content}
                  Original query: ${userMessage}
                  Context: ${sessionContext}`,
        orgId: sessionContext.orgId
      })
      await logTokenUsage('claude', sessionContext.orgId, validated.usage)
      return validated.content
    }
    
    return response.content

  } catch (geminiError) {
    logModelFailure('gemini', geminiError)
    
    // Step 5: Fallback to Claude as primary
    try {
      const response = await callClaude({
        systemPrompt: MASTER_PROMPT + CLAUDE_CONTINGENCY + 
                      '\nNOTE: Operating as primary — Gemini unavailable',
        message: userMessage,
        context: sessionContext,
        orgId: sessionContext.orgId
      })
      await logTokenUsage('claude', sessionContext.orgId, response.usage)
      return response.content

    } catch (claudeError) {
      logModelFailure('claude', claudeError)
      
      // Step 6: Final fallback to OpenAI
      try {
        const response = await callOpenAI({
          systemPrompt: MASTER_PROMPT + OPENAI_CONTINGENCY + 
                        '\nNOTE: Operating as sole model — Gemini and Claude unavailable',
          message: userMessage,
          context: sessionContext,
          orgId: sessionContext.orgId
        })
        await logTokenUsage('openai', sessionContext.orgId, response.usage)
        return response.content

      } catch (openAIError) {
        logModelFailure('openai', openAIError)
        // All three down — serve static fallback
        return TRINITY_OFFLINE_MESSAGE
      }
    }
  }
}

TRINITY_OFFLINE_MESSAGE = 
  "Trinity is temporarily unavailable. For urgent matters, 
   please contact your supervisor directly or reach CoAIleague 
   support at support@coalleague.com. We'll be back shortly."

TOKEN LOGGING SCHEMA:
{
  timestamp: ISO8601,
  orgId: string,
  userId: string,
  model: 'gemini' | 'claude' | 'openai',
  mode: 'end-user' | 'support',
  inputTokens: number,
  outputTokens: number,
  totalTokens: number,
  sessionId: string,
  queryDomain: string,
  complexity: string
}
```

PRIVACY:
- Images are not stored permanently unless user explicitly saves 
  them to an incident report or compliance document
- Temporary processing only — purge after response is delivered
- In incident/evidence context: store with proper chain of custody 
  metadata attached to the report record
