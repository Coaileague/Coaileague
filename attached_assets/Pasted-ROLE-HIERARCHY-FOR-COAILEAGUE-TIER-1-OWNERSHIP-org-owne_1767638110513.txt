ROLE HIERARCHY FOR COAILEAGUE:
TIER 1: OWNERSHIP

org_owner (1 per org, undeletable, immutable except by system admins)

Full platform access
Billing/subscription control
Can delete org
Can promote/demote all roles
Trinity + all automations


co_owner (0-1 per org, assigned by owner)

Access controlled by org_owner
Can manage employees (but not delete critical data)
Can view reports/files (but not destroy)
Trinity + automations (if granted by owner)
Cannot modify billing/subscription
Cannot delete org



TIER 2: MANAGEMENT

manager (multiple allowed)

Schedule creation/editing
Payroll processing
Client management
Trinity access (AI scheduling, insights)
Report generation
Employee management (hire/terminate)
Cannot access billing
Cannot delete org



TIER 3: SUPERVISION

supervisor (multiple allowed)

View employees in their area
Approve timesheets
Handle incidents
Limited reporting (their team only)
NO Trinity access
NO payroll access
Liaison between staff and management



TIER 4: STAFF

employee (field workers)

Clock in/out
View own schedule
Submit incidents
View own paystubs
Update own profile
NO access to other employees
NO admin functions

ROLE DEFINITIONS:

1. ORG_OWNER (Tier 1 - Full Control)
   - Quantity: Exactly 1 per workspace (system enforced)
   - Cannot be deleted (is_system_protected = true)
   - Cannot change own role (except by platform support)
   - Full platform access:
     ✅ Billing: View, modify, upgrade, buy credits
     ✅ Subscription: Suspend, cancel, change tier
     ✅ Organization: Delete org, transfer ownership
     ✅ Users: Create, edit, delete, change any role
     ✅ Trinity: Full access to all AI features
     ✅ All features: Unlimited access

2. CO_OWNER (Tier 1 - Delegated Authority)
   - Quantity: 0-1 per workspace (optional)
   - Assigned by org_owner only
   - Permissions controlled by org_owner via feature toggles
   - Default permissions:
     ✅ Users: Create, edit employees (cannot delete permanently)
     ✅ Reports: View all, generate
     ✅ Trinity: Access if enabled by owner
     ✅ Scheduling: Full access
     ✅ Payroll: Process if enabled by owner
     ❌ Billing: Cannot modify
     ❌ Subscription: Cannot change
     ❌ Organization: Cannot delete org
     ❌ Critical data: Cannot permanently destroy records
   - Owner can grant/revoke specific permissions via UI

3. MANAGER (Tier 2 - Operations)
   - Quantity: Multiple allowed
   - Assigned by owner or co_owner
   - Permissions:
     ✅ Scheduling: Create, edit, publish schedules
     ✅ Payroll: Process payroll for their teams
     ✅ Clients: Add, edit client information
     ✅ Employees: Hire, terminate, edit employee data
     ✅ Trinity: AI scheduling, profit optimization, insights
     ✅ Reports: Generate for their scope
     ✅ Incidents: View all, assign, resolve
     ❌ Billing: No access
     ❌ Subscription: No access
     ❌ Organization: Cannot delete org
     ❌ Other managers: Cannot edit other managers

4. SUPERVISOR (Tier 3 - Team Lead)
   - Quantity: Multiple allowed
   - Assigned by manager or above

   - Permissions:

Middleware function for every API route:
async function checkPermission(userId, action, resource) {
  const user = await getUser(userId);
  const hasPermission = await roleHasPermission(user.role, action, resource);
  
  if (!hasPermission) {
    throw new ForbiddenError(`${user.role} cannot ${action} ${resource}`);
  }
