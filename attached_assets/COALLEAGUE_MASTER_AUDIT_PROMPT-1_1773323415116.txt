- Plan upgrade bonus credits — log with upgrade reference

CREDIT DEDUCTION EVENTS — all must create ledger entries:
- Smart scheduling execution — correct deduction, log action type
- Scheduling session fee — correct deduction, log session reference
- AI contract negotiation — exactly 30 credits per contract, log contract ID
- Payroll analysis — log credit amount and scope
- Schedule analysis — log credit amount and scope
- Notification sends — log per batch
- Any Trinity autonomous action — log action type and exact credit cost
- Any future action type — framework supports new deduction types without architectural change

LEDGER INTEGRITY:
- Balance can never go below zero through deduction — check before every credit action
- If insufficient credits — action blocked immediately, org notified with current balance and cost of attempted action
- Race condition prevention — credit check and deduction are atomic transaction — two simultaneous actions cannot both pass a balance check and overdraw
- Monthly reset never overwrites purchased credits — only resets allocated monthly credits, purchased credits carry forward
- Purchased credits consumed after monthly allocation — monthly credits spent first
