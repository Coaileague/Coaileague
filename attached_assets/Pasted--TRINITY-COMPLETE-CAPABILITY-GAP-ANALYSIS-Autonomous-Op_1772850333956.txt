```typescript
// Runs automatically on the 25th of each month for the following month
// Also triggerable manually: "Trinity, build next month's schedule"

async function trinityMonthlyCycleTrigger(orgId: string, targetMonth: string) {

  // STEP 1: Scan staffing needs
  const staffingNeeds = await scanStaffingRequirements(orgId, targetMonth)
  // Pulls from: client contracts, site requirements, historical schedules,
  // minimum coverage SLAs, day-of-week patterns

  // STEP 2: Scan available workforce  
  const workforce = await getAvailableWorkforce(orgId, targetMonth)
  // Pulls from: employee availability, certifications, license expiry,
  // requested time off, max hours, preferred sites

  // STEP 3: Scan inbound staffing pipeline
  const inboundNeeds = await scanEmailPipeline(orgId)
  // Pulls from: calloff@ inbox, client requests, manually created open shifts
  // Trinity reads these emails and converts them to scheduling needs

  // STEP 4: Run schedule generation
  const draftSchedule = await trinityScheduleEngine({
    orgId,
    period: targetMonth,
    staffingNeeds,
    workforce,
    inboundNeeds,
    rules: await getOrgSchedulingRules(orgId)
    // rules include: OT limits, fatigue windows, certification requirements,
    // client-specific officer requirements, site post orders
  })

  // STEP 5: Self-validate
  const validation = await validateSchedule(draftSchedule)
  // Checks: no conflicts, no expired licenses on armed posts,
  // no OT violations, all SLA minimums met, margin positive on all shifts

  // STEP 6: Auto-resolve conflicts
  const resolved = await autoResolveConflicts(draftSchedule, validation.conflicts)

  // STEP 7: Flag unresolvable items for human review
  const escalations = resolved.unresolvable
  // These get pushed to manager approval queue

  // STEP 8: Publish what's clean, hold what needs review
  await publishSchedule(resolved.clean, orgId)
  await holdForReview(resolved.pending, orgId)

  // STEP 9: Notify
  await notifyManagers(orgId, {
    scheduled: resolved.clean.length,
    pendingReview: resolved.pending.length,
    escalations: escalations
  })
}
```

#### 1B — SCHEDULE INTELLIGENCE RULES TRINITY MUST KNOW

```
HARD RULES (never violate, auto-block):
- Expired PERC/license = no assignment, flag immediately
- Armed post = only armed or above
- Supervisor post = supervisor level only
- No officer on two simultaneous shifts
- No shift starting within 8 hours of previous shift end (fatigue rule)
- Never schedule past max weekly hours without manager flag

SOFT RULES (optimize for, note when violated):
- Match officer to preferred site when possible
- Distribute hours evenly across workforce
- Minimize overtime — schedule to 38hrs to leave 2hr buffer
- Pair new officers with experienced officers on first week at a site
- Honor recurring availability patterns (officer always works weekends, etc.)
- Balance armed/unarmed officer utilization across sites

BUSINESS RULES (pulled from org settings):
- Minimum coverage per site per shift (from client SLA)
- Specific officer requirements per client (client requested a specific officer)
- Blackout dates from org calendar
- Holiday staffing premiums
- On-call officer pool for gap coverage
```

// When a manager manually creates an open shift,

async function onOpenShiftCreated(shift: OpenShift) {

    avoidOvertime: true,

### 2A — PAYROLL GENERATION TRIGGER

```typescript
// Runs automatically at end of each pay period
// Also triggerable: "Trinity, generate payroll for this period"

async function trinityPayrollCycle(orgId: string, payPeriod: PayPeriod) {

  // STEP 1: Pull all approved timesheets for period
  const timesheets = await getApprovedTimesheets(orgId, payPeriod)

  // STEP 2: Pull schedule for cross-reference
  const schedule = await getScheduleForPeriod(orgId, payPeriod)

  // STEP 3: Identify discrepancies
  const discrepancies = await crossReferenceTimesheetVsSchedule(timesheets, schedule)
  // Flags: worked but not scheduled, scheduled but not worked,
  // clock time differs from shift time by more than threshold

  // STEP 4: Apply pay rules
  const payroll = await calculatePayroll(timesheets, {
    overtimeRule: org.overtimeRule,       // TX: weekly 40
    overtimeMultiplier: 1.5,
    payRates: await getEmployeePayRates(orgId),
    deductions: await getDeductions(orgId),
    taxWithholding: await getTaxSettings(orgId)
  })

  // STEP 5: Validate math
  const validation = await validatePayrollMath(payroll)
  // Cross-checks: total gross = sum of individual gross
  // No employee paid for unworked hours
  // OT calculated correctly per state rule
  // No negative net pay

  // STEP 6: Generate payroll draft
  const draft = await createPayrollDraft(payroll, orgId)

  // STEP 7: Flag exceptions for human review
  // Any discrepancy, any OT, any missing punch → human approval required

  // STEP 8: Notify owner for approval
  await notifyPayrollReady(orgId, {
    totalGross: payroll.totalGross,
    employeeCount: payroll.employees.length,
    otHours: payroll.totalOTHours,
    exceptions: discrepancies.length,
    approvalDeadline: payPeriod.payDate
  })
}
```

### 2B — PAYROLL MATH TRINITY MUST ALWAYS VERIFY

```
FOR EACH EMPLOYEE:
  Regular hours = hours up to weekly threshold (40 in TX)
  OT hours = hours above threshold
  Regular pay = regular hours × pay rate
  OT pay = OT hours × pay rate × OT multiplier
  Gross pay = regular pay + OT pay
  
  Employer burden:
  FICA employer share = gross × 0.0765
  FUTA = gross × 0.006 (first $7,000)
  TX SUI = gross × current TX SUI rate (check annually)
  Workers comp = gross × org workers comp rate
  
  Total cost to org = gross + all employer burden

CROSS-CHECKS TRINITY RUNS:
  Total payroll = sum(all employee gross) — must balance to penny
  Total burden = sum(all employer burden) — must balance to penny
  No shift has pay amount > bill amount (margin must be positive)
  OT shifts: margin checked individually — flag if OT inverts margin
```

async function trinityInvoiceCycle(orgId: string, billingPeriod: BillingPeriod) {

  // STEP 1: Pull completed shifts for period
  const completedShifts = await getCompletedShifts(orgId, billingPeriod)

  // STEP 2: Group by client
  const byClient = groupShiftsByClient(completedShifts)

  // STEP 3: For each client — apply their billing rules
  for (const client of byClient) {
    
    const clientSettings = await getClientBillingSettings(client.id)
    // Includes: bill rates by level, invoice cycle, payment terms,
    // billing contacts, PO number requirements, invoice format preference

    const invoice = await buildInvoice({
      client,
      shifts: client.shifts,
      billRates: clientSettings.rates,
      // Bill rate applied by officer level per shift
      // unarmed rate for unarmed shifts
      // armed rate for armed shifts  
      // supervisor rate for supervisor shifts
      period: billingPeriod,
      paymentTerms: clientSettings.invoiceDueNet,
      poNumber: clientSettings.poNumber
    })

    // STEP 4: Validate invoice math
    await validateInvoiceMath(invoice)
    // Every shift appears exactly once
    // Correct rate applied per officer level
    // Hours match approved timesheets
    // Total = sum of all line items (to the penny)

    // STEP 5: Generate PDF
    const pdf = await generateInvoicePdf(invoice)

    // STEP 6: Send to client per their billing contact settings
    await sendInvoice(invoice, pdf, clientSettings.billingContact)

    // STEP 7: Push to QuickBooks
    await pushInvoiceToQB(invoice, orgId)

    // STEP 8: Schedule follow-up if unpaid by due date
    await schedulePaymentFollowup(invoice, clientSettings.invoiceDueNet)
  }
}

INVOICE VALIDATION:
  No shift billed twice

  Client PO number included if required

OVERDUE TRACKING:

## PART 5: PROACTIVE INTELLIGENCE — WHAT TRINITY MUST DO WITHOUT BEING ASKED

---

These are the behaviors that make Trinity feel like a COO, not a chatbot.
Every item below is something Trinity runs on a schedule or event trigger — not waiting for a human to ask.

### DAILY (Runs every morning at 6 AM org local time)

```
- Scan today's schedule: any uncovered shifts? → Alert supervisor
- Scan clock-ins for overnight/early shifts: any missed punches? → Alert supervisor
- Check compliance: any expirations in next 7 days? → Alert manager
- Check overdue invoices: any past due date? → Send reminder or escalate
- Check pending approvals: any timesheets/shifts waiting > 24hrs? → Nudge approver
- Check open shifts: any unfilled for today? → Escalate immediately
- Weather check (if integrated): extreme weather at any client site? → Alert for coverage contingency
```

### WEEKLY (Runs every Monday at 7 AM)

```
- Run overtime risk scan: any officer projected to hit 40hrs by Thursday?
  → Suggest schedule adjustment before OT occurs
- Check next week's schedule completeness: any open shifts?
  → Auto-attempt to fill, escalate what can't be filled
- Run compliance scan: any expirations in next 30 days?
  → Notify officer directly with renewal instructions
- Generate weekly workforce summary for manager
- Check client SLA compliance: all sites covered per contract minimums?
- Scan shift marketplace: any offers pending > 48hrs? → Re-notify eligible officers
```

### MONTHLY (Runs on the 25th)

```
- Build next month's schedule (full autonomous cycle — Part 1A)
- Generate current period payroll draft
- Generate current period invoices for all clients
- Push invoices to QuickBooks
- Run compliance audit: all officers current for next month's work?
- Revenue forecast for next 30 days
- Client health scores: any at-risk clients?
- Generate executive summary for org owner
```

### EVENT-TRIGGERED (Real-time)

```
ON officer calls off:
  → Find replacement immediately
  → If no replacement found in 30 min → escalate to supervisor
  → Notify client if coverage will be affected

ON new open shift created:
  → Attempt fill within 60 seconds (Part 1D)

ON invoice payment received:
  → Mark paid, update QB, notify owner, update client health score

ON missed clock-in (15 min after shift start):
  → Text officer, alert supervisor, note on daily report

ON license/cert expiration:
  → Remove officer from future shifts at that post level
  → Notify officer with renewal steps
  → Notify manager

ON timesheet submitted:
  → Auto-review for anomalies
  → Auto-approve if clean
  → Flag and hold if discrepancy detected

ON incident report filed:
  → Route to supervisor immediately
  → Notify client contact if client property involved
  → Add to compliance review queue if use-of-force involved
```
