**DOCUMENT SIGNING UX ENHANCEMENT — Add to Phase 18 and Phase 23 notes in replit.md:**

The document signing experience must match enterprise-grade signing platforms in visual quality and mobile usability. Reference standard: Adobe Acrobat Sign / DocuSign quality on mobile.

**Required signing experience features — to be built or verified in Phase 23:**

**Mobile document viewer:**
Full-screen document rendering on mobile — document fills the viewport correctly, text is readable at default zoom without pinching, pages scroll vertically with smooth momentum scrolling. Document never overflows horizontally. Page indicator visible (Page 1 of 3). Navigation arrows or swipe between pages. Zoom in/out supported without breaking layout.

**Guided field navigation:**
Prominent "Next" button in the top-right corner (matches the reference UI) that jumps the user to the next required field in sequence. Fields highlighted in a distinct color (amber/gold for required, blue for optional) so the signer immediately knows where to go. "Start Signing" button that begins the guided flow from the first required field. Progress indicator showing fields completed vs remaining.

**Field types rendered correctly:**
- Signature field: clearly demarcated box with "Click to Sign" or "Tap to Sign" CTA, signer's name shown below the field
- Date field: auto-populated with today's date, editable, shown in correct format
- Initials field: smaller version of signature field
- Text field: clearly bordered input with label above
- Checkbox field: touch-friendly minimum 44x44px tap target
- Required field indicator: asterisk or red border until completed

**Recipient field assignment:**
Each field visually attributed to a specific recipient — color-coded by signer (Signer 1 in blue, Signer 2 in green, etc.). Fields assigned to the current signer are active and interactive. Fields assigned to other recipients shown as read-only with that recipient's name/color. Platform staff or document creator can assign fields to recipients during document preparation — drag-and-drop field placement on desktop, tap-to-place on mobile.

**Document preparation mode:**
When creating a document for signing the preparer gets a document editor view where they can: place signature fields by tapping/clicking the desired location, assign each field to a specific recipient by name, mark fields as required or optional, add instructional tooltips to complex fields, preview the document as each recipient will see it. This is the equivalent of the yellow highlighted sections in the reference — the preparer defines what needs attention.

**Signing completion:**
After all fields complete — clear confirmation screen: "Document signed successfully", document title, date and time, a summary of all signatories and their completion status. Option to download a copy immediately. NDS notification fires to all parties confirming execution. Executed document available in portal immediately.

**Typography and layout:**
Document text renders at minimum 14px on mobile — never smaller. Line height comfortable for reading (1.5). Font clean and professional — not monospaced or distorted. Headers, body text, and field labels visually distinct. No text truncation on any field label.
