## STEP 5: PAYROLL CALCULATION

For each officer, Trinity calculates:

```
FOR EACH OFFICER:
  regularHours = all hours up to 40/week per pay period week
  overtimeHours = all hours above 40/week
  regularPay = regularHours × payRate
  overtimePay = overtimeHours × payRate × 1.5
  grossPay = regularPay + overtimePay

  employerFICA = grossPay × 0.0765
  workersComp = grossPay × 0.032
  totalLaborCost = grossPay + employerFICA + workersComp

PAYROLL VERIFICATION CHECK:
  [ ] Every shift has a corresponding pay line
  [ ] Overtime calculated at 1.5x (not straight time)
  [ ] No officer paid for hours not on schedule
  [ ] Total payroll = sum of all officer gross pay (verify arithmetic)
  [ ] Flag any officer with 0 hours (were they supposed to work?)
  [ ] Flag any officer with >160 hours for the month (sanity check)
```

## STEP 7: MARGIN & MATH VERIFICATION

Trinity runs this final check across the entire month:

```
GLOBAL MATH VERIFICATION:

totalBillable = sum of all invoice totals across all clients
totalPayroll = sum of all officer gross pay
totalLaborBurden = sum of all employer taxes + workers comp
totalLaborCost = totalPayroll + totalLaborBurden
grossMargin = totalBillable - totalLaborCost
grossMarginPercent = (grossMargin / totalBillable) × 100

VERIFY:
[ ] grossMargin > 0 (org is not losing money)
[ ] grossMarginPercent >= 15% (minimum viable margin for security industry)
[ ] No individual shift has billAmount < payAmount (never bill less than you pay)
[ ] Overtime shifts: confirm OT pay increase is reflected — check margin
    doesn't go negative on any OT shift
[ ] Cross-check: totalBillable = sum(all client invoices)
[ ] Cross-check: totalPayroll = sum(all officer pay)
[ ] Both cross-checks must balance to the penny

IF ANY CHECK FAILS:
```
