**HOW IT WORKS TECHNICALLY**

```
SENDING (outbound):
  User hits Send in platform inbox UI
  Platform calls Resend API
  Resend delivers email from @coaileague.com address
  Sent copy stored in platform_emails table direction=outbound
  Thread linked via message_id and in_reply_to

RECEIVING (inbound):
  Someone sends email to any @coaileague.com address
  Resend catches it via MX records
  Resend fires webhook to POST /api/inbound/email
  Platform looks up email_routing table by to address
  Routes to correct user mailbox or Trinity for auto-processing
  Stored in platform_emails table direction=inbound
  User sees email in platform inbox via WebSocket
  Trinity generates one-line summary and draft reply

AUTO-PROCESSING by Trinity:
  calloffs-acme@coaileague.com → creates calloff_record
  incidents-acme@coaileague.com → creates incident_report
  support-acme@coaileague.com → creates support_ticket
  docs-acme@coaileague.com → stores in document vault
  trinity@coaileague.com → Trinity responds directly
```

**THE DNS CHANGES NEEDED**

This is what needs to change in the domain registrar for coaileague.com. Do these in the exact order listed. DO NOT change MX records until everything else is tested and working.

```
STEP 1 — UPDATE SPF RECORD (edit existing do not create new):
  Current value is probably:
    v=spf1 include:_spf.google.com ~all
  Change to:
    v=spf1 include:amazonses.com ~all
  Remove Google. Add Resend which uses Amazon SES.
  Confirm exact include value in Resend dashboard
  under Domains settings.

STEP 2 — ADD RESEND DKIM RECORD (new record):
  Type: TXT
  Name/Host: resend._domainkey.coaileague.com
  Value: get exact value from Resend Dashboard
         Domains > coaileague.com > DKIM section
  Also DELETE Google's DKIM entry:
    google._domainkey.coaileague.com — remove it entirely

STEP 3 — CREATE DMARC RECORD:
  Type: TXT
  Name/Host: _dmarc.coaileague.com
  Value: v=DMARC1; p=quarantine; rua=mailto:dmarc@coaileague.com; pct=100

STEP 4 — VERIFY coaileague.com IN RESEND:
  Resend Dashboard > Domains > Add Domain
  Enter coaileague.com
  Add each DNS record Resend provides
  Wait for Verified status before touching MX records

STEP 5 — ENABLE INBOUND ROUTING IN RESEND:
  Resend Dashboard > Inbound
  Enable wildcard routing for *.coaileague.com
  Set webhook URL:
    https://app.coaileague.com/api/inbound/email

STEP 6 — CHANGE MX RECORDS (do this LAST after everything tested):
  DELETE all existing Google Workspace MX records
  ADD Resend inbound MX records from Resend dashboard
  Go to: Resend Dashboard > Domains > coaileague.com > Inbound
  Copy exact MX hostname and priority Resend provides

STEP 7 — AFTER MX CHANGE:
  Remove mail.coaileague.com subdomain from DNS
  Remove mail.coaileague.com from Resend domain list
  coaileague.com becomes the only sending domain
```
