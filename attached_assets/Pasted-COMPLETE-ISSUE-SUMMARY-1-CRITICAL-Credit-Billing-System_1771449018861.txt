2. Chat Name/Role Display Bug
Problem: Dashboard shows "Marcus Rivera" but chat shows "Admin Marcus" with wrong role prefix.
Root cause: formatUserDisplayNameForChat() prepends role titles like "Admin" for platform staff. The dashboard uses formatUserDisplayName() which shows clean names. These two functions disagree.

3. Duplicate "New Update Available" Notifications
Problem: Two identical notifications showing.
Root cause: The What's New system queries the platformUpdates database table but has no deduplication by title - if the same update was inserted twice (e.g., by the event bus and seed), it shows twice.

4. "Not a Participant" Error When Leaving Help Desk
Problem: Users get a 403 error trying to leave rooms they should be in.
Root cause: The verifyParticipant() function in chat-management checks chatParticipants table, but Help Desk rooms use a different participant tracking mechanism (the ChatServerHub/WebSocket system). So the user IS in the room via WebSocket but NOT recorded in the chatParticipants database table.
