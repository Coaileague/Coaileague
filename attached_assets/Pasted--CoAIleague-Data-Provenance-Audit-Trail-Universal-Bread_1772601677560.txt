# CoAIleague — Data Provenance, Audit Trail & Universal Breadcrumb Architecture
## "Nothing Gets Lost. Everything Has a Home. Every Action Has a Trail."

**Date:** March 4, 2026
**Context:** CoAIleague has 109+ pages/routes, multiple AI agents (Trinity, ClockBot, ReportBot, MeetingBot, GuardBot, HelpBot), and dozens of CRUD operations. Every piece of data — whether created by a human, an AI bot, or the system itself — MUST be permanently attributed to: (1) the user who triggered it, (2) the org/workspace that owns it, and (3) an unbroken audit trail of every mutation. No orphaned data. No lost documents. No "who created this?" mysteries.

**THE RULE: If it touches the database, it leaves a breadcrumb.**

# PART 1: THE PROBLEM — HOW DATA GETS LOST
# ═══════════════════════════════════════════════════

## 1A: Common Data Loss Patterns in Large Apps

```
PATTERN 1: Missing workspace_id
  A route creates a record but doesn't attach workspace_id.
  Result: The record exists but doesn't belong to any org.
  Nobody can find it. It doesn't show up in any dashboard.
  The AI generated a report but it's floating in space.

PATTERN 2: Missing created_by / triggered_by
  A cron job or bot creates a document but doesn't record who/what triggered it.
  Result: "Where did this invoice come from?" Nobody knows.
  Was it auto-generated? Did a manager create it? Which bot?

PATTERN 3: Stale references
  An employee is deactivated but their documents, reports, clock records
  still reference their user_id. When the employee record is deleted (if ever),
  all those references break. Foreign keys fail or data becomes unqueryable.

PATTERN 4: Implicit org context
  Route assumes the org from the logged-in user's session.
  But a bot or cron job runs without a session. Now the route doesn't know
  which org context to use. Data gets created under the wrong org or no org.

PATTERN 5: AI-generated data with no attribution
  Trinity generates a report. ReportBot polishes an incident.
  MeetingBot creates minutes. The output goes to the document vault.
  But which bot created it? On behalf of which user? Triggered by what event?
  If the attribution chain is broken, the document is an orphan.

PATTERN 6: Cross-table consistency
  An invoice is created with line items, but the line items reference
  employees or shifts that have since been edited or deleted.
  The invoice now contains stale data that doesn't match current records.

PATTERN 7: File uploads without metadata
  A photo is uploaded but not attached to a shift room, incident, or employee.
  It sits in cloud storage consuming space with no way to find it.
```

## 1B: The Solution — Three Layers

```
LAYER 1: OWNERSHIP COLUMNS        → Every table knows who owns every row
LAYER 2: AUDIT TRAIL              → Every mutation is logged with who/what/when/why  
LAYER 3: TRINITY GATEWAY           → All AI actions flow through one orchestrator
                                     that enforces attribution before execution
```

## 2A: The Rule

**EVERY table in the database that stores org-specific data MUST have these columns:**

```sql
-- REQUIRED on every org-data table:
workspace_id    UUID NOT NULL REFERENCES workspaces(id),  -- Which org owns this
created_by      UUID REFERENCES users(id),                 -- Which user created it (NULL if system/bot)
created_by_type VARCHAR(20) NOT NULL DEFAULT 'user',       -- 'user', 'bot', 'system', 'cron', 'trinity'
created_by_bot  VARCHAR(30),                               -- If created_by_type = 'bot': which bot
updated_by      UUID REFERENCES users(id),                 -- Last user to edit (NULL if system/bot)
updated_by_type VARCHAR(20),                               -- Same enum as created_by_type
created_at      TIMESTAMP NOT NULL DEFAULT NOW(),
updated_at      TIMESTAMP NOT NULL DEFAULT NOW()
```

**ALSO REQUIRED on tables that reference employees, clients, or sites:**

```sql
-- When the record is "about" a specific employee:
employee_id     UUID REFERENCES users(id),                 -- Which employee this relates to

-- When the record is "about" a specific client/site:
client_id       UUID REFERENCES clients(id),               -- Which client/site this relates to
```

## 2D: Middleware Enforcement

To prevent ANY route from creating data without proper attribution, implement a middleware layer:

```typescript
// server/middleware/dataAttribution.ts

/**
 * Middleware that runs BEFORE every database write operation.
 * Ensures workspace_id, created_by, and created_by_type are set.
 * 
 * This is the SINGLE POINT OF ENFORCEMENT.
 * No route, no service, no cron job can bypass this.
 */

export function enforceAttribution(context: RequestContext) {
  return {
    // For INSERT operations
    beforeInsert(table: string, data: Record<string, any>) {
      // workspace_id is REQUIRED (except for global tables)
      const globalTables = ['workspaces', 'subscription_tiers', 'system_config'];
      if (!globalTables.includes(table)) {
        if (!data.workspace_id) {
          throw new Error(`ATTRIBUTION ERROR: Cannot insert into ${table} without workspace_id`);
        }
      }
      
      // created_by attribution
      if (context.type === 'user_request') {
        data.created_by = context.userId;
        data.created_by_type = 'user';
      } else if (context.type === 'bot_action') {
        data.created_by = null;
        data.created_by_type = 'bot';
        data.created_by_bot = context.botName; // 'clock_bot', 'report_bot', etc.
      } else if (context.type === 'cron_job') {
        data.created_by = null;
        data.created_by_type = 'cron';
      } else if (context.type === 'trinity') {
        data.created_by = null;
        data.created_by_type = 'trinity';
      } else if (context.type === 'system') {
        data.created_by = null;
        data.created_by_type = 'system';
      }
      
      // Timestamps
      data.created_at = data.created_at || new Date();
      data.updated_at = new Date();
      
      return data;
    },
    
    // For UPDATE operations
    beforeUpdate(table: string, data: Record<string, any>) {
      data.updated_at = new Date();
      
      if (context.type === 'user_request') {
        data.updated_by = context.userId;
        data.updated_by_type = 'user';
      } else if (context.type === 'bot_action') {
        data.updated_by = null;
        data.updated_by_type = 'bot';
      }
      // ... same pattern
      
      return data;
    },
    
    // For SELECT operations — scope to workspace
    beforeSelect(table: string, query: any) {
      const globalTables = ['workspaces', 'subscription_tiers', 'system_config'];
      if (!globalTables.includes(table) && context.workspaceId) {
        query.where.workspace_id = context.workspaceId;
      }
      return query;
    }
  };
}
```

**CRITICAL: This middleware must wrap the database client itself, not individual routes.** If it's applied at the route level, someone will forget to add it to a new route. If it wraps the DB client, it's impossible to bypass.

## 3A: The Audit Log Table

Every meaningful action across the entire platform is logged:

```sql
CREATE TABLE IF NOT EXISTS audit_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  
  -- WHO
  workspace_id UUID NOT NULL,                    -- Which org
  actor_id UUID,                                 -- Which user (NULL if bot/system)
  actor_type VARCHAR(20) NOT NULL,               -- 'user', 'bot', 'system', 'cron', 'trinity'
  actor_bot VARCHAR(30),                         -- Which bot (if actor_type = 'bot')
  actor_role VARCHAR(30),                        -- Role at time of action: 'owner', 'manager', 'field_worker', etc.
  actor_ip VARCHAR(45),                          -- IP address (for user actions)
  
  -- WHAT
  action VARCHAR(50) NOT NULL,                   -- The action performed (see action catalog below)
  entity_type VARCHAR(50) NOT NULL,              -- Which table/entity was affected
  entity_id UUID,                                -- ID of the specific record
  entity_name VARCHAR(300),                      -- Human-readable name (e.g., employee name, invoice #)
  
  -- HOW (what changed)
  change_type VARCHAR(10) NOT NULL,              -- 'create', 'update', 'delete', 'read', 'action'
  changes JSONB,                                 -- For updates: {field: {old: X, new: Y}, ...}
  metadata JSONB DEFAULT '{}',                   -- Additional context (route, page, request details)
  
  -- WHERE
  source_route VARCHAR(200),                     -- API route that triggered the action
  source_page VARCHAR(100),                      -- UI page that initiated the request
  
  -- WHEN
  created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

-- Indexes for fast querying
CREATE INDEX idx_audit_workspace ON audit_log(workspace_id, created_at DESC);
CREATE INDEX idx_audit_actor ON audit_log(actor_id, created_at DESC);
CREATE INDEX idx_audit_entity ON audit_log(entity_type, entity_id, created_at DESC);
CREATE INDEX idx_audit_action ON audit_log(action, created_at DESC);
CREATE INDEX idx_audit_bot ON audit_log(actor_bot, created_at DESC) WHERE actor_bot IS NOT NULL;

-- Partition by month for performance (optional but recommended at scale)
-- CREATE INDEX idx_audit_created ON audit_log(created_at);
```

## 3C: Audit Log Service

```typescript
// server/services/auditLogService.ts

interface AuditEntry {
  workspaceId: string;
  actorId?: string;            // User ID or null for bots
  actorType: 'user' | 'bot' | 'system' | 'cron' | 'trinity';
  actorBot?: string;           // Which bot
  actorRole?: string;          // Role at time of action
  actorIp?: string;
  action: string;              // From action catalog
  entityType: string;          // Table name
  entityId?: string;
  entityName?: string;         // Human-readable
  changeType: 'create' | 'update' | 'delete' | 'read' | 'action';
  changes?: Record<string, { old: any; new: any }>;
  metadata?: Record<string, any>;
  sourceRoute?: string;
  sourcePage?: string;
}

class AuditLogService {
  /**
   * Log an action. This should be called by every service layer function
   * that modifies data. NOT by routes directly — routes call services,
   * services call audit log.
   */
  async log(entry: AuditEntry): Promise<void> {
    await db.insert(auditLog).values({
      workspace_id: entry.workspaceId,
      actor_id: entry.actorId,
      actor_type: entry.actorType,
      actor_bot: entry.actorBot,
      actor_role: entry.actorRole,
      actor_ip: entry.actorIp,
      action: entry.action,
      entity_type: entry.entityType,
      entity_id: entry.entityId,
      entity_name: entry.entityName,
      change_type: entry.changeType,
      changes: entry.changes,
      metadata: entry.metadata,
      source_route: entry.sourceRoute,
      source_page: entry.sourcePage,
    });
  }
  
  /**
   * Query audit trail for an entity (e.g., "show me everything that 
   * happened to this employee" or "show me everything this bot did")
   */
  async getEntityHistory(entityType: string, entityId: string, limit = 50) {
    return db.select().from(auditLog)
      .where(and(
        eq(auditLog.entity_type, entityType),
        eq(auditLog.entity_id, entityId)
      ))
      .orderBy(desc(auditLog.created_at))
      .limit(limit);
  }
  
  /**
   * Query audit trail for a user
   */
  async getUserHistory(userId: string, limit = 50) {
    return db.select().from(auditLog)
      .where(eq(auditLog.actor_id, userId))
      .orderBy(desc(auditLog.created_at))
      .limit(limit);
  }
  
  /**
   * Query audit trail for a bot
   */
  async getBotHistory(botName: string, workspaceId: string, limit = 50) {
    return db.select().from(auditLog)
      .where(and(
        eq(auditLog.actor_bot, botName),
        eq(auditLog.workspace_id, workspaceId)
      ))
      .orderBy(desc(auditLog.created_at))
      .limit(limit);
  }
  
  /**
   * Detect changes between old and new record versions
   * Automatically generates the changes JSONB
   */
  static diffChanges(oldRecord: Record<string, any>, newRecord: Record<string, any>): Record<string, { old: any; new: any }> | null {
    const changes: Record<string, { old: any; new: any }> = {};
    const ignoreFields = ['updated_at', 'updated_by', 'updated_by_type'];
    
    for (const key of Object.keys(newRecord)) {
      if (ignoreFields.includes(key)) continue;
      if (JSON.stringify(oldRecord[key]) !== JSON.stringify(newRecord[key])) {
        changes[key] = { old: oldRecord[key], new: newRecord[key] };
      }
    }
    
    return Object.keys(changes).length > 0 ? changes : null;
  }
}
```

async updatePayRate(employeeId: string, newRate: number, context: RequestContext) {

## 4A: The Concept

Trinity isn't just the scheduling AI — she's the universal intelligence layer that oversees ALL data flow. Every bot reports to Trinity. Every AI action goes through Trinity. Trinity enforces:

1. **Attribution** — Before any bot acts, Trinity ensures workspace_id + actor info is attached
2. **Authorization** — Trinity checks: "Does this bot have permission to do this in this workspace?"
3. **Credit deduction** — Trinity deducts credits before allowing AI processing
4. **Audit logging** — Trinity logs every AI action through the audit service
5. **Error containment** — If a bot fails mid-action, Trinity rolls back cleanly

```
DATA FLOW THROUGH TRINITY:

  EVENT OCCURS                   TRINITY GATEWAY                    BOT EXECUTION
  (clock-in, message,            (validates, authorizes,            (specialized action)
   shift start, incident)         deducts credits, logs)
        │                              │                                  │
        ▼                              ▼                                  ▼
  Event detected ──────────► Trinity receives event ──────────► Bot performs action
                              │                                         │
                              ├─ Validates workspace_id exists          │
                              ├─ Validates actor attribution            │
                              ├─ Checks org has sufficient credits      │
                              ├─ Deducts credits (if AI action)         │
                              ├─ Logs to audit trail                    │
                              └─ Passes to appropriate bot              │
                                                                        │
                                                              Bot completes action
                                                                        │
                                                                        ▼
                                                              Trinity confirms success
                                                              Logs completion to audit
                                                              Updates entity records
```

## 4B: Trinity Orchestrator Service

```typescript
// server/services/trinityOrchestrator.ts

class TrinityOrchestrator {
  /**
   * ALL bot actions MUST go through this method.
   * This is the single enforcement point for attribution + credits + audit.
   */
  async executeAction(params: {
    workspaceId: string;
    botName: string;              // 'clock_bot', 'report_bot', 'meeting_bot', 'guard_bot'
    action: string;               // What the bot wants to do
    creditCost: number;           // Credits this action costs (0 for free actions)
    entityType: string;           // What entity is being affected
    entityId?: string;
    triggeredBy?: string;         // User ID if user-triggered, null if event-triggered
    triggerEvent?: string;        // 'shift_start', 'clock_in', 'message_detected', etc.
    payload: any;                 // Action-specific data
  }): Promise<ActionResult> {
    
    // STEP 1: Validate workspace exists
    const workspace = await this.getWorkspace(params.workspaceId);
    if (!workspace) {
      throw new Error(`Trinity: Invalid workspace ${params.workspaceId}`);
    }
    
    // STEP 2: Check credits (if action costs credits)
    if (params.creditCost > 0) {
      const balance = await creditBalanceService.getBalance(params.workspaceId);
      if (balance.total_credits < params.creditCost) {
        // Graceful degradation — don't hard-fail, degrade
        await auditLog.log({
          workspaceId: params.workspaceId,
          actorType: 'trinity',
          action: 'credits.insufficient',
          entityType: params.entityType,
          entityId: params.entityId,
          changeType: 'action',
          metadata: { required: params.creditCost, available: balance.total_credits, bot: params.botName },
        });
        
        return { 
          success: false, 
          degraded: true, 
          reason: 'insufficient_credits',
          fallbackAction: this.getGracefulFallback(params.action),
        };
      }
      
      // Deduct credits
      await creditBalanceService.deduct(params.workspaceId, params.creditCost, {
        feature_key: `bot.${params.botName}.${params.action}`,
        reference_id: params.entityId,
        reference_type: params.entityType,
        triggered_by: params.botName,
      });
    }
    
    // STEP 3: Execute the bot action
    const bot = this.getBot(params.botName);
    let result;
    
    try {
      result = await bot.execute(params.action, {
        workspaceId: params.workspaceId,
        entityType: params.entityType,
        entityId: params.entityId,
        payload: params.payload,
        // Attribution context — bot MUST use these when creating records
        attribution: {
          created_by: null,
          created_by_type: 'bot',
          created_by_bot: params.botName,
          workspace_id: params.workspaceId,
        },
      });
    } catch (error) {
      // STEP 3B: If bot fails, refund credits and log error
      if (params.creditCost > 0) {
        await creditBalanceService.refund(params.workspaceId, params.creditCost, {
          feature_key: `bot.${params.botName}.${params.action}.refund`,
          reference_id: params.entityId,
          triggered_by: 'trinity',
          description: `Refund: ${params.botName} action failed — ${error.message}`,
        });
      }
      
      await auditLog.log({
        workspaceId: params.workspaceId,
        actorType: 'trinity',
        action: 'bot.action_failed',
        entityType: params.entityType,
        entityId: params.entityId,
        changeType: 'action',
        metadata: { bot: params.botName, action: params.action, error: error.message },
      });
      
      throw error;
    }
    
    // STEP 4: Log successful action
    await auditLog.log({
      workspaceId: params.workspaceId,
      actorType: 'bot',
      actorBot: params.botName,
      action: `bot.${params.action}`,
      entityType: params.entityType,
      entityId: params.entityId,
      changeType: result.changeType || 'action',
      metadata: {
        trigger_event: params.triggerEvent,
        triggered_by_user: params.triggeredBy,
        credits_consumed: params.creditCost,
        result_summary: result.summary,
      },
    });
    
    return { success: true, data: result };
  }
}
```

// Database wrapper that enforces attribution on every operation
// This replaces direct db access throughout the app

import { db as rawDb } from './database';

export function createScopedDb(context: RequestContext) {
  return {
    insert(table: any) {
      return {
        values(data: any) {
          // Auto-inject attribution
          const enriched = {
            ...data,
            workspace_id: data.workspace_id || context.workspaceId,
            created_by: context.type === 'user' ? context.userId : null,
            created_by_type: context.type,
            created_by_bot: context.botName || null,
            created_at: new Date(),
            updated_at: new Date(),
          };
          
          // Validate workspace_id exists
          if (!enriched.workspace_id && !isGlobalTable(table)) {
            console.error(`ATTRIBUTION VIOLATION: Insert into ${table.name} without workspace_id`);
            throw new Error(`workspace_id required for ${table.name}`);
          }
          
          return rawDb.insert(table).values(enriched);
        }
      };
    },
    
    select() {
      // All selects auto-scoped to workspace
      return rawDb.select();
      // Note: Individual queries must still add .where(workspace_id = X)
      // but the middleware logs a warning if they don't
    },
    
    // ... update, delete with similar wrappers
  };
}

# ═══════════════════════════════════════════════════
# PART 7: SOFT DELETE — NEVER ACTUALLY DELETE ANYTHING
# ═══════════════════════════════════════════════════

## 7A: The Rule

**NOTHING is ever hard-deleted from the database.** Everything uses soft delete:

```sql
-- Every table that supports deletion must have:
is_deleted      BOOLEAN DEFAULT false,
deleted_at      TIMESTAMP,
deleted_by      UUID REFERENCES users(id),
deleted_by_type VARCHAR(20),                  -- 'user', 'system', 'cron'
deleted_reason  TEXT
```

**Why:** An employee who files a wage claim 2 years after termination needs their records. A client who disputes an invoice from 18 months ago needs the proof of service. If you hard-delete, the evidence is gone.

## 7B: How Soft Delete Works

```typescript
// NEVER do this:
await db.delete(employees).where(eq(employees.id, employeeId));  // ❌ NEVER

// ALWAYS do this:
await db.update(employees).set({
  is_deleted: true,
  deleted_at: new Date(),
  deleted_by: context.userId,
  deleted_by_type: 'user',
  deleted_reason: 'Employee terminated — voluntary resignation',
  updated_at: new Date(),
}).where(eq(employees.id, employeeId));

// All SELECT queries must exclude soft-deleted records by default:
await db.select().from(employees)
  .where(and(
    eq(employees.workspace_id, workspaceId),
    eq(employees.is_deleted, false)           // Always include this
  ));

// Except when explicitly looking at deleted records (admin/audit view):
await db.select().from(employees)
  .where(and(
    eq(employees.workspace_id, workspaceId),
    eq(employees.is_deleted, true)             // Show deleted only
  ));
```
