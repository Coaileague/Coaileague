## 1A. Create the Decision Log Schema

Every AI decision Trinity makes must be logged. This is non-negotiable for
enterprise customers who need audit trails and for us to improve Trinity's
intelligence over time.

**Create or update the `decision_logs` table with these fields:**

```
decision_logs:
  id                  UUID PRIMARY KEY
  timestamp           TIMESTAMP NOT NULL DEFAULT NOW()
  org_id              UUID NOT NULL (INDEXED — critical for multi-tenancy)
  workspace_id        UUID NOT NULL
  trigger_event       VARCHAR — what kicked this off ("calloff", "auto_schedule",
                      "compliance_check", "invoice_generate", "shift_fill")
  task_type           VARCHAR — classification ("scheduling", "compliance",
                      "payroll", "invoicing", "reporting", "communication")
  task_complexity     VARCHAR — ("simple", "operational", "strategic", "complex")
  
  # What Trinity considered
  context_data        JSONB — full context: the original request, relevant data
                      pulled, constraints identified
  candidates_evaluated JSONB — array of candidates with scoring:
                      [{
                        candidate_id: UUID,
                        name: STRING,
                        rank_score: FLOAT (0-1),
                        proximity_miles: FLOAT,
                        ot_risk: BOOLEAN,
                        ot_hours_projected: FLOAT,
                        compliance_status: "clear" | "expiring" | "expired" | "missing",
                        reliability_score: FLOAT (0-1),
                        cost_impact: FLOAT,
                        reasoning: STRING
                      }]
  
  # What Trinity decided
  selected_candidate_id  UUID (nullable — null if no selection made)
  decision_reasoning     TEXT — Trinity's explanation of why she chose this option
  confidence_score       FLOAT (0-1)
  
  # Triad Justice Layer — Claude as the Judge
  triad_review_triggered BOOLEAN DEFAULT FALSE
  judge_model            VARCHAR — "claude" or "openai" (whoever judged)
  original_score         FLOAT — Trinity's original confidence
  claude_verdict         VARCHAR — "AFFIRM", "OVERRIDE", "ESCALATE"
  claude_reasoning       TEXT — Claude's deliberation on why he agreed or overrode
  claude_suggested_alternative UUID (nullable — if OVERRIDE, who did Claude suggest?)
  
  # Outcome tracking (for learning)
  outcome_status         VARCHAR — "pending", "executed", "failed", "overridden_by_human"
  human_override         BOOLEAN DEFAULT FALSE
  human_override_by      UUID (nullable — who overrode)
  human_override_reason  TEXT (nullable)
  
  # Cost tracking
  primary_model          VARCHAR — "gemini", "claude", "openai"
  tokens_used            INTEGER
  cost_usd               FLOAT
  fallback_chain_used    BOOLEAN DEFAULT FALSE
  models_attempted       JSONB — ["gemini", "claude"] if fallback occurred

  created_at             TIMESTAMP DEFAULT NOW()
  updated_at             TIMESTAMP DEFAULT NOW()
```

**CRITICAL INDEXES:**
```sql
CREATE INDEX idx_decision_logs_org ON decision_logs(org_id);
CREATE INDEX idx_decision_logs_workspace ON decision_logs(workspace_id);
CREATE INDEX idx_decision_logs_trigger ON decision_logs(trigger_event);
CREATE INDEX idx_decision_logs_timestamp ON decision_logs(timestamp);
CREATE INDEX idx_decision_logs_triad ON decision_logs(triad_review_triggered);
```

**VERIFY AFTER CREATION:**
```
[ ] Table exists in database
[ ] All columns present with correct types
[ ] Indexes created
[ ] org_id is NOT NULL and indexed
[ ] Foreign key to workspaces table
```
